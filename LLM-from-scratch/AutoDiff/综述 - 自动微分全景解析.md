## 一、为什么我们需要自动微分？

### 1.1 四种微分方法的本质对比

从**第一性原理**看，计算导数只有四种可能的方法：

| 方法       | 核心思想                                                            | 致命缺陷                           |
| -------- | --------------------------------------------------------------- | ------------------------------ |
| **手动微分** | 人工推导公式并编码                                                       | 费时、易错、不可扩展                     |
| **数值微分** | $\frac{\partial f}{\partial x} \approx \frac{f(x+h) - f(x)}{h}$ | **精度与效率的双重困境**                 |
| **符号微分** | 计算机代数系统操作表达式                                                    | **Expression Swell**（表达式指数级膨胀） |
| **自动微分** | 程序化应用链式法则                                                       | 需理解原理才能正确使用                    |

### 1.2 数值微分的根本性困境

数值微分面临一个**不可调和的矛盾**：

**误差分析图**：
```
Error
 |    Round-off error dominant (舍入误差主导)
 |    /
 |   /
 |  /
 | / Truncation error dominant (截断误差主导)
 |/____________________ h (步长)
```

- **截断误差**：当 $h \to 0$ 时趋于零，但无法真正取 $h=0$
- **舍入误差**：当 $h$ 太小，$f(x+h)$ 与 $f(x)$ 在有限精度下几乎相同，有效位数丢失

**效率问题的数学本质**：

对于 $f: \mathbb{R}^n \to \mathbb{R}$（典型机器学习场景：$n$ 是百万级参数），数值微分需要：

$$\nabla f = \left[\frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, \ldots, \frac{\partial f}{\partial x_n}\right]$$

计算每个偏导数需要一次函数评估，总计 **$O(n)$ 次函数评估**！

**实验数据**（Helmholtz free energy, $n=43$）：
| 方法 | 相对时间 |
|------|----------|
| 原函数 | 1x |
| 数值微分 | **~1986x** |
| Forward AD | ~470x |
| **Reverse AD** | **~2x** |

### 1.3 符号微分的 Expression Swell 问题

以 **logistic map** 为例：
$$l_{n+1} = 4l_n(1-l_n), \quad l_1 = x$$

重复迭代几次后，$\frac{dl_n}{dx}$ 的**表达式会指数级膨胀**！

**根本原因**：符号微分试图构建完整的导数表达式，而不是数值。

## 二、自动微分的数学基础

### 2.1 核心洞察：程序 = 基本操作的复合

**第一性原理**：任何数值计算程序都是由**有限的基本操作**组成的，而每个基本操作的导数是已知的！

**基本操作集**：
- **二元算术**：$+, -, \times, \div$
- **一元操作**：符号反转、绝对值
- **超越函数**：$\exp, \log, \sin, \cos, \tan, \arcsin, \ldots$
- **线性代数**：矩阵乘法、转置、求逆、特征分解、SVD

### 2.2 计算图与 Wengert List

以 $y = f(x_1, x_2) = \ln(x_1) + x_1 x_2 - \sin(x_2)$ 为例：

**Evaluation Trace（Wengert List）**：
```
v_{-1} = x_1 = 2
v_0    = x_2 = 5
v_1    = ln(v_{-1}) = ln(2) ≈ 0.693
v_2    = v_{-1} × v_0 = 10
v_3    = sin(v_0) ≈ 0.959
v_4    = v_1 + v_2 = 10.693
v_5    = v_4 - v_3 ≈ 9.734
y      = v_5
```

这可以表示为**有向无环图（DAG）**：

```
    x_1 ─────┬─────────→ [×] ──→ v_2 ──┐
             │                         │
             ↓                         ↓
          [ln] → v_1 ──→ [+] ──→ v_4 ──→ [-] ──→ y
                                    ↑         ↑
    x_2 ────────────────→ [×] ──────┘         │
                │                             │
                └──→ [sin] → v_3 ─────────────┘
```

### 2.3 链式法则的程序化应用

**链式法则**（最核心的数学工具）：

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial x}$$

**扩展到计算图**：

$$\frac{\partial L}{\partial \theta} = \frac{\partial L}{\partial T_n} \cdot \frac{\partial T_n}{\partial T_{n-1}} \cdots \frac{\partial T_2}{\partial T_1} \cdot \frac{\partial T_1}{\partial \theta}$$

## 三、Forward Mode AD（前向模式）

### 3.1 Dual Numbers（对偶数）的数学基础

定义 **Dual Number** 为：
$$v + \dot{v}\epsilon, \quad \epsilon^2 = 0, \epsilon \neq 0$$

**类比**：虚数 $i^2 = -1$，对偶数 $\epsilon^2 = 0$

**关键性质**（自动导数计算）：
$$f(v + \dot{v}\epsilon) = f(v) + f'(v)\dot{v}\epsilon$$

**示例**：

对于 $y = x_1 \cdot x_2$：

$$(x_1 + \dot{x}_1\epsilon)(x_2 + \dot{x}_2\epsilon) = x_1 x_2 + (x_1\dot{x}_2 + \dot{x}_1 x_2)\epsilon$$

提取 $\epsilon$ 的系数即得导数！

### 3.2 Forward Tangent Trace

对于每个中间变量 $v_i$，引入**切线变量** $\dot{v}_i = \frac{\partial v_i}{\partial x}$：

```
\dot{v}_{-1} = \dot{x}_1 = 1      (设置 \dot{x}_1 = 1)
\dot{v}_0    = \dot{x}_2 = 0

\dot{v}_1    = \dot{v}_{-1} / v_{-1} = 1/2 = 0.5
\dot{v}_2    = \dot{v}_{-1} × v_0 + v_{-1} × \dot{v}_0 = 1×5 + 2×0 = 5
\dot{v}_3    = \dot{v}_0 × cos(v_0) = 0
\dot{v}_4    = \dot{v}_1 + \dot{v}_2 = 5.5
\dot{v}_5    = \dot{v}_4 - \dot{v}_3 = 5.5
```

**结果**：$\frac{\partial y}{\partial x_1} = \dot{v}_5 = 5.5$

### 3.3 复杂度分析

对于 $f: \mathbb{R}^n \to \mathbb{R}^m$：

- **Forward Mode**：需要 $n$ 次前向传播来计算完整 Jacobian
- 每次传播计算 Jacobian 的**一列**

**适用场景**：$n \ll m$（输入少，输出多）

## 四、Reverse Mode AD（反向模式）—— 机器学习的核心！

### 4.1 伴随变量的概念

**Adjoint（伴随变量）**：
$$\bar{v}_i = \frac{\partial y}{\partial v_i}$$

**物理意义**：输出 $y$ 对中间变量 $v_i$ 的**敏感度**

### 4.2 两阶段过程

**Phase 1: Forward Pass（前向传播）**
1. 正常执行程序
2. 记录所有中间变量的值
3. 构建/记录计算图

**Phase 2: Reverse Pass（反向传播）**
1. 初始化：$\bar{y} = 1$
2. 按计算图的**逆拓扑序**传播 adjoint
3. 应用链式法则：$\bar{v}_i = \sum_j \bar{v}_j \cdot \frac{\partial v_j}{\partial v_i}$

### 4.3 Reverse Adjoint Trace 示例

```
初始化: \bar{v}_5 = \bar{y} = 1

\bar{v}_4 = \bar{v}_5 × ∂v_5/∂v_4 = 1 × 1 = 1
\bar{v}_3 = \bar{v}_5 × ∂v_5/∂v_3 = 1 × (-1) = -1

\bar{v}_2 = \bar{v}_4 × ∂v_4/∂v_2 = 1 × 1 = 1
\bar{v}_1 = \bar{v}_4 × ∂v_4/∂v_1 = 1 × 1 = 1

\bar{v}_0 = \bar{v}_3 × cos(5) + \bar{v}_2 × 2 = -cos(5) + 2 ≈ 2.716
\bar{v}_{-1} = \bar{v}_1 × (1/2) + \bar{v}_2 × 5 = 0.5 + 5 = 5.5
```

**关键观察**：**一次反向传播同时计算了两个偏导数！**

### 4.4 复杂度优势的数学证明

对于 $f: \mathbb{R}^n \to \mathbb{R}^m$：

| 模式 | 传播次数 | 计算量 |
|------|----------|--------|
| Forward Mode | $n$ | $n \times \text{ops}(f)$ |
| **Reverse Mode** | **$m$** | $m \times c \times \text{ops}(f)$, $c \in [2, 3]$ |

**机器学习场景**：$f: \mathbb{R}^n \to \mathbb{R}$（标量 loss，百万参数）
- Forward Mode: $n$ 次传播
- **Reverse Mode: 1 次传播！**

这就是 **Reverse Mode AD = Backpropagation**！

### 4.5 Reverse Mode 的代价：内存开销

**核心问题**：需要存储所有中间变量！

内存需求 = $O(\text{计算图中节点数})$

这是深度学习中**显存爆炸**的根本原因！

## 五、AD 的两种实现方法

### 5.1 Tracing（追踪法）—— PyTorch, Autograd

**工作原理**：
```
def f(x):
    y = x * x
    return y

# 调用时：
# 1. 执行 f(x)，记录每个操作到 "tape"
# 2. 反向遍历 tape，应用链式法则
```

**优点**：
- 实现简单
- 支持动态控制流

**缺点**：
- 每次调用都要重新 trace
- 运行时开销
- 调试困难

### 5.2 Source Code Transformation（SCT）—— Tangent, Tapenade, Zygote

**工作原理**：
```
原始代码:
def f(x):
    y = x * x
    return y

↓ SCT 转换

梯度代码:
def df(x, dy=1.0):
    dx = 2 * x * dy
    return dx
```

**Tangent 的技术实现**：

```
┌─────────────────────────────────────────────────────────────┐
│                    Tangent Architecture                      │
├─────────────────────────────────────────────────────────────┤
│  用户代码                                                    │
│         ↓                                                   │
│  ┌─────────────────┐                                       │
│  │ AST Parser      │  Python 内置 ast 模块                │
│  └────────┬────────┘                                       │
│           ↓                                                 │
│  ┌─────────────────┐                                       │
│  │ Control Flow    │  构建控制流图                        │
│  │ Graph (CFG)     │                                       │
│  └────────┬────────┘                                       │
│           ↓                                                 │
│  ┌─────────────────┐                                       │
│  │ Activity        │  分析哪些变量影响输出                │
│  │ Analysis        │                                       │
│  └────────┬────────┘                                       │
│           ↓                                                 │
│  ┌─────────────────┐                                       │
│  │ Adjoint         │  每个操作的梯度模板                  │
│  │ Templates       │                                       │
│  └────────┬────────┘                                       │
│           ↓                                                 │
│  ┌─────────────────┐                                       │
│  │ AST Rewrite     │  生成新的 AST                        │
│  └────────┬────────┘                                       │
│           ↓                                                 │
│  梯度代码                                             │
└─────────────────────────────────────────────────────────────┘
```

**优点**：
- **零运行时开销**
- 生成的代码**可读、可调试**
- 支持 `with grad_of(x)` 语法注入自定义梯度逻辑

### 5.3 性能对比分析

```
Tracing (Autograd):
┌─────────────────────────────────────────────┐
│ 每次调用:                                    │
│   1. 执行前向，记录操作     │
│   2. 解释执行 tape         │
│   3. 数值计算                                │
└─────────────────────────────────────────────┘
总开销 = tracing + interpretation + computation

SCT (Tangent):
┌─────────────────────────────────────────────┐
│ 首次调用:                                    │
│   1. AST 解析和转换 (一次性)                 │

│                                              │
│ 后续调用:                                    │
│   1. 直接执行生成的代码                      │

总开销 = (transform_cost / num_calls) + computation

当 num_calls → ∞, overhead → 0
```

## 六、Zygote：Julia 的 Source-to-Source AD

### 6.1 微分算子的优雅设计

Zygote 定义了**微分算子** $J$：

$$J(f) := x \rightarrow (f(x), J_f(x) \cdot z)$$

**含义**：
- $J(f)(x)$ 返回元组 $(f(x), \text{Jacobian-vector product function})$
- $J_f(x) \cdot z$ 是 Jacobian 矩阵与向量 $z$ 的乘积

**梯度的定义**：
$$\nabla g(x) := [J(g)(x)]_2^\top(1)$$

### 6.2 可扩展的 $\partial$ 算子

**关键创新**：定义用户可扩展的 $\partial$ 算子

```julia
# 定义基本操作的导数
∂(+)(a::Real, b::Real) = a+b, z -> (z, z)
∂(*)(a::Real, b::Real) = a*b, z -> (z*b, a*z)
```

**意义**：
- AD 系统不依赖于新类型的原语定义
- 用户自定义梯度与系统提供的梯度使用**同一机制**
- 完美支持 Julia 的整个生态系统

### 6.3 零开销的保证

**Source-to-Source 变换**：
```
原始 Julia 代码 → 编译器 IR → 变换后的 IR → 编译 → 高效梯度函数
```

**性能数据**：
| 层数 | 总开销 | 操作数 | 每操作开销 |
|------|--------|--------|-----------|
| 1 | 147.0 μs | 255 | 576.3 ns |
| 2 | 280.5 μs | 491 | 571.3 ns |
| 3 | 406.1 μs | 727 | 558.6 ns |

**平均开销**：568.8 ns/操作（PyTorch 至少 1μs）

## 七、高级技术：解决 AD 的核心挑战

### 7.1 数值稳定的线性代数反向传播

**SVD 反向传播公式**（张量网络文章中的核心技术）：

$$\bar{A} = \frac{1}{2}U\left[F^+ \circ (U^T\bar{U} - \bar{U}^TU) + F^- \circ (V^T\bar{V} - \bar{V}^TV)\right]V^T + U\bar{D}V^T + \ldots$$

其中：
$$[F^\pm]_{ij} = \frac{1}{d_j - d_i} \pm \frac{1}{d_j + d_i}$$

**数值稳定性处理**：使用 **Lorentzian 展宽**
$$\frac{1}{x} \to \frac{x}{x^2 + \varepsilon}, \quad \varepsilon = 10^{-12}$$

### 7.2 Checkpointing：内存优化

**问题**：Reverse Mode 需要存储所有中间变量

**解决方案**：
```
前向传播：每 k 步存储一次张量
反向传播：需要时重新计算小段计算图

代价：计算时间最多翻倍
收益：内存大幅降低（BPTT 中可节省 95% 显存）
```

### 7.3 不动点迭代的反向传播

对于收敛到不动点 $T^*$ 的迭代：$T_{i+1} = f(T_i, \theta)$

**隐函数定理方法**：

$$\bar{\theta} = \bar{T^*} \cdot \left[I - \frac{\partial f(T^*, \theta)}{\partial T^*}\right]^{-1} \frac{\partial f(T^*, \theta)}{\partial \theta}$$

展开为**几何级数**：
$$\bar{\theta} = \sum_{n=0}^{\infty} \bar{T^*} \cdot \left[\frac{\partial f(T^*, \theta)}{\partial T^*}\right]^n \frac{\partial f(T^*, \theta)}{\partial \theta}$$

**优势**：
- 避免存储长链中间结果
- 收敛速率与前向迭代相同

### 7.4 高阶导数

**原理**：梯度本身也是计算图，可以再次应用 AD

**Hessian-向量积**（无需显式构造 Hessian）：
$$\sum_j \frac{\partial^2 L}{\partial \theta_i \partial \theta_j} x_j = \frac{\partial}{\partial \theta_i}\left(\sum_j \frac{\partial L}{\partial \theta_j} x_j\right)$$

$$\Delta w = -\eta \nabla L(w)$$

### 8.2 科学计算

**Adjoint Sensitivity Analysis**（ODE 系统）：

$$\frac{du}{dt} = f(u, p, t)$$

其 **adjoint equation**：
$$\frac{d\lambda^*}{dt} = \lambda^* \frac{\partial f}{\partial u} + \frac{\partial f}{\partial p}$$

**关键洞察**：$\lambda^* \frac{\partial f}{\partial u}$ 正是反向传播的**原始形式**！

| 传统方法 | 对解析梯度表达式近似收缩 | 传统方法 |

**应用**：反铁磁 Heisenberg 模型的 iPEPS 优化
- 达到 **state-of-the-art** 精度
- D=7 时相对误差 ~$10^{-6}$

### 8.4 可微分物理仿真

**优势**：训练后实现 **100× 加速**！

### 8.5 可微分光线追踪

**逆渲染问题**：
```
点光源位置 → 光线追踪器 → 渲染图像 → 与目标图像比较
```

## 十、第一性原理总结

### 10.1 AD 的本质

从最根本的角度看，**自动微分就是程序化的链式法则**：

1. **程序 = 基本操作的组合**：每个操作都有已知的导数
2. **链式法则**：连接各操作的导数
3. **Forward vs Reverse**：从输入端开始 vs 从输出端开始

### 10.2 为什么 Reverse Mode 是机器学习的核心？

**数学本质**：Reverse Mode 计算的是 **Transposed Jacobian-Vector Product**

$$J_f^\top \cdot r = \begin{bmatrix} \frac{\partial y_1}{\partial x_1} & \cdots & \frac{\partial y_m}{\partial x_1} \\ \vdots & \ddots & \vdots \\ \frac{\partial y_1}{\partial x_n} & \cdots & \frac{\partial y_m}{\partial x_n} \end{bmatrix} \begin{bmatrix} r_1 \\ \vdots \\ r_m \end{bmatrix}$$

对于 $f: \mathbb{R}^n \to \mathbb{R}$（$m=1$）：
- 一次反向传播得到完整 gradient
- 计算量是原函数的**常数倍**（2-3倍）

### 10.3 AD ≠ Backpropagation

**澄清**：
- **AD**：一系列计算导数的技术，适用于任意程序
- **Backpropagation**：Reverse Mode AD 在神经网络训练中的特例

**历史渊源**：
- **Pontryagin Maximum Principle** (1959)：控制论中的连续时间形式
- **Linnainmaa (1970, 1976)**：最早发表的 Reverse Mode 描述
- **Rumelhart et al. (1986)**：将 Backpropagation 引入机器学习社区

## 十一、关键公式汇总

### 11.1 链式法则

$$\frac{\partial L}{\partial x} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial x}$$

### 11.2 常见操作的梯度

| 操作 $y = f(x)$ | 梯度 $\frac{\partial L}{\partial x}$ |
|----------------|--------------------------------------|
| $y = x_1 + x_2$ | $\frac{\partial L}{\partial y}$ (各输入相同) |
| $y = x_1 \cdot x_2$ | $x_2 \cdot \frac{\partial L}{\partial y}$, $x_1 \cdot \frac{\partial L}{\partial y}$ |
| $y = \exp(x)$ | $\exp(x) \cdot \frac{\partial L}{\partial y}$ |
| $y = \log(x)$ | $\frac{1}{x} \cdot \frac{\partial L}{\partial y}$ |
| $y = \tanh(x)$ | $(1 - \tanh^2(x)) \cdot \frac{\partial L}{\partial y}$ |
| $y = \text{dot}(A, x)$ | $\frac{\partial L}{\partial y} \cdot x^T$, $A^T \cdot \frac{\partial L}{\partial y}$ |

### 11.3 Softmax Cross-Entropy

$$\frac{\partial L}{\partial x} = \text{softmax}(x) - y$$
