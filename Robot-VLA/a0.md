# A₀: 通用机器人操作的 Affordance-Aware 层次模型 详解

## 1. 论文核心 Motivation: 为什么需要 A₀?

当前 robotic manipulation 领域存在两条主要技术路线，它们在 spatial affordance 理解上各有缺陷：

**Modular-based methods**（如 ReKep [[1]](https://arxiv.org/abs/2409.01652), MOKA [[2]](https://moka-robot.github.io/)）直接调用 LVM (Large Vision Model) 像 SAM、GPT-4V，但没有真正学到 object operability 的 spatial representation，面对 wipe board 这种 trajectory-following 任务时表现脆弱。

**End-to-end VLA methods**（如 RDT [[3]](https://arxiv.org/abs/2410.07864), π₀ [[4]](https://arxiv.org/abs/2410.24164), OpenVLA [[5]](https://openvla.github.io/)）直接从 image+language 输出 action chunk，skip 了 explicit spatial reasoning。在 wipe board、stacking 等任务上，模型不知道 "擦哪里"、"按什么轨迹擦"，导致 success rate 在 trajectory 任务上几乎为 0（RDT 在 Wipe Board 是 0%，π₀ 是 10%）。

A₀ 的核心 insight：**把 manipulation 分解为两层 hierarchy —— 先理解 "在哪、怎么动" (spatial affordance)，再执行 (action)**。这把语义推理和 motor control 解耦，让 high-level 部分能跨 platform 泛化。

## 2. Embodiment-Agnostic Affordance Representation: 统一的 Affordance 表示

### 2.1 Representation 公式解析

论文 Eq.(1) 定义了一个统一 representation $\mathcal{R}$：

$$\mathcal{R} = \mathcal{R}_{\mathrm{R}} \cup \mathcal{R}_{\mathrm{H}} \cup \mathcal{R}_{\mathrm{C}} = \{(I, L, C, T)\}$$

其中：
- $\mathcal{R}_{\mathrm{R}}$: real-world 或 synthetic robotic data (如 DROID, ManiSkill)
- $\mathcal{R}_{\mathrm{H}}$: Hand-Object Interaction data (HOI4D)
- $\mathcal{R}_{\mathrm{C}}$: custom internet data (PixMo-Points)
- $I$: object-centric RGB image (current frame)
- $L$: natural language instruction
- $C = (c_0^{2D})$: **contact point**，2D 像素坐标，即 "where to grab"
- $T = (t_0^{2D}, t_1^{2D}, t_2^{2D}, \cdots)$: **post-contact trajectory waypoints**，2D 像素坐标序列，即 "how to move after contact"

**关键 intuition**: 把 robot action 抽象成 object-centric 的 2D 像素点序列，丢弃 robot-specific 的 joint angles、end-effector pose 等信息。这样无论是 Franka、Kinova 还是 Realman，只要它能看到同一个 object 并理解同样的 affordance，输出就一致。这实现了 **embodiment agnostic** 的核心思想。

### 2.2 数据来源的多样性

这个 representation 之所以能 scale up，是因为它可以从四类数据中提取：

| Dataset | Source | Samples | Annotation 方式 |
|---------|--------|---------|----------------|
| PixMo-One-Point | Internet images | 1,000,000 | 单个 contact point (来自 PixMo-Points [[6]](https://arxiv.org/abs/2409.17146)) |
| HOI4D-22k | Human-Object Interaction videos | 22,140 segments | 2D mask center per frame 作为 waypoint |
| DROID-3k | Real robot dataset | 3,056 trajectories | 人标 contact point + CoTracker [[7]](https://arxiv.org/abs/2307.07635) 跟踪轨迹 |
| ManiSkill-5k | Simulation | 4,965 trajectories | 3D 转 2D |

**这里有一个非常重要的设计 choice**: 用 HOI (人手操作物体) 数据训练 robot policy。为什么 work？因为 affordance 是 object property，人怎么抓杯子、机器人怎么抓杯子，contact point 在杯子 handle 上的位置是一致的。这极大扩展了训练数据池。

## 3. A₀ Model Architecture: Diffusion Transformer 的细节

### 3.1 整体数据流

参考论文 Figure 3 的架构：

```
Input:
  - $I_{t-1}$ (previous frame) + $I_t$ (current frame)  → SigLip (400M) [8]  → image tokens
  - $\ell$ (language instruction)                       → Qwen2.5-7B [9]     → text tokens
  - $k$ (diffusion timestep)                            → sinusoidal embedding
  - $\mathbf{x}^k$ (noisy waypoints)                    → input embedding

  ↓ N=28 层 DiT blocks (cross-attention + self-attention)

  ↓ Spatial Information Aggregation Layer (MLP decoder)

Output: $\mathbf{x}_{t:t+T} \in [0,1]^{2 \times T}$  (denoised waypoints)
```

### 3.2 Waypoint 的数学定义

Eq.(2) 定义了 waypoint 表示：

$$\mathbf{x}_{t:t+T}, \quad \text{where} \quad \mathbf{x}_t = (u, v) \in [0,1]^2 \subset \mathbb{R}^2$$

- $t$: trajectory 中的第 $t$ 个 timestep
- $T$: future step chunk size (论文中 $T=5$)
- $\mathbf{x}_t = (u, v)$: 归一化到 $[0,1]^2$ 的 2D 像素坐标（除以 image width/height）
- 第一个点 $\mathbf{x}_t$ 是 **start point (contact point)**，后续 $T-1$ 个点是 post-contact directional cues

**为什么用 diffusion 而不是直接 regression?** Diffusion policy 的优势在于：
1. **多模态分布建模**: 同一个 task 可以有多种合理 trajectory (e.g. 抓杯把 vs 抓杯壁)
2. **action space 的 uncertainty 处理**: 在 occlusion 或模糊 instruction 时输出 distribution 而非单点
3. 与 RDT、π₀ 等 diffusion-based VLA 一脉相承，但作用于 **2D pixel space** 而非高维 action space，这极大降低了学习难度

### 3.3 Position Offset Attention (POA) —— 核心 motion-aware 创新

这是 A₀ 的一个关键技术贡献。直觉：**robotic manipulation 是动态过程，单帧图像无法表达 motion，必须利用帧间差异**。

具体做法：

1. 用 SigLip 分别编码 $I_{t-1}$ 和 $I_t$，得到两组 tokens:
   - $I_{t-1}^1, I_{t-1}^2, \ldots, I_{t-1}^{N_S}$
   - $I_t^1, I_t^2, \ldots, I_t^{N_S}$
   
2. **逐 token 相减**得到 motion token:
   $$I_m^i = I_t^i - I_{t-1}^i$$
   
   这相当于在 token embedding 空间做帧差，让模型直接 access motion 信息，类似 video 模型中的 frame differencing，但在 feature space 操作。

3. Concatenate 得到最终视觉特征:
   $$o_t = \text{concat}([I_t^i, I_m^i], \text{dim}=1)$$

4. 加入 **sinusoidal positional embeddings in multi-dimensional grids**（来自 RDT [[3]](https://arxiv.org/abs/2410.07864) 的设计），让模型区分 viewpoint 和 timestep。

**Ablation 验证**（Table 1）：去掉 POA，ManiSkill-5k 上 MAE 从 5.5 → 6.3，DROID-3k 从 17.5 → 18.5。在 wiping、stacking 这种 motion-sensitive 任务上影响尤其明显。

### 3.4 Cross-attention 的交替注入

Image tokens 和 text tokens **交替注入** successive layers 的 cross-attention blocks。这是借鉴 RDT 的设计，目的是：
- 不同 modality 在不同层深度 fuse，避免单点 fusion 导致信息损失
- Compress 并 address different dimensions

整个模型学习的是条件分布：
$$p(\mathbf{x}_t \mid \ell, I_{t-1:t})$$

### 3.5 Spatial Information Aggregation Layer (SIAL)

这是 DiT blocks 之后的 **非线性 MLP decoder**，作用是把 latent space 的 representation 投影回 physical 2D space。

**Ablation 验证**（Table 1）：去掉 SIAL，HOI4D-22k 上 MAE 从 47.5 → 61.1（增加 13.2 pixels），ManiSkill-5k 从 5.5 → 10.2。**SIAL 的影响最大**，因为它负责 latent → coordinate 的精确映射，没有它，diffusion 输出的 latent 无法准确转化为像素坐标。

### 3.6 Inference: Diffusion ODE

Eq.(3) 定义了 diffusion 的反向 ODE：

$$\frac{d\mathbf{x}^k}{dk} := f(k)\mathbf{x}^k + \frac{g^2(k)}{2\sigma_k}\epsilon_\theta(\mathbf{x}^k, k), \quad \mathbf{x}^k \sim \mathcal{N}(0, \tilde{\sigma}^2 I)$$

- $k$: diffusion timestep（从 $K_F$ 到 0 反向）
- $f(k)\mathbf{x}^k$: **drift term**，$\mathbf{x}^k$ 的线性函数（来自 noise schedule）
- $g^2(k)$: **diffusion coefficient**，控制噪声强度
- $\sigma_k$: noise scale at timestep $k$
- $\epsilon_\theta(\mathbf{x}^k, k)$: neural network 预测的 noise（即 A₀ 的 DiT）
- $\frac{g^2(k)}{2\sigma_k}\epsilon_\theta$: **score function** 相关项，nonlinear 因为 $\epsilon_\theta$ 是神经网络

**关键 efficiency**: Forward 过程 $K_F = 1000$ 步，Inference 用 DPM-Solver [[10]](https://arxiv.org/abs/2206.00927) 只需 $K_D = 5$ 步 denoising。这让 single-shot inference 在毫秒级完成，对比 RDT/π₀ 的 step-by-step inference（25-50 步），execution 极快。

## 4. 两阶段训练策略

### 4.1 Stage 1: Pre-training on Contact Point Localization

**目标**: 学 foundational object localization 能力——给定 image + object label，预测 contact point。

- 数据: PixMo-One-Point 的 100 万张 internet images
- 输入: 仅用单张图 $I_t$（不需要 $I_{t-1}$）
- 输出: 只监督第一个 waypoint $\mathbf{x}_t^0$（归一化 2D 坐标）
- Loss (Eq.(4)):

$$\mathcal{L}_p(\theta) = \frac{1}{n}\sum_{i=1}^{n}\left((\mathbf{x}_t^0)_i - (f_\theta(k, \mathbf{x}_t^k, I_t, \ell))_i\right)^2$$

- $n$: batch size
- $(\mathbf{x}_t^0)_i$: 第 $i$ 个样本的 ground truth contact point
- $f_\theta(k, \mathbf{x}_t^k, I_t, \ell)$: 模型预测的 contact point
- $\mathbf{x}_t^k$: 加噪后的 waypoint

**训练成本**: 80,000 steps，5 days，4×A100 80GB，batch size 200，每卡 73GB memory。

### 4.2 Stage 2: Supervised Fine-tuning on Trajectories

**目标**: 扩展输出从单点到 $T$ waypoints，扩展 condition 从 object label 到 instruction。

- 数据: HOI4D-22k + DROID-3k + ManiSkill-5k
- 输入: $I_{t-1}, I_t$（启用 motion 信息）
- 输出: $T=5$ waypoints

Forward process 加噪：
$$\mathbf{x}_{t:t+T}^k = \sqrt{\bar{\alpha}^k}\mathbf{x}_{t:t+T}^0 + \sqrt{1-\bar{\alpha}^k}\epsilon^k$$

- $\bar{\alpha}^k = \prod_{s=1}^{k}\alpha_s$: cumulative product of noise schedule coefficients
- $\epsilon^k \sim \mathcal{N}(0, \mathbf{I})$: 标准高斯噪声
- $k \in \mathcal{K} \subset \mathbb{N}^+$: 从 forward timestep 集合采样

Loss (Eq.(5)):
$$\mathcal{L}_s(\theta) = \frac{1}{n}\sum_{i=1}^{n}\left((\mathbf{x}_{t:t+T}^0)_i - (f_\theta(k, \mathbf{x}_{t:t+T}^k, I_{t-1:t}, \ell))_i\right)^2$$

**训练成本**: 30,000 steps，50 hours。

### 4.3 Pre-training 的效果（Figure 4）

在三个数据集上做了 Real-to-Sim 和 Sim-to-Real 测试，pre-training 始终降低 MAE。这表明 internet-scale contact point pre-training 提供了 **object affordance 的 general foundation**，类似 LLM 的 pre-training → fine-tuning paradigm。

## 5. Action Execution: 从 2D 到 SE(3) 的桥梁

A₀ 只负责输出 2D pixel waypoints，但 robot 需要在 3D 空间执行。这是一个 modular 的 downstream module。

### 5.1 2D-to-3D Projection (Eq.(6))

$$X_i = D(\mathbf{x}_i) K^{-1}\tilde{\mathbf{x}}_i, \quad i = t+1, t+2, \ldots, t+T$$

- $D(\mathbf{x}_i)$: depth map 在 pixel $\mathbf{x}_i$ 处的深度值（米）
- $K \in \mathbb{R}^{3\times3}$: camera intrinsic matrix，形式为
  $$K = \begin{pmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{pmatrix}$$
  其中 $f_x, f_y$ 是 focal length，$c_x, c_y$ 是 principal point
- $\tilde{\mathbf{x}}_i = (u_i, v_i, 1)^T$: pixel $\mathbf{x}_i = (u_i, v_i)$ 的齐次坐标
- $X_i \in \mathbb{R}^3$: 3D point in camera frame

这是标准的 pinhole camera back-projection。

### 5.2 Grasp Pose Estimation (Eq.(7))

从 GraspNet [[11]](https://graspnet.net/) 或其他 sampler 生成 grasp candidate 集合 $\mathcal{G}$，选最接近 projected contact point $X_t$ 的：

$$G^* = \arg\min_{G \in \mathcal{G}} \|G - X_t\|$$

### 5.3 Waypoint Height Selection

post-contact waypoints $\{\mathbf{x}_2, \ldots, \mathbf{x}_T\}$ 投影到 3D 后，**自由空间中的高度**需要额外决定。论文采用 Set-of-Mark 风格 [[12]](https://arxiv.org/abs/2310.11441) 的 discrete selection：
- VLM 被 prompt 选择 height category (e.g. "at target level" 或 "above target")
- 根据 category 采样最终 3D waypoint

最终 grasp pose + 3D waypoints 在 SE(3) 空间生成 motion trajectory，由 robot controller 执行。

| Kinova | RDT | 20 | 0 | 25 | 0 | 11.25 |
| Kinova | **A₀-1B** | **60** | **65** | **40** | **50** | **53.75** |

| Franka | **A₀-1B** | **60** | **75** | **70** | **45** | **62.50** |

**关键观察**：

1. **Wipe Board 是 trajectory-following 的 stress test**: ReKep 在 Kinova 上 Wipe Board = 0%，RDT = 0%，π₀ = 10%（Table 3），但 A₀ = 50%。这直接证明了 A₀ 的 hierarchical affordance design 在 trajectory 任务上的优越性——它显式预测了 post-contact waypoints，而 end-to-end VLA 没有这种 spatial planning。

2. **Platform agnostic 体现**: 同一个 A₀-1B 模型在 Franka (62.50%) 和 Kinova (53.75%) 上都达到 SOTA，**不需要为不同 robot 重新训练**。这是因为 affordance representation 是 object-centric 2D points，与 robot kinematics 无关。

3. **Open Drawer 任务优势明显**: A₀ 在 Franka 上达到 75%，远超 MOKA 的 50%、Molmo 的 40%。Drawer 操作需要准确预测 handle 的 contact point + 正确的 pulling trajectory，A₀ 的设计恰好覆盖这两个层面。

### 6.2 对比 End-to-End VLA（Table 3）

| Method | Place | Open | Press | Wipe | Steps | Avg |
|--------|-------|------|-------|------|-------|-----|
| RDT-1B | 20 | 0 | 25 | 0 | 25-50 | 11.25 |

**Steps 列揭示了 A₀ 的根本效率优势**: 
- VLA methods: 每一步都 model inference，25-50 步完成一个 task
- A₀: single-shot inference，一次预测 4-5 个 keypoints，后续由 planner 执行

这相当于把 model inference 从 50 次降到 1 次，**inference 成本降低 25-50 倍**。对于 real-time robot control 这是巨大的 practical 优势。

## 7. Limitations 与 Future Work

### 7.1 当前 Limitations

1. **依赖外部 grasp sampler**: GraspNet 等 sampler 的 candidate 集合 $\mathcal{G}$ 限制了 final action quality。A₀ 没有学习 grasp pose，只是 select nearest candidate。
2. **Depth map 依赖**: 自由空间高度估计需要 depth + VLM prompt，在 occlusion 场景下 fail。
3. **Orientation-sensitive tasks**: A₀ 输出只有 2D (u, v)，丢失了 6D pose 信息。对于 pouring、revolute drawer opening 这种任务，需要选择 optimal viewpoint，或借助 ATM [[14]](https://arxiv.org/abs/2410.05811) 风格的 track-guided policy。

### 7.2 Future Work 方向

1. **VLM-assisted grasp selection**: 用 VLM 从 grasp candidates 中 visually select 最佳 gripper position
2. **Direct height prediction**: 把 depth、gripper length 等作为 condition，直接 predict height
3. **Long-horizon planning**: 用 GPT-4o 分解长程任务（类似 π₀.₅ [[15]](https://www.physicalintelligence.company/blog/pi05）），每个 subtask 由 A₀ 执行

### 7.3 VLM-assisted Complex Tasks（Figure 8）

论文展示了一个 extension: 用 VLM 做 high-level planning，A₀ 做 low-level execution。例如:
- "put fruits on a plate" → VLM 分解为 "pick apple" + "place on plate" → A₀ 逐步执行
- "insert flowers into bottle" → VLM 分解为 "grasp flower" + "align with bottle mouth" + "insert" → A₀ 执行

这是 hierarchical system 的进一步延伸，A₀ 作为可插拔的 spatial affordance module。

## 8. 与相关工作的 positioning

### 8.1 vs. Point-based Methods (SpatialVLA, Robopoint, Track2Act)

- Robopoint: 输出 interaction region（多点分布），精度低，没有 trajectory
- Track2Act [[16]](https://arxiv.org/abs/2405.01527): 用 internet video 的 point track 作为 policy，但 track 本身不是 affordance，可能包含 irrelevant motion
- A₀: 输出 **structured** contact point + post-contact trajectory，object-centric，可解释

### 8.2 vs. Flow-based Methods (General Flow, Im2Flow2Act)

- General Flow [[17]](https://chengeccn.github.io/generalflow/): dense optical flow as affordance，computationally expensive，embodiment-specific
- A₀: sparse keypoints (5 个)，computationally light，embodiment-agnostic

### 8.3 vs. Hierarchical Models (Helix, Dex-GraspNet)

- Helix [[18]](https://www.figure.ai/): VLM 学 latent semantic representation，不 explicit object-centric spatial affordance
- A₀: explicit 2D pixel affordance，更 interpretable，更 controllable

### 8.4 vs. End-to-End VLA (RDT, π₀, OpenVLA)

| 维度 | VLA (RDT/π₀) | A₀ |
|------|--------------|-----|
| Output space | High-dim action (128D for RDT) | 2D pixel (10D for T=5) |
| Spatial reasoning | Implicit | Explicit |
| Embodiment | Per-robot training | Agnostic |
| Inference steps | 25-50 | 1 (single-shot) |
| Wipe Board | 0-10% | 45-50% |
| Long-horizon | 内部 struggle | 需 VLM 分解 |

## 9. Build Intuition: A₀ 的核心设计哲学

### 9.1 为什么 2D pixel space 是 "甜点"？

考虑三个空间：
- **3D action space**: 高维，包含 robot kinematics，embodiment-specific
- **2D pixel space**: 中等维度，object-centric，camera-view-dependent 但 embodiment-agnostic
- **Latent action space**: 低维，但需要大量 robot data 学习

A₀ 选择 2D pixel space 是 trade-off 的最优解：
1. Internet data (PixMo) 只提供 2D annotation，可以直接利用
2. HOI videos 自然投影到 2D
3. 不同的 robot 都能用 RGB-D camera 观察同一个 scene，输出一致
4. Depth + camera intrinsic 足以 back-project 到 3D，不损失信息

### 9.2 为什么 Diffusion 而非 Regression?

考虑一个 task: "open the drawer"。Drawer handle 的 contact point 在图像中是确定的（一个点），但 pulling trajectory 有多模态性：
- 向左 pull
- 向右 pull
- 向前 pull
都符合 task。Regression 输出单点会 average 这些 mode，得到不合理的中间 trajectory。Diffusion 自然建模 multi-modal distribution。

### 9.3 为什么 Hierarchical 而非 End-to-End?

End-to-End VLA 把 perception、planning、control 压缩到一个 network，所有 inference 都需要 forward 整个网络。A₀ 分离：
- High-level (A₀): semantic + spatial reasoning，重 but 只跑一次
- Low-level (action execution): grasp selection + trajectory generation + robot controller，轻 但跑很多次

这类似 human cognition: 大脑 plan "我要抓那个杯子"，然后小脑和肌肉执行 smooth motion，不需要每一步都重新 plan。

### 9.4 Pre-training → Fine-tuning 的 data scaling law

A₀ 复刻了 LLM 的 scaling paradigm:
- Pre-training: 1M internet images + object label → contact point（foundational localization）
- Fine-tuning: ~30k trajectory samples → 5 waypoints（task-specific dynamics）

这意味着对于新 task，只需要少量 trajectory annotation，因为 spatial foundation 已经建立。这是 **data-efficient deployment** 的关键。

## 10. 我的 Critical Thoughts

### 10.1 优势
1. ** elegant decomposition**: spatial reasoning 与 motor control 解耦，每层都可以独立优化
2. **Data efficiency**: 利用 internet + HOI data，避开 robot data 稀缺问题
3. **Platform transferability**: 实验证明 Franka/Kinova/Realman/Dobot 都能用
4. **Inference speed**: single-shot vs. step-by-step，real-time 可行

### 10.2 潜在问题
1. **2D 表示的信息损失**: 6D pose、force、tactile 信息无法表达。Pouring、in-hand rotation 这种 task 可能不适合
2. **Depth + VLM 的 robustness**: 在 specular/transparent 物体上 depth 不可靠
3. **Single camera viewpoint dependency**: 多视角场景需要重新设计
4. **Grasp candidate 依赖**: 如果 GraspNet 在某 object 上 fail，A₀ 也 fail
5. **Long-horizon 仍需 VLM 外挂**: 没有内置的 subtask decomposition

### 10.3 与最新 trend 的关联

A₀ 的思路与以下几个最新方向共鸣：
- **π₀.₅**: hierarchical VLA + flow matching，A₀ 是其 affordance-centric 版本
- **ATM (Any-point Trajectory Modeling)** [[14]](https://arxiv.org/abs/2410.05811): 用 track 学 policy，A₀ 用 predicted waypoints 类似
- **VLM as planner**: 如 RoboBridge [[19]](https://arxiv.org/abs/2505.01709) 的 cognitive-bridging 设计
- **SpatialVLA** [[20]](https://arxiv.org/abs/2501.15830): 显式学 spatial representation，A₀ 更激进地只用 2D point
