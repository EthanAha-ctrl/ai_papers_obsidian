## 一、第一性原理：核心问题是什么？

> **数据规模多样性与行为质量一致性的矛盾**

传统 VLA 训练只使用高质量 demonstration data（人类遥操作数据），因为如果直接混合低质量数据（失败案例、自主执行数据），模型会把不同策略"平均"起来，产生次优行为——这就是所谓 **mode averaging** 问题。

> **如果你能精确地告诉模型"这段数据是怎么来的、质量如何"，那么即使低质量数据也能成为有用的训练信号——因为模型学会了"什么是好的、什么是坏的"，从而在推理时选择好的行为。**

这和 LLM 中 **prompt expansion** 的思想一脉相承：给模型更丰富的 context，不仅说"做什么"（what），还说"怎么做"（how），模型就能从更广的数据中提取更结构化的知识，并产生组合泛化（compositional generalization）。

## 二、模型架构详解

### 2.1 总体架构（5B 参数）

```
┌─────────────────────────────────────────────────────┐
│                    π₀.7 Architecture                 │
│                                                      │
│  ┌──────────────┐   ┌──────────────┐                │
│  │  VLM Backbone │   │  MEM Video   │                │
│  │  (Gemma3 4B)  │   │  History     │                │
│  │  (含400M      │   │  Encoder     │                │
│  │   Vision Enc) │   │  (400M params)│               │
│  └──────┬───────┘   └──────┬───────┘                │
│         │                  │                          │
│         └───────┬──────────┘                          │
│                 │                                     │
│         ┌───────▼───────┐                            │
│         │  Action Expert │  860M params              │
│         │  (Flow Matching)│  50 tokens → 50-step    │
│         │                │  action chunk              │
│         └───────────────┘                             │
│                                                      │
│  Total: ~5B parameters                               │
└─────────────────────────────────────────────────────┘
```

**各模块细节：**

| 组件 | 参数量 | 初始化来源 | 功能 |
|------|--------|-----------|------|
| VLM Backbone | ~4B | Gemma3 4B [106] | 理解语言、图像、生成 FAST tokens |
| Vision Encoder (MEM) | ~400M | Gemma3 | 处理多帧历史观测 + subgoal images |
| Action Expert | 860M | 从头训练 | Flow matching 预测连续动作 |

### 2.2 输入处理

**观测输入**：
- 最多 4 个相机（front, 2×wrist, 可选 rear）
- 每个相机最多 6 帧历史（stride = 1秒）
- 最多 3 个 subgoal images（省略 rear view）
- 所有图像 resize 到 448×448

**Proprioception**：
- 和 π₀.6 不同（使用 discretized text tokens 表示 $q_t$），π₀.7 使用 **linear projection** 将状态维度映射到 backbone 维度
- 每个历史状态是一个独立 token

### 2.3 Attention Pattern（关键设计）

π₀.7 使用 **block-causal masking**：

```
Observation tokens ←→ 双向注意力（内部）
Subgoal image tokens ←→ 双向注意力（内部），且可以 attend 到 observation
Text tokens → 因果注意力（causal）
Action expert tokens ←→ 双向注意力（内部），可以 attend 到 VLM backbone
```

这个设计的关键 intuition：观测和 subgoal 图像之间需要全局理解（所以双向），但 text 输出需要因果性（所以 causal），action tokens 需要互相协调（所以双向）。

### 2.4 Knowledge Insulation (KI) 训练策略

这是来自 [103] 的关键技术：

- **VLM backbone** 使用 **FAST tokens** [104] 监督（离散 cross-entropy loss）
- **Action expert** 可以 attend 到 VLM backbone 的所有激活
- **但 action expert 的梯度不回传到 VLM backbone**

为什么要这样？因为 flow matching loss 波动大，如果直接反传会破坏 VLM 已经学好的语义表示。KI 让 VLM 通过更稳定的离散 loss 学习，action expert 则从 VLM 的激活中"读取"信息。

### 2.5 Flow Matching Action Expert

Action expert 用 **flow matching** [102] 目标训练，而非传统 diffusion：

给定当前噪声状态 $a^{(k)}$（flow step $k$），目标是最小化：

$$\mathcal{L}_{\text{CFM}} = \mathbb{E}_{k, \epsilon} \left[ \| v_\theta(a^{(k)}, k, o_{t-T:t}, C_t) - (a_{t:t+H} - \epsilon) \|^2 \right]$$

其中：
- $a_{t:t+H}$ 是未来 $H=50$ 步的 ground-truth action chunk
- $\epsilon \sim \mathcal{N}(0, I)$ 是噪声
- $a^{(k)} = (1-k) \cdot \epsilon + k \cdot a_{t:t+H}$ 是 flow 插值
- $v_\theta$ 是模型预测的 velocity field
- $k \sim \text{Uniform}(0,1)$ 是 flow timestep

推理时用 5 步 denoising，执行 $\hat{H} \in \{15, 25\}$ 步。

使用 **Adaptive RMSNorm** 注入 flow timestep 信息。

### 2.6 训练时 Real-Time Chunking (RTC)

模拟推理时 0-12 步的延迟（对应 50Hz 机器人上最大 240ms 的推理延迟），使模型学会在已有部分执行的 action chunk 基础上生成新的平滑轨迹。

## 三、π₀.7 的核心创新：Diverse Prompting

### 3.1 传统 VLA 的 Prompt

传统方法：
$$C_t = (\ell_t)$$
只有语言指令，比如"clean up the kitchen"。

### 3.2 π₀.7 的扩展 Prompt

$$C_t = (\ell_t, \hat{\ell}_t, g_t, m, c)$$

各组件详解：

#### (A) Subtask Instructions $\hat{\ell}_t$

不仅给整体任务描述（$\ell_t$ = "clean the kitchen"），还给**当前语义子任务**（$\hat{\ell}_t$ = "open the fridge door"）。

**推理时**，$\hat{\ell}_t$ 由学到的 **high-level policy** 或人类 supervisor 提供。

**关键应用 — Verbal Coaching**：
因为 π₀.7 能跟随多样语言指令，人类可以通过逐步语言指令"教练"它执行新任务（如使用 air fryer），然后把这些 coaching 数据用来训练 high-level policy，使 π₀.7 能自主执行新任务。

#### (B) Subgoal Images $g_t$

**多视角 subgoal**：$g_t = [G_t^1, \ldots, G_t^n]$，其中 $G_t^i$ 是第 $i$ 个相机的期望未来图像。

**为什么需要 subgoal images？**
- 语言有歧义："open the fridge door" 没说抓手应该怎么抓
- Subgoal image 直接描绘了期望的未来状态
- Base view 擅长表达环境/物体目标，wrist view 擅长表达手/夹爪目标

**World Model $g_\psi$**：
- 基于 **BAGEL** [105]，14B Mixture-of-Transformers 模型
- 训练目标：
$$\max_\psi \mathbb{E}_{\mathcal{D}_g} [\mathcal{L}_{\text{CFM}}(g_t^\star, g_\psi(o_t, \hat{\ell}_t, m))]$$
- $g_t^\star = o_{t_{\text{end}}}$（segment 最后一帧作为 ground-truth subgoal）
- 混合了 robot data、egocentric human video、web data、image editing datasets

**训练时的 subgoal 采样策略**：
- 25% 概率：segment 末尾帧
- 75% 概率：从当前时间步前 0-4 秒均匀采样
- 额外加入 **world model 生成的 subgoal images**，减少 train-test mismatch

#### (C) Episode Metadata $m$

这是 π₀.7 最关键的 innovation 之一：

| Metadata | 含义 | 格式 |
|----------|------|------|
| Overall Speed | Episode 长度（timesteps） | 离散化为 500 步间隔，如 "2000 steps" |
| Overall Quality | 任务执行质量 | 1-5 分，5 = 最高 |
| Mistake | 当前 segment 是否有错误 | True/False |

**推理时的 metadata 设置**：
- Speed：设为该任务数据的 15th percentile（追求快速执行）
- Quality：始终设为 5（最高质量）
- Mistake：始终设为 false（无错误）

**直觉**：通过告诉模型"请以高质量、快速度、无错误的方式执行"，模型就知道要从训练数据中选择最优策略模式，而不是平均所有策略。

#### (D) Control Mode $c$

$c \in \{\text{joint}, \text{ee}\}$，指定使用关节空间还是末端执行器控制。

#### (E) 训练时 Dropout 策略

| 组件 | Dropout 概率 |
|------|-------------|
| Subgoal images | 75% 时不使用（只 25% 有） |
| Subtask instruction $\hat{\ell}_t$（有 subgoal 时） | 30% drop |
| Episode metadata 整体 | 15% drop |
| Speed / Quality / Mistake 各个 | 各 5% drop |
| Control mode | 不 drop |

这样推理时可以灵活组合 prompt 组件。

### 4.2 关键创新：使用次优数据

**π₀.7 大量使用 suboptimal robot data**：
- 低质量 demonstration（失败 episodes、多错误 episodes）
- 先前模型（π*₀.6）在 RL 训练中收集的自主数据

**这相当于一种"蒸馏"**：generalist π₀.7 模型可以继承 RL-trained specialist 的能力，因为：
- 高质量自主数据被标注为 Quality=5, Mistake=false
- 低质量数据被标注为 Quality=1, Mistake=true
- 推理时请求 Quality=5，模型就知道要模仿高质量行为

### 4.3 数据量与多样性

- 覆盖多种机器人平台（bimanual mobile manipulator, static bimanual "BiPi", bimanual UR5e, single-arm）
- 多种环境（lab-like, home-like, in-the-wild homes）
- **排除了所有 generalization evaluation 任务的自主数据**（确保公平评估）

## 五、推理流程

### Algorithm 1 详解

```
输入: 初始观测 o₀, 任务指令 ℓ, episode metadata m, 控制模式 c

1. 初始化子任务 ℓ̂（来自 high-level policy 或 coaching）
2. 生成 subgoal image: g* ~ p_ψ(g*|o₀, ℓ̂, m)
3. 构建完整 prompt: C = {ℓ, ℓ̂, g*, m, c}
4. 生成 action chunk: a_{t:t+H} ~ π_θ(a|o_{t-T:t}, C)
   [可选: 使用 CFG]

5. for t = 0, 1, 2, ... do:
   a. if ℓ̂ 改变 OR ∆=4秒计时器到期:
      → 异步生成新 subgoal image
   b. if Ĥ 步已执行:
      → 异步生成新 action chunk（带 RTC）
   c. 执行 a_t
```

### Classifier-Free Guidance (CFG)

对任意 prompt 部分可以使用 CFG：

$$\nabla_a \log \pi_\theta(a_{t:t+H}|o_t, C_t) + \beta \left( \nabla_a \log \pi_\theta(a_{t:t+H}|o_t, C_t) - \nabla_a \log \pi_\theta(a_{t:t+H}|o_t, C_t^{\text{uncond}}) \right)$$

其中：
- $C_t^{\text{uncond}}$ 是"无条件"模式下使用的 context（对应部分被 dropout）
- $\beta \in \{1.3, 1.7, 2.2\}$ 是 CFG 权重

实践中主要对 **episode metadata** 应用 CFG，以激励 dexterous tasks 中的强表现。

### 推理速度

| 配置 | 延迟 |
|------|------|
| 最小配置（3 cameras, 5 denoising steps, training-time RTC） | 38ms |
| 加上 MEM + subgoal images | 127ms (worst case) |
| Subgoal image 生成（14B model, 25 steps, 4×H100, 8-bit quant, SageAttention） | 1.25s |

**与 SFT Specialist 对比**（更多 dexterous tasks）：

| 任务 | π₀.7 Task Progress |

### 6.2 Ablation: Metadata 与 Eval Data 的贡献

| 模型变体 | 效果 |
|----------|------|
| π₀.7 (full) | 最好 |
| π₀.7 (no metadata) | **显著下降**，尤其在 throughput |
| π₀.7 (no eval data) | **显著下降**，在所有任务上 |

**结论**：
1. 没有 metadata，混合质量数据反而伤害性能（mode averaging）
2. 没有 autonomous eval data，模型无法蒸馏 RL specialist 的能力
3. **Metadata + 多样数据 = 协同效应**

### 6.3 Instruction Following

**在 4 个未见厨房 + 2 个未见卧室中测试**：

- π₀.7 >> π₀.5, π₀.6
- 3-6 步指令序列的成功率显著更高

**Complex Referential Instructions**：

| 指令类型 | π₀.5 | π₀.6 | π₀.7 | π₀.7 (GC) |

| Reverse Fridge→Microwave | ~0% | ~5% | ~30% | ~55% |

**关键**：π₀.7 能遵循与数据偏差相反的指令！Subgoal images 在 Reverse Fridge→Microwave 任务中至关重要。

### 6.4 Cross-Embodiment Transfer

**简单重排任务**：
- Table Setting（多机器人数据→static bimanual）：所有模型都好
- Bag In Backpack / Organize Tupperware（UR5e→static bimanual）：π₀.5 崩溃，π₀.6 和 π₀.7 仍然好
- Shirt Bagging（static bimanual→single-arm UR5e）：**π₀.7 显著优于前代**

**Dexterous 任务 — Laundry Folding**（static bimanual→UR5e bimanual，**零训练数据**）：

| 方法 | Towel Folding | Shirt Folding |
|------|---------------|---------------|
| π₀.5 | ~0% | ~0% |
| π₀.6 | ~15% | ~10% |
| π₀.7 | ~55% | ~50% |
| π₀.7 (GC) | ~65% | ~65% |
| **Human (零样本)** | — | **90.9% progress / 80.6% success** |

π₀.7 (GC) 达到 **85.6% task progress, 80% success rate**，与人类 expert teleoperators 相当！

**Emergent Strategy 适配**：
- 源机器人（static bimanual）：人类用一只手撑开袋子，另一只手插入
- 目标机器人（UR5e）：π₀.7 自动发现**单臂 pick-and-place 策略**（因为 UR5e 手臂更长）
- 源机器人：tilted end-effector grasp 衬衫
- UR5e：π₀.7 自动用 **vertical grasp**（更适合 UR5e 运动学）

**Coaching → Autonomous**：

### 6.6 Scaling 与 Data Diversity

**实验 1：Mixed-quality data 是否有害？**

将 laundry 数据按质量/速度分为 4 个桶（top 30%, top 50%, top 80%, all data）：
- π₀.7 (without metadata)：**更多低质量数据 → 性能下降**
- π₀.7 (with metadata)：**更多数据（即使平均质量更低）→ 性能持续提升**

**实验 2：Task diversity 是否重要？**

- π₀.7 (w/o most diverse 20%) vs π₀.7 (w/o random 20%)
- 去掉最多样的 20% 数据，在 unseen short-horizon tasks 上显著下降
- 去掉随机 20% 数据，影响很小

**结论**：**数据多样性 + 详细 context = 组合泛化的驱动力**

## 七、核心公式的变量含义总结

### 训练目标

$$\max_\theta \mathbb{E}_\mathcal{D} \left[ \log \pi_\theta(a_{t:t+H} | o_{t-T:t}, C_t) \right]$$

- $\theta$：模型参数
- $\mathcal{D}$：训练数据集
- $a_{t:t+H}$：从时间 $t$ 开始的 $H$ 步 action chunk（$H=50$）
- $o_{t-T:t}$：从时间 $t-T$ 到 $t$ 的观测历史
- $C_t$：完整的 prompt/context

### Context

$$C_t = (\ell_t, \hat{\ell}_t, g_t, m, c)$$

- $\ell_t$：整体任务描述（如 "clean the kitchen"）
- $\hat{\ell}_t$：当前子任务指令（如 "open the fridge door"）
- $g_t = [G_t^1, \ldots, G_t^n]$：多视角 subgoal images
- $m$：episode metadata = {speed, quality, mistake}
- $c \in \{\text{joint}, \text{ee}\}$：控制模式

### 观测

$$o_t = [I_t^1, \ldots, I_t^n, q_t]$$

- $I_t^i$：第 $i$ 个相机在时间 $t$ 的图像
- $q_t$：机器人的 joint configuration（本体感受）

### World Model 目标

$$\max_\psi \mathbb{E}_{\mathcal{D}_g} [\mathcal{L}_{\text{CFM}}(g_t^\star, g_\psi(o_t, \hat{\ell}_t, m))]$$

- $\psi$：world model 参数
- $g_t^\star = o_{t_{\text{end}}}$：ground-truth subgoal（segment 末尾帧）
- $\mathcal{L}_{\text{CFM}}$：conditional flow matching loss

### CFG

$$\nabla_a \log \pi_\theta(a_{t:t+H}|o_t, C_t) + \beta \left( \nabla_a \log \pi_\theta(\cdot|C_t) - \nabla_a \log \pi_\theta(\cdot|C_t^{\text{uncond}}) \right)$$

- $\beta$：CFG 权重（$\in \{1.3, 1.7, 2.2\}$）
- $C_t^{\text{uncond}}$：无条件 context（对应部分被 mask）

## 九、局限性与未来方向

1. **Zero-shot 成功率仍低于 in-distribution**：seen tasks >90%，unseen tasks 60-80%
2. **难以确定什么是真正"未见"的**：数据集太大太杂，"remixing" vs "真正泛化"难以区分
3. **World model 生成延迟**：1.25s（需要 4×H100 + 8-bit quantization）
4. **Episode metadata 需要人工标注**：quality 和 mistake 标签
5. **未来方向**：利用 steerability 做 efficient finetuning、autonomous RL、更详细的 language coaching

## 十、Intuition 总结

用一个类比来 build intuition：

> **传统 VLA** = 一个只会做"示范菜谱"的厨师，换道菜就不会了
>
> **π₀.7** = 一个受过各种训练的厨师，你不仅告诉他"做菜"（what），还告诉他"用高级技法、快速完成、不要犯错"（how），甚至给他看一张成品照片（subgoal image）。因为他见过好厨师和差厨师的做法（metadata），知道区别在哪，所以当你说"用最好的方式做"时，他能组合出最佳策略。

**核心公式化的 intuition**：

$$\underbrace{\text{Diverse Data}}_{\text{更多知识}} \times \underbrace{\text{Rich Context}}_{\text{更好的索引}} = \underbrace{\text{Compositional Generalization}}_{\text{emergent capability}}$$

没有 rich context，diverse data 只会制造混乱（mode averaging）；没有 diverse data，rich context 也没有足够的内容去索引。两者缺一不可，这正是 π₀.7 的精髓。
