# MT-Opt：大规模连续多任务机器人强化学习

## 论文核心概述

这篇论文来自 **Google Robotics**，提出了 **MT-Opt** 系统——一个可扩展的**多任务机器人强化学习**框架。该系统通过**参数共享**和**数据共享**，在真实世界中同时学习多种机器人操作技能，显著降低了学习新任务的成本。

论文地址：https://arxiv.org/abs/2104.08212  
项目主页：https://karolhausman.github.io/mt-opt

---

## 一、问题动机与挑战

### 1.1 核心问题

传统的**单任务强化学习**方法（如 **QT-Opt**）需要大量的训练数据（例如超过50万次试验），这使得训练一个通用的、具备多种技能的机器人变得极其昂贵。论文的核心问题是：

> **能否通过多任务学习的思想，分摊学习多种技能的成本？**

### 1.2 多任务学习的潜在优势

1. **共享表征**：不同任务共享基本的视觉特征和操作概念
2. **探索促进**：简单任务可以为复杂任务提供探索基础
3. **数据复用**：一个任务的经验可以被其他任务利用
4. **持续学习**：学习新任务时可以借助已有技能

### 1.3 主要挑战

| 挑战 | 描述 |
|------|------|
| **优化困难** | 多任务RL的优化极其困难，多任务干扰（Multi-task Interference）会导致性能下降 |
| **奖励定义** | 需要为大量任务直观地定义奖励函数 |
| **数据共享** | 不相关任务的数据共享可能对学习有害 |
| **可扩展性** | 需要显著扩展算法规模、任务数量和机器人数量 |

│                              ↓        ↑                     │

### 2.2 工作流程

1. **用户定义任务**：通过提供成功/失败的视频示例来训练成功检测器（Success Detector, SD）
2. **数据收集**：多个机器人同时收集多个任务的数据
3. **数据共享与训练**：通过Task Impersonation和Data Re-balancing扩展和重新平衡训练数据
4. **策略部署**：MT-Opt策略在机器人上执行，SD评估结果并给出稀疏奖励

## 三、核心方法详解

### 3.1 多任务强化学习算法

#### 3.1.1 符号定义

- **策略**：`π(a|s, Tᵢ)`，其中：
  - `a ∈ A`：动作（包括机械臂位置、方向、夹爪命令）
  - `s ∈ S`：状态（机器人摄像头的图像）
  - `Tᵢ`：第i个任务的编码，服从分类任务分布 `Tᵢ ∼ p(T)`，有n个可能的类别
  - `rᵢ(a, s, Tᵢ)`：任务相关的奖励

- **Q函数**：`Q_θ(s, a, Tᵢ)`，参数为θ，估计在状态s执行动作a对任务Tᵢ的期望累积奖励

#### 3.1.2 Bellman损失函数

基于单任务QT-Opt的Q学习损失：

$$
L_i(\theta) = \mathbb{E}_{(s,a,s') \sim p(s,a,s')} \left[ \mathcal{D}\left(Q_\theta(s, a), Q^T(s, a, s')\right) \right]
$$

其中：
- `(s, a, s')` 是从之前观察到的转移中采样的
- `Q^T(s, a, s') = r(s, a) + γ V(s')` 是目标Q值
- `γ` 是折扣因子
- `V(s') = \max_{a'} Q(s', a')` 是下一状态的目标值函数
- `𝒟` 是 divergence metric，如交叉熵（cross-entropy）
- `V(s')` 通过交叉熵方法（CEM）进行随机优化计算

#### 3.1.3 多任务扩展

多任务损失函数：

$$
L_{\text{multi}}(\theta) = \mathbb{E}_{T_i \sim p(T)} \left[ L_i(\theta) \right] = \mathbb{E}_{T_i \sim p(T)} \left[ \mathbb{E}_{p(s^{(i)}, a^{(i)}, s'^{(i)})} \left[ \mathcal{D}\left(Q_\theta(s^{(i)}, a^{(i)}, T_i), Q^T(s^{(i)}, a^{(i)}, s'^{(i)}, T_i)\right) \right] \right]
$$

其中 `(s^{(i)}, a^{(i)}, s'^{(i)})` 是由任务 `Tᵢ` 生成的转移。

### 3.2 任务模仿（Task Impersonation）

#### 3.2.1 核心思想

**Task Impersonation** 允许将一个任务收集的episode用于训练其他相关任务。

定义模仿函数 `f_I`：

$$
\{k_i\} = f_I(e^{(i)})
$$

其中：
- `e^{(i)}` 是由任务 `Tᵢ` 生成的原始episode
- `{k_i}` 是该episode可以用于训练的一组任务集合
- `{k_i}` 是所有任务 `{n}` 的子集

#### 3.2.2 三种模仿策略

| 策略 | 定义 | 优缺点 |
|------|------|--------|
| `f_I^orig` | `f_I^orig(e^{(i)}) = e^{(i)}` | 无数据共享，不利用多任务数据 |
| `f_I^all` | `e^{(i)} → e^{\{n\}}` | 最大数据共享，但会导致负样本"稀释" |
| `f_I^skill` | 只在同一skill的任务间共享 | 平衡数据共享与负样本质量 |

#### 3.2.3 Skill定义

**Skill** 是一组共享语义和动力学但初始条件或操作对象不同的任务：

```
示例：
- place-object-on-plate
- place-object-in-bowl

都属于同一skill：Object Manipulation Skill
```

#### 3.2.4 随机模仿函数

为解决过度模仿问题，引入概率：

$$
\text{以概率 } p_s \text{ 路由成功的episode} \\
\text{以概率 } p_f \text{ 路由失败的episode}
$$

其中 `p_s = 1.0`（总是利用成功的模仿示例），`p_f ≤ 1.0`（只利用部分失败的模仿示例）。

### 3.3 数据重新平衡（Data Re-balancing）

#### 3.3.1 问题

由于任务难度和数据收集策略不同，训练数据高度不平衡：

```
不平衡来源：
1. 不同任务的数据量差异
2. 成功/失败episode的比例差异
```

#### 3.3.2 解决方案

在每个batch中重新平衡：
1. **任务间平衡**：确保每个任务的训练数据比例相等
2. **任务内平衡**：保持成功和失败示例的相对比例恒定

### 3.4 神经网络架构

#### 3.4.1 Q函数架构

```
输入：
├── 图像 I_t (472, 472, 3)
├── 动作向量：
│   ├── 笛卡尔向量 t
│   ├── 夹爪旋转 r
│   ├── 打开夹爪 g_open
│   ├── 关闭夹爪 g_close
│   └── 终止动作 e
└── 任务ID T_i (one-hot编码)
└── 状态特征：
    ├── 夹爪状态 g_opened
    └── 夹爪高度 g_height

视觉处理路径：
I_t → [Repeat x6 Conv(64,6,2) → MaxPool(3)] → 
[Repeat x3 Conv(64,5,1) → MaxPool(3)] → 
Conv(64,3,1) → MaxPool(2)

状态/动作/任务处理路径：
[a, g_opened, g_height, T_i] → FC(64) → FC(64) → 
Reshape(1,1,64) → Tile(Width, Height)

融合：视觉特征 + 状态/动作特征 (广播逐元素相加)

后续处理：
→ [Repeat x2 Conv(64,3,1) → MaxPool(3)] → 
[Repeat x6 Conv(64,3,1) → MaxPool(3)] → 
FC(64) → Sum → FC(256) → Sigmoid → Q_θ(s,a,T_i) ∈ [0,1]
```

#### 3.4.2 单头 vs 多头架构

论文对比了两种架构：

| 架构 | 描述 | 性能 |
|------|------|------|
| 单头 | 共享所有层的参数，任务通过one-hot ID区分 | 更好 |
| 多头 | 共享早期层，输出层有n个独立头（每任务3层FC） | 较差 |

**多头架构的问题**：过度隔离任务策略，难以利用跨任务共享表征。

## 四、成功检测器（Success Detector）

### 4.1 奖励设计

MT-Opt使用**稀疏奖励**：只在episode结束时根据最终图像给出奖励0或1。

### 4.2 成功检测器训练

#### 4.2.1 训练数据生成

1. **用户提供的视频示例**：
   - 5Hz视频，3个不同摄像头角度
   - 成功视频 + 失败视频（成对收集）
   - 改变光照、物体和背景以解耦虚假相关性

2. **策略数据重新标注**：
   - 利用偶尔产生成功的现有策略（如lift-any）
   - 重新标注为新任务（如lift-carrot）的成功/失败

#### 4.2.2 数据增强

为提高泛化性，训练时进行数据增强：
- 裁剪
- 变亮
- 旋转
- 叠加随机阴影

#### 4.2.3 模型架构

- 基于与MT-Opt Q函数相同的架构
- 不需要动作条件
- 输入：3张图像（过肩摄像头 + 2个侧边摄像头）
- 输出：每个任务的成功概率

#### 4.2.4 训练策略

- 平衡成功/失败示例
- 平衡不同任务的数据
- 基于任务可行性的数据共享逻辑

### 4.3 成功检测器性能

| 任务 | 总数 | 成功数 | 失败数 | 成功率 | 假负率 | 假正率 |
|------|------|--------|--------|--------|--------|--------|
| lift-any | 160647 | 39586 | 7246 | 46% | 1% | 2% |
| lift-carrot | 64819 | 11557 | 01% | 70% | 0% | 1% |
| place-any | 30871 | 3631 | 7244 | 44% | 1% | 2% |

### 6.4 消融实验

#### 6.4.1 参数共享的影响

| 模型 | lift-any | place-any |
|------|----------|-----------|
| 2-Task模型 | 0.82 | 0.63 |
| 12-Task模型 | 0.89 | 0.85 |

**结论**：12任务模型在两个任务上都优于2任务模型，说明多任务训练通过共享表征提高了性能。

#### 6.4.2 数据共享策略的影响

| Impersonation | Re-balancing | Min | Mean | Low Data Mean |
|---------------|--------------|-----|------|---------------|
| f_I^orig | uniform | 0.10 | 0.32 | 0.18 |
| f_I^orig | task-balanced | 0.16 | 0.55 | 0.42 |
| f_I^all | uniform | 0.07 | 0.21 | 0.13 |
| f_I^all | task-balanced | 0.02 | 0.35 | 0.21 |
| f_I^skill | uniform | 0.17 | 0.46 | 0.32 |
| f_I^skill(1, 0.15) | task-balanced | 0.29 | 0.58 | 0.50 |
| **f_I^skill(1, 1) (Ours)** | **task-balanced** | **0.29** | **0.58** | **0.50** |

**结论**：
1. Skill-based impersonation优于all和original
2. Task re-balancing显著提升性能，特别是对低数据任务
3. 最佳组合是skill-based impersonation + task re-balancing

#### 6.4.3 Bootstrap效果

**lift-sausage任务**：
- 独立数据（仅lift-sausage）：16,600 episodes → 3%成功率
- MT-Opt（利用lift-any数据 + impersonation）：39%成功率
- **改进：≈10倍**

**关键发现**：
1. 利用简单任务（lift-any）收集复杂数据（lift-sausage）
2. 利用失败样本（抓错其他物体）提供学习信号

## 八、技术细节与实现

### 8.1 交叉熵方法（CEM）

用于计算目标值函数 `V(s')`：

$$
V(s') = \max_{a'} Q(s', a')
$$

CEM步骤：
1. 从动作空间采样N个动作
2. 计算每个动作的Q值
3. 选择前K个最高Q值的动作
4. 拟合高斯分布到这些动作
5. 从新分布采样，重复直到收敛

### 8.2 Loss函数详细实现

**交叉熵损失**：

$$
\mathcal{D}(Q_\theta(s, a), Q^T) = -Q^T \log Q_\theta(s, a) - (1 - Q^T) \log(1 - Q_\theta(s, a))
$$

**多任务训练过程**：

```python
# 伪代码
for each training iteration:
    # 1. 从多个任务采样batch
    for T_i in sample_tasks(p(T)):
        # 2. 获取该任务的impersonated episodes
        episodes = get_impersonated_episodes(T_i)
        
        # 3. 重新平衡成功/失败示例
        balanced_episodes = re_balance(episodes)
        
        # 4. 计算Bellman目标
        for (s, a, s', r) in balanced_episodes:
            # 计算目标Q值
            Q_target = r + gamma * compute_target_V(s', T_i)
            
            # 计算损失
            loss += cross_entropy_loss(Q_theta(s, a, T_i), Q_target)
    
    # 5. 反向传播更新
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

## 十三、实际应用建议

### 13.1 何时使用MT-Opt？

✅ **适用场景**：
- 需要学习多种相关任务
- 有多个机器人可以并行训练
- 任务间有共享的视觉/动力学结构
- 可以通过最终状态定义成功

❌ **不适用场景**：
- 只需学习单个任务
- 任务间完全无关
- 无法定义成功/失败状态
- 机器人数量有限

### 13.2 实现建议

1. **从简单任务开始**：先训练最通用的任务（如lift-any）
2. **仔细设计技能分组**：基于任务语义而非表面特征
3. **平衡训练数据**：特别注意低数据任务
4. **迭代改进SD**：持续收集和标注新数据

### 13.3 超参数建议

根据论文，建议：
- `p_s = 1.0`（总是利用成功的模仿示例）
- `p_f ≤ 1.0`（部分利用失败的模仿示例）
- 成功/失败比例：保持每个任务内平衡
- 任务比例：所有任务等比例采样
