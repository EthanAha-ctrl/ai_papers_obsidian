



# MFU (Model FLOPS Utilization) - 模型FLOPS利用率

## 一、MFU的核心定义

**MFU (Model FLOPS Utilization)** 是一个衡量LLM训练中GPU计算资源利用效率的关键指标，由Google在PaLM论文中首次提出。

### 基本定义

```
┌─────────────────────────────────────────────────────────────────┐
│                     MFU 核心定义                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  MFU = 实际达到的训练吞吐量 / 理论最大吞吐量                      │
│                                                                 │
│  理论最大吞吐量 = 模型所需的理论FLOPs / 硬件峰值FLOPS             │
│                                                                 │
│  MFU回答的核心问题:                                              │
│  "在模型理论上所需的全部浮点运算中，有多少百分比被硬件实际利用？"  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 为什么需要MFU

```
┌─────────────────────────────────────────────────────────────────┐
│              传统硬件指标的局限性                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  GPU Utilization (70%):                                         │
│    ❌ 不同代际GPU之间不可比较                                    │
│    ❌ 无法反映模型特定的计算需求                                 │
│    ❌ 高利用率可能掩盖低层kernel低效问题                         │
│                                                                 │
│  SM/Tensor Core Utilization:                                    │
│    ❌ 硬件依赖性强                                               │
│    ❌ 缺乏模型中心的视角                                         │
│                                                                 │
│  MFU:                                                           │
│    ✅ 硬件无关 (Hardware-agnostic)                               │
│    ✅ 模型中心 (Model-centric)                                   │
│    ✅ 可跨硬件比较                                               │
│    ✅ 反映真实训练效率                                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、MFU的计算公式

### 基础公式

```
┌─────────────────────────────────────────────────────────────────┐
│                   MFU 计算公式                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  MFU = (T × F) / max_flops                                      │
│                                                                 │
│  其中:                                                          │
│    T = 观测到的训练吞吐量 (tokens/second/GPU)                    │
│    F = 每个token所需的理论FLOPs                                 │
│    max_flops = GPU的理论峰值FLOPS (精度相关)                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 快速近似公式（适用于大多数Transformer模型）

```
┌─────────────────────────────────────────────────────────────────┐
│              MFU 快速近似计算                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  对于标准Transformer:                                           │
│    F ≈ 6 × N_params (FLOPs/token)                              │
│                                                                 │
│  因此:                                                          │
│    MFU ≈ (T × 6 × N_params) / max_flops                        │
│                                                                 │
│  这个近似基于:                                                   │
│    - Forward pass: ≈ 2 × N_params                               │
│    - Backward pass: ≈ 4 × N_params                              │
│    - 总计: ≈ 6 × N_params                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 详细数学推导

#### 1. Forward Pass FLOPs推导

```
┌─────────────────────────────────────────────────────────────────┐
│           Forward Pass FLOPs 推导                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  对于decoder-only transformer的单个token:                        │
│                                                                 │
│  Matrix Multiplication FLOPs计算:                                │
│    对于矩阵乘法 Y = X × W:                                       │
│    FLOPs ≈ 2 × (output_dimensions)                              │
│                                                                 │
│  每层包含:                                                      │
│    • Attention QKV projection: 3 × 2 × h² = 6h²                  │
│    • Attention O projection: 2 × h² = 2h²                       │
│    • MLP layers:                                                │
│      - Standard (4h): 2 × (h×4h) + 2 × (4h×h) = 16h²           │
│      - SwiGLU (3 gates): ~18h²                                  │
│    • Layer Norm: 忽略 (相比matmul很小)                           │
│                                                                 │
│  总计 per layer:                                                 │
│    Standard GPT: ≈ 24h² FLOPs/token                             │
│                                                                 │
│  对于L层模型 (假设每层h参数量 ≈ total_params / L):                │
│    Flops_forward ≈ 2 × N_params                                  │
│                                                                 │
│  为什么是"≈"?                                                   │
│    - 忽略了sequence length dependent的attention计算              │
│    - 对于长序列需要更精确公式                                    │
│    - 但对于大模型，MLP和projection占主导                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 2. Backward Pass FLOPs推导

```
┌─────────────────────────────────────────────────────────────────┐
│           Backward Pass FLOPs 推导                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  对于一个线性层 Y = X × W + b:                                   │
│                                                                 │
│  Forward:                                                       │
│    Y = X × W  (主计算)                                          │
│    FLOPs_fwd ≈ 2 × B × I × O                                    │
│    (B=batch, I=input_dim, O=output_dim)                         │
│                                                                 │
│  Backward需要计算两个梯度:                                       │
│                                                                 │
│  (1) ∂L/∂X = (∂L/∂Y) × Wᵀ  (用于传递梯度)                       │
│      FLOPs ≈ 2 × B × O × I ≈ FLOPs_fwd                          │
│                                                                 │
│  (2) ∂L/∂W = Xᵀ × (∂L/∂Y)  (用于更新权重)                       │
│      FLOPs ≈ 2 × I × B × O ≈ FLOPs_fwd                          │
│                                                                 │
│  因此:                                                          │
│    Flops_backward ≈ 2 × Flops_forward                           │
│                                                                 │
│  总训练FLOPs (forward + backward):                               │
│    Flops_training ≈ 3 × Flops_forward                           │
│    ≈ 3 × 2 × N_params = 6 × N_params                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 三、不同模型架构的精确FLOPs计算

### GPT类模型（标准Transformer）

```
┌─────────────────────────────────────────────────────────────────┐
│          GPT类模型 FLOPs 计算 (考虑序列长度)                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  前提: MLP hidden_size = 4 × embedding_size                      │
│                                                                 │
│  Forward Pass FLOPs (单个batch):                                │
│    E = 24 × B × s × l × h² + 4 × B × s² × l × h                 │
│                                                                 │
│  其中:                                                          │
│    B = batch_size                                               │
│    s = sequence_length (context length)                         │
│    l = number of layers                                         │
│    h = embedding dimension (hidden_size)                        │
│                                                                 │
│  化简形式:                                                      │
│    E = 24 B s l h² [1 + s/(6h)]                                 │
│                                                                 │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━  │
│                                                                 │
│  完整训练FLOPs:                                                 │
│                                                                 │
│  1. 使用Full Activation Recomputation:                          │
│     F = 4E + 6 B s h V                                          │
│       = 96 B s l h² [1 + s/(6h) + V/(16lh)]                    │
│                                                                 │
│  2. 不使用Activation Recomputation:                             │
│     F = 3E + 6 B s h V                                          │
│       = 72 B s l h² [1 + s/(6h) + V/(12lh)]                    │
│                                                                 │
│  3. 使用FlashAttention (带recomputation):                       │
│     F = 3E + 4 B s² l h + 6 B s h V                            │
│       = 72 B s l h² [1 + 2s/(9h) + V/(12lh)]                    │
│                                                                 │
│  (V = vocabulary_size)                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### LLaMA类模型（SwiGLU激活函数）

```
┌─────────────────────────────────────────────────────────────────┐
│        LLaMA类模型 FLOPs 计算 (SwiGLU)                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  特点: MLP使用SwiGLU，包含3个线性层                              │
│        MLP hidden_size ≠ 4 × embedding_size                     │
│                                                                 │
│  Forward Pass FLOPs (单个batch):                                │
│    E = 8 B s l h² + 6 B s l h f + 4 B s² l h                   │
│                                                                 │
│  其中:                                                          │
│    f = MLP hidden_size (通常 > 4h)                              │
│                                                                 │
│  完整训练FLOPs:                                                 │
│                                                                 │
│  1. Full Activation Recomputation:                              │
│     F = 4E + 6 B s h V                                          │
│                                                                 │
│  2. 不使用Activation Recomputation:                             │
│     F = 3E + 6 B s h V                                          │
│                                                                 │
│  3. 使用FlashAttention:                                         │
│     F = 3E + 4 B s² l h + 6 B s h V                            │
│                                                                 │
│  MFU计算:                                                       │
│    MFU = (F × T) / (B × s × C)                                  │
│    (其中C = GPU peak FLOPS)                                     │
│    注意B会抵消，MFU与batch_size无关                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、MFU的实际案例分析

### 案例1: LLaMA-1 65B模型

```
┌─────────────────────────────────────────────────────────────────┐
│              LLaMA-1 65B MFU 计算                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  已知参数:                                                      │
│    • N_params = 65B                                             │
│    • 训练tokens = 1.4T                                          │
│    • 训练时间 = 1,022,362 GPU-hours                             │
│    • GPU数量 = 2048 A100                                        │
│    • 训练天数 ≈ 21天                                            │
│                                                                 │
│  Step 1: 计算实际吞吐量 T                                        │
│    T = 总tokens / (GPU数量 × 训练时间(秒))                       │
│      = 1.4×10¹² / (2048 × 1,022,362 × 3600)                    │
│      ≈ 380.38 tokens/s/GPU                                     │
│                                                                 │
│  Step 2: 计算理论FLOPs/token (F)                                │
│    F ≈ 6 × N_params                                             │
│      = 6 × 65×10⁹                                               │
│      = 390×10⁹ FLOPs/token                                      │
│                                                                 │
│  Step 3: 计算MFU (假设A100 SXM, BF16)                           │
│    max_flops(A100) = 312 TFLOPS = 312×10¹² FLOPS               │
│                                                                 │
│    MFU = (T × F) / max_flops                                   │
│        = (380.38 × 390×10⁹) / (312×10¹²)                        │
│        = 148.348×10⁹ / 312×10¹²                                 │
│        ≈ 0.475 = 47.5%                                         │
│                                                                 │
│  结果分析:                                                      │
│    MFU ≈ 47.5% 是相当不错的水平                                 │
│    说明训练基础设施优化良好                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 案例2: Skywork-13B模型

```
┌─────────────────────────────────────────────────────────────────┐
│            Skywork-13B MFU 计算 (精确方法)                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  模型配置:                                                      │
│    • h (hidden_size) = 4608                                    │
│    • f (MLP hidden_size) = 12288                               │
│    • l (layers) = 52                                           │
│    • s (seq_len) = 4096                                        │
│    • V (vocab_size) = 65519                                    │
│    • B = 1 (per device)                                        │
│    • T (throughput) = 1873 tokens/s/GPU                         │
│                                                                 │
│  Step 1: 计算Forward FLOPs (E)                                  │
│    E = 8 B s l h² + 6 B s l h f + 4 B s² l h                    │
│      = 8×1×4096×52×4608² + 6×1×4096×52×4608×12288              │
│        + 4×1×4096²×52×4608                                      │
│      = 18.06×10¹² + 73.04×10¹² + 16.22×10¹²                     │
│      = 107.32×10¹² FLOPs                                        │
│                                                                 │
│  Step 2: 计算总训练FLOPs (F) - 无activation recomputation       │
│    F = 3E + 6 B s h V                                           │
│      = 3×107.32×10¹² + 6×1×4096×4608×65519                     │
│      = 321.96×10¹² + 7.46×10¹²                                  │
│      = 329.42×10¹² FLOPs                                        │
│                                                                 │
│  Step 3: 计算有效FLOPS                                          │
│    Effective FLOPS = F × T / (B × s)                            │
│                     = (329.42×10¹² × 1873) / 4096              │
│                     = 617.0×10¹² / 4096                         │
│                     = 150.64 TFLOPS                             │
│                                                                 │
│  Step 4: 计算MFU (A100 SXM BF16)                                │
│    MFU = Effective FLOPS / Peak FLOPS                           │
│        = 150.64 / 312                                           │
│        = 48.28%                                                 │
│                                                                 │
│  与论文报告对比:                                                 │
│    论文报告: MFU ≈ 56.47%                                       │
│    差异来源: 论文可能使用了更精确的FLOPs计数                      │
│              或者考虑了FlashAttention的优化                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 关键优化技术对MFU的影响

```
┌─────────────────────────────────────────────────────────────────┐
│        优化技术对MFU的影响 (定量分析)                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. FlashAttention                                               │
│     • 减少HBM访问，提升compute-to-memory ratio                   │
│     • MFU提升: +5-15%                                           │
│     • 原因: IO-aware tiling减少memory traffic                   │
│                                                                 │
│  2. Mixed Precision (BF16/FP16)                                 │
│     • 利用Tensor Core加速                                        │
│     • MFU提升: +10-20%                                          │
│     • 注意: 需要loss scaling防止underflow                       │
│                                                                 │
│  3. Activation Checkpointing                                     │
│     • 以计算换内存                                               │
│     • MFU影响: 可能降低5-10% (增加recomputation)               │
│     • 但允许更大batch size，整体收益可能为正                     │
│                                                                 │
│  4. 3D/4D Parallelism (TP+PP+DP+CP)                             │
│     • 精细的并行策略提升整体利用率                                │
│     • MFU提升: +5-20% (取决于优化程度)                          │
│     • 通信overlap是关键                                          │
│                                                                 │
│  5. CUDA Graphs / Torch Compile                                 │
│     • 减少kernel launch overhead                                │
│     • MFU提升: +2-5%                                            │
│     • 对小batch size效果明显                                     │
│                                                                 │
│  6. Sequence Parallelism / Context Parallelism                   │
│     • 对长序列至关重要                                           │
│     • MFU提升: +10-30% (对于序列长度>32K)                      │
│                                                                 │
│  7. Data Loading Optimization                                    │
│     • Prefetch, async loading                                   │
│     • MFU提升: +3-8%                                            │
│     • 消除GPU空闲等待                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### MFU与序列长度的关系

```
┌─────────────────────────────────────────────────────────────────┐
│           MFU vs Sequence Length 关系                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  FLOPs公式中的序列长度相关项:                                    │
│    Attention FLOPs = O(B × s² × l × h)                          │
│                                                                 │
│  MFU变化趋势:                                                   │
│                                                                 │
│  MFU (%)                                                        │
│    55 │              ┌──────────                               │
│    50 │          ┌───┘                                          │
│    45 │      ┌───┘                                              │
│    40 │  ┌───┘                                                  │
│    35 │──┘                                                      │
│    30 │                                                         │
│       └───────────────────────────────────────────             │
│       0    2K    8K    32K   64K   128K  Sequence Length      │
│                                                                 │
│  分析:                                                          │
│    • 短序列(2K-8K): MFU较高，MLP计算占主导                      │
│    • 中等序列(8K-32K): MFU稳定                                  │
│    • 长序列(>32K): MFU下降，attention成为瓶颈                   │
│    • 使用Context Parallelism可以恢复MFU                         │
│                                                                 │
│  为什么长序列MFU下降?                                           │
│    - Attention FLOPs增长为O(s²)                                 │
│    - Memory访问模式不够优化                                     │
│    - KV cache占用增加                                            │
│    - 需要更多通信                                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 九、总结

### MFU核心要点

```
┌─────────────────────────────────────────────────────────────────┐
│                  MFU 核心要点总结                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 定义                                                       │
│     • MFU = 实际训练FLOPS / GPU峰值FLOPS                        │
│     • 硬件无关，可跨系统比较                                     │
│     • 反映真实训练效率                                           │
│                                                                 │
│  2. 计算公式                                                   │
│     • 快速估算: MFU ≈ (T × 6 × N_params) / max_flops           │
│     • 精确计算: 需要考虑模型架构、序列长度、优化技术            │
│                                                                 │
│  3. 目标值                                                     │
│     • 业界平均: 20-35%                                         │
│     • 良好优化: 40-55%                                         │
│     • 顶尖水平: 55-75%                                         │
│                                                                 │
│  4. 关键影响因素                                               │
│     • Kernel优化 (FlashAttention等)                             │
│     • 并行策略 (TP/PP/DP/CP)                                   │
│     • 通信优化                                                  │
│     • 混合精度                                                  │
│     • 数据流水线优化                                            │
│                                                                 │
│  5. 优化价值                                                   │
│     • MFU提升可显著降低训练成本                                 │
│     • 30% → 45% 可节省30%+成本                                  │
│     • 大规模训练中MFU优化至关重要                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
