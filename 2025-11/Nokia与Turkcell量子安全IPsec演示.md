## 二、第一性原理：量子计算为何威胁当前密码学

### 2.1 当前公钥密码的数学基础

现代公钥密码学（如RSA、ECC）依赖于以下数学难题：

| 密码算法 | 数学难题 | 密钥长度 | 安全性基础 |
|---------|---------|---------|-----------|
| **RSA** | 大整数分解 | 2048-4096 bits | N = p × q，已知N求p、q极其困难 |
| **ECC** | 椭圆曲线离散对数问题(ECDLP) | 256-521 bits | Q = kP，已知P、Q求k极其困难 |
| **DH** | 离散对数问题(DLP) | 2048+ bits | g^x mod p，已知g、p、g^x求x极其困难 |

**RSA加密核心公式**：
$$C = M^e \mod N$$

其中：
- $M$ = 明文消息
- $e$ = 公开指数（通常为65537）
- $N = p \times q$ = 模数（两个大素数的乘积）
- $C$ = 密文

**解密公式**：
$$M = C^d \mod N$$

其中 $d$ 是私钥，满足 $d \times e \equiv 1 \mod \phi(N)$

### 2.2 Shor算法的威胁

1994年，Peter Shor提出了著名的**Shor算法**，证明了量子计算机可以在**多项式时间**内解决上述数学难题。

**Shor算法复杂度对比**：

| 问题 | 经典计算机复杂度 | 量子计算机复杂度 |
|-----|----------------|-----------------|
| 整数分解N | $O(e^{(\ln N)^{1/3}(\ln\ln N)^{2/3}})$ | $O((\log N)^3)$ |
| 离散对数 | $O(e^{(\ln p)^{1/3}(\ln\ln p)^{2/3}})$ | $O((\log p)^3)$ |

这意味着：
- RSA-2048：经典计算机需要约 $10^{20}$ 年，量子计算机理论上只需几小时
- ECC P-256：同样脆弱，甚至更易被攻破（密钥更短）

**Shor算法核心步骤**：
$$f(x) = a^x \mod N$$

其中：
- $a$ = 随机选择的基数，$\gcd(a, N) = 1$
- $N$ = 待分解的合数
- 通过**量子傅里叶变换(QFT)**找到函数$f(x)$的周期$r$

量子傅里叶变换：
$$\text{QFT}: |x\rangle \rightarrow \frac{1}{\sqrt{Q}}\sum_{k=0}^{Q-1} e^{2\pi i x k / Q}|k\rangle$$

找到周期$r$后，通过经典算法计算：
$$\gcd(a^{r/2} \pm 1, N)$$

即可得到$N$的因子$p$和$q$。

### 2.3 "Harvest Now, Decrypt Later"攻击

这是当前最紧迫的威胁场景：

```
┌─────────────────────────────────────────────────────────────────┐
│                    攻击者时间线                                  │
├─────────────────────────────────────────────────────────────────┤
│  Today (2024)          Future (2030+)        量子计算机时代     │
│       │                     │                     │            │
│       ▼                     ▼                     ▼            │
│  ┌─────────┐          ┌──────────┐          ┌──────────┐        │
│  │ 截获加密 │   →     │ 存储密文 │   →     │ 量子破解  │        │
│  │ 流量数据 │          │  等待... │          │ 明文获取 │        │
│  └─────────┘          └──────────┘          └──────────┘        │
│                                                                │
│  当前无法解密，但量子计算机出现后可以破解所有历史数据！           │
└─────────────────────────────────────────────────────────────────┘
```

**这就是为什么现在就需要部署量子安全密码学的原因**——即使量子计算机尚未实用化，攻击者可以现在截获并存储加密流量，等待未来技术成熟后批量解密。

### 3.3 传统IPsec密码套件的量子脆弱性

**IKEv2默认密码套件**：
```
IKE:AES-CBC_256-HMAC_SHA2_256-MODP3072
```

各组件的量子脆弱性分析：

| 组件 | 用途 | 量子安全性 |
|-----|------|-----------|
| AES-CBC_256 | 数据加密 | ✓ 安全（Grover算法仅将复杂度从2^256降至2^128） |
| HMAC-SHA2-256 | 完整性认证 | ✓ 安全 |
| MODP3072 (DH) | 密钥交换 | ✗ 不安全（Shor算法可破解） |

**问题核心**：Diffie-Hellman密钥交换使用的离散对数问题可被Shor算法在多项式时间内解决。

## 四、量子安全密码学技术详解

### 4.1 NIST后量子密码学标准（2024年最终版）

经过8年遴选，NIST于2024年正式发布了三个后量子密码学标准：

| 标准编号 | 算法名称 | 用途 | 数学基础 | 密钥/签名大小 |
|---------|---------|------|---------|--------------|
| **FIPS 203** | ML-KEM (原Kyber) | 密钥封装(KEM) | 格基密码学 | pk: 1184B, ct: 1088B |
| **FIPS 204** | ML-DSA (原Dilithium) | 数字签名 | 格基密码学 | sig: 2420B |
| **FIPS 205** | SLH-DSA (原SPHINCS+) | 数字签名 | 哈希签名 | sig: 17088B |

### 4.2 ML-KEM (Kyber) 深度解析

**ML-KEM全称**：Module-Learning-With-Errors Key Encapsulation Mechanism

#### 4.2.1 数学基础：Learning With Errors (LWE)问题

**原始LWE问题**：
给定$(A, b)$，其中：
$$b = A \cdot s + e \mod q$$

判断$b$是真随机的还是由上述关系生成的？

其中：
- $A \in \mathbb{Z}_q^{n \times n}$ = 公开随机矩阵
- $s \in \mathbb{Z}_q^n$ = 秘密向量
- $e \in \mathbb{Z}_q^n$ = 小误差向量（从高斯分布采样）
- $q$ = 模数（通常为质数如3329）

**求解$s$的困难性**：当$n$足够大时，即使使用量子计算机，这也是一个困难问题。

#### 4.2.2 ML-KEM算法流程

**密钥生成**：
```
输入：安全参数λ
输出：(pk, sk)

1. 生成随机矩阵 Â ∈ Z_q^{k×k}
2. 采样秘密向量 ŝ ← η(Z_q^{k×1})  // η是错误分布
3. 采样误差向量 ê ← η(Z_q^{k×1})
4. 计算 t̂ = Â · ŝ + ê
5. 公钥 pk = (Â, t̂)
6. 私钥 sk = ŝ
```

**封装**：
```
输入：公钥 pk = (Â, t̂)
输出：共享密钥K，密文 ct

1. 采样随机向量 ŕ ← η(Z_q^{1×k})
2. 采样误差 e1 ← η(Z_q^{k×1}), e2 ← η(Z_q)
3. 计算 û = Â^T · ŕ + e1
4. 计算 v̂ = t̂^T · ŕ + e2 + ⌊q/2⌋ · m  // m是消息位
5. 密文 ct = (û, v̂)
6. 共享密钥 K = H(û, v̂)  // H是哈希函数
```

**解封装**：
```
输入：密文 ct = (û, v̂), 私钥 sk = ŝ
输出：共享密钥K

1. 计算 m = ⌊2/q · (v̂ - ŝ^T · û)⌉  // ⌊·⌉表示四舍五入
2. 重新计算 ŕ', e1', e2'
3. 验证密文一致性
4. 共享密钥 K = H(û, v̂)
```

#### 4.2.3 ML-KEM参数集

| 参数集 | 安全级别 | 维度k | 模数q | 公钥大小 | 密文大小 |
|-------|---------|------|------|---------|---------|
| ML-KEM-512 | NIST Level 1 (128-bit) | 2 | 3329 | 800 B | 768 B |
| ML-KEM-768 | NIST Level 3 (192-bit) | 3 | 3329 | 1184 B | 1088 B |
| ML-KEM-1024 | NIST Level 5 (256-bit) | 4 | 3329 | 1568 B | 1568 B |

**NIST安全级别定义**：
- Level 1: 等效于AES-128安全性
- Level 3: 等效于AES-192安全性
- Level 5: 等效于AES-256安全性

### 4.3 Hybrid密钥交换——PR中的实际方案

PR中提到的"defense-in-depth crypto-resilient approach"很可能采用了**混合密钥交换**，即同时使用传统算法和后量子算法：

```
┌────────────────────────────────────────────────────────────────────┐
│                    Hybrid Key Exchange流程                         │
├────────────────────────────────────────────────────────────────────┤
│                                                                    │
│  Initiator                                    Responder            │
│     │                                            │                │
│     │  1. 生成两个密钥对：                        │                │
│     │     - ECDH密钥对 ← P-384                   │                │
│     │     - PQC密钥对 ← ML-KEM-768               │                │
│     │                                            │                │
│     │  ─────── IKE_SA_INIT (Hybrid) ──────────►  │                │
│     │    (SAi1, KEi_ECDH, KEi_PQC, Ni)           │                │
│     │                                            │                │
│     │  ◄────── IKE_SA_INIT (Hybrid) ───────────  │                │
│     │    (SAr1, KEr_ECDH, KEr_PQC, Nr)           │                │
│     │                                            │                │
│     │  2. 计算共享密钥：                          │                │
│     │     K_ECDH = DH(私钥_ECDH, 公钥_ECDH)       │                │
│     │     K_PQC  = Decap(私钥_PQC, ct_PQC)       │                │
│     │                                            │                │
│     │  3. 组合密钥：                              │                │
│     │     K_final = KDF(K_ECDH || K_PQC)        │                │
│     │                                            │                │
│     ▼                                            ▼                │
│                                                                    │
│  最终共享密钥K_final的安全保证：                                  │
│  - 只要ECDH或PQC其中之一安全，密钥就安全                          │
│  - 对抗量子计算机：PQC提供保护                                    │
│  - 对抗实现漏洞：双算法冗余                                       │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

**Hybrid方案的密钥组合公式**：
$$K_{final} = \text{HKDF}(K_{classical} \| K_{PQC})$$

其中：
- $K_{classical}$ = ECDH计算的传统共享密钥
- $K_{PQC}$ = ML-KEM封装的后量子共享密钥
- $\|$ = 拼接操作
- HKDF = HMAC-based Key Derivation Function

| 操作 | 传统ECC-P384 | ML-KEM-768 | 性能比 |
|-----|-------------|------------|-------|

### 8.2 QKD vs PQC技术路线对比

| 特性 | QKD (Quantum Key Distribution) | PQC (Post-Quantum Cryptography) |
|-----|-------------------------------|--------------------------------|
| 物理基础 | 量子力学测不准原理 | 计算复杂性假设 |
| 部署成本 | 高（专用光纤/设备） | 低（软件升级） |
| 距离限制 | 有（~100km，中继可延长） | 无 |
| 标准化 | ETSI/ISO进展中 | NIST已发布 |
| 应用场景 | 高安全专线 | 互联网规模部署 |
| 量子安全级别 | 理论无条件安全 | 假设算法安全 |

**PR文章采用PQC路线的原因**：
- PQC更适合大规模移动网络部署
- 成本更低，部署更灵活
- 标准化程度更高
- 与现有设备兼容性更好

### 9.2 第一性原理总结

**为什么量子计算威胁密码学？**
- Shor算法将大数分解和离散对数问题从指数复杂度降为多项式复杂度
- 当前公钥密码学的数学基础被量子计算"降维打击"

**为什么选择PQC而非QKD？**
- PQC基于算法复杂性，QKD基于物理原理
- PQC更适合大规模、低成本部署
- 移动网络的安全需求更匹配PQC方案

**为什么现在就要部署？**
- **Harvest Now, Decrypt Later攻击**：今天的加密数据可能在10-20年后被破解
- 敏感数据（医疗、金融、国家安全）需要数十年保护期
- 网络基础设施升级周期长，需提前规划

### 9.3 技术关键公式回顾

**Shor算法核心**（周期查找）：
$$\text{QFT}: |x\rangle \rightarrow \frac{1}{\sqrt{Q}}\sum_{k=0}^{Q-1} e^{2\pi i x k / Q}|k\rangle$$

**ML-KEM核心**（LWE问题）：
$$b = A \cdot s + e \mod q$$

**Hybrid密钥组合**：
$$K_{final} = \text{HKDF}(K_{classical} \| K_{PQC})$$
