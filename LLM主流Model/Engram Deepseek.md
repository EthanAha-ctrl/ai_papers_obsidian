**根本性问题**：语言建模本质上涉及两种**异质性任务**：

| 任务类型     | 特征         | 传统处理方式                | 效率问题 |
| -------- | ---------- | --------------------- | ---- |
| **组合推理** | 深度、动态计算    | 通过Attention + FFN高效处理 | ✓ 合理 |
| **知识检索** | 局部、静态、高度刻板 | 通过计算**模拟**检索          | ✗ 低效 |

### 1.2 关键洞察

论文发现：

1. **静态模式的大量存在**：文本中如命名实体、公式化模式等本质上是局部的、静态的、高度刻板的
2. **NN-gram依然有效**：经典NN-gram模型（n>=2）在捕获这类局部依赖上非常有效
3. **计算浪费**：标准Transformer缺乏原生知识查找原语，被迫通过计算**重构**静态查找表

**案例**：识别"Diana, Princess of Wales"实体，LLM需要消耗6层Attention和FFN逐步组合特征（Table 3）：

| Layer | 内部表示 | 阶段 |
|-------|---------|------|
| 1-2 | United Kingdom中的国家 | 位置识别 |
| 3 | Europe中的国家 | 地理泛化 |
| 4 | 女性统治者的头衔 | 头衔类型 |
| 5 | 威尔士亲王妻子的头衔 | 头衔具体化 |
| 6 | Diana, Princess of Wales (1961-1997) | 完整实体识别 |

这个过程本质上是**昂贵的运行时重构**，应该通过 O(1) 查找完成！

### 1.3 解决方案：条件记忆 + 条件计算

| 稀疏维度 | 原语 | 优化目标 | 代表技术 |
|---------|------|---------|---------|
| **条件计算** | 稀疏激活 | 动态逻辑 | MoE |
| **条件记忆** | 稀疏查找 | 静态知识 | Engram (本文提出) |

## 二、Engram架构详解

### 2.1 整体架构

```
输入序列 X = (x₁, x₂, ..., x_T)
    ↓
第ℓ层隐藏状态 H^(ℓ) ∈ ℝ^(T×d)
    ↓
┌─────────────────────────────────────────┐
│         Engram Module                   │
├─────────────────────────────────────────┤
│ Phase 1: Sparse Retrieval               │
│  ├─ Tokenizer Compression               │
│  └─ Multi-Head Hashed NN-gram Lookup    │
│    → 生成静态记忆向量 e_t ∈ ℝ^d_mem      │
│                                         │
│ Phase 2: Context-aware Gating           │
│  ├─ Query: 当前隐藏状态 h_t             │
│  ├─ Key/Value: 检索的记忆 e_t           │
│  └─ Gate: α_t = σ(RMSNorm(h_t)ᵀ · k_t/√d) │
│                                         │
│ Phase 3: Convolution Fusion             │
│  └─ 深度因果卷积 +残差连接              │
└─────────────────────────────────────────┘
    ↓
输出: H^(ℓ) ← H^(ℓ) + Y
```

### 2.2 Phase 1: 稀疏检索

#### 2.2.1 Tokenizer Compression

**问题**：标准subword tokenizer优先无损重构，将语义等价项分配不同ID（如 Apple vs ␣apple）

**解决方案**：基于NFKC归一化、小写化等创建满射映射

```
映射函数 𝒫: V → V'
原始token ID: x_t
压缩ID: x'_t = 𝒫(x_t)

形成后缀NN-gram: g_{t,n} = (x'_{t-n+1}, ..., x'_t)

效果：128k tokenizer → 23% 词汇表缩减
```

#### 2.2.2 Multi-Head Hashing

为了参数化NN-gram的巨大组合空间，采用哈希方法：

**公式定义**：

对于每个NN-gram阶数 n 和哈希头 k：

```
z_{t,n,k} ≜ φ_{n,k}(g_{t,n})  # 哈希索引
e_{t,n,k} = E_{n,k}[z_{t,n,k}]  # 嵌入查找
```

其中：
- φ_{n,k}: 乘法-XOR轻量级哈希函数
- E_{n,k}: 嵌入表（大小为素数 M_{n,k}）

**记忆向量拼接**：

```
e_t ≜ ∥_{n=2}^{N} ∥_{k=1}^{K} e_{t,n,k}  ∈ ℝ^d_mem
```

**配置示例**（Engram-27B）：
- NN-gram阶数: N = 3（使用2-gram和3-gram）
- 哈希头数: K = 8
- 内存维度: d_mem = 1280
- 词汇表大小: 2,262,400（压缩后）

### 2.3 Phase 2: 上下文感知门控

检索的嵌入 e_t 是上下文无关的，可能因哈希碰撞或多义性产生噪声。门控机制利用当前隐藏状态进行动态调制。

**注意力式门控**：

```
Key:   k_t = W_K · e_t
Value: v_t = W_V · e_t
Query: q_t = h_t (当前隐藏状态)

门控系数:
α_t = σ( (RMSNorm(h_t)ᵀ · RMSNorm(k_t)) / √d )
```

**门控输出**：

```
ṽ_t = α_t · v_t
```

**RMSNorm用于梯度稳定性**（避免Softmax饱和）

**语义对齐**：当检索记忆与当前上下文矛盾时，α_t → 0，有效抑制噪声。

### 2.4 Phase 3: 卷积融合

为扩展感受野并增强非线性，添加深度因果卷积：

```
Y = SiLU( Conv1D( RMSNorm(Ṽ) ) ) + Ṽ
```

- 卷积核大小: w = 4
- 膨胀系数: δ = N（最大NN-gram阶数）
- 激活函数: SiLU (Swish)

**残差连接**：最终输出通过残差连接融入主干

```
H^(ℓ) ← H^(ℓ) + Y
```

### 2.5 多分支架构集成

论文采用Manifold-Constrained Hyper-Connections (M=4，四种分支)作为主干架构。

**参数共享策略**：

```
共享: 单个稀疏嵌入表 +单一Value投影矩阵 W_V

分支特定: M个不同的Key投影矩阵 {W_K^(m)}_m=1^M

第m个分支的特定门控:
α_t^(m) = σ( (RMSNorm(h_t^(m))ᵀ · RMSNorm(W_K^(m)·e_t)) / √d )

分支特定输出:
u_t^(m) = α_t^(m) · (W_V · e_t)
```

**优化**：线性投影合并为单个FP8矩阵乘法，最大化GPU利用率。

## 三、系统效率：计算与内存解耦

### 3.1 训练阶段

- **模型并行**：大规模嵌入表分片到可用GPU
- **All-to-All通信**：在forward前向传播中收集激活行，在backward后向传播中分发梯度
- **线性扩展**：总内存容量随加速器数量线性增长

### 3.2 推理阶段：预取与重叠

**关键优势**：Engram的检索索引仅依赖于输入token序列，确定性地址支持预取。

```
时间轴:
t₀: 确定Engram层的内存索引
t₁: 主机异步预取嵌入 (PCIe传输)
t₂: 执行前序Transformer层 (GPU计算)
t₃: Engram嵌入就绪，无缝执行查找
```

**分层缓存架构**：
- 热数据 → GPU HBM
- 温数据 → Host DRAM
- 冷数据 → NVMe SSD

**Zipf分布利用**：20%的NN-gram模式占80%的访问量。

## 四、Scaling Laws与稀疏分配

### 4.1 稀疏分配问题

**定义**：给定固定总参数预算，如何在MoE专家和Engram嵌入之间分配稀疏容量？

**关键指标**：

```
P_tot   : 总可训练参数（不含词嵌入和LM head）
P_act   : 每个token激活的参数（决定训练成本/FLOPs）
P_sparse ≜ P_tot - P_act: 非激活参数（"免费"参数预算）
```

**分配比例**：

```
ρ ∈ [0,1]: 分配给MoE专家能力的非激活参数比例

P_MoE^(sparse) = ρ · P_sparse
P_Engram = (1-ρ) · P_sparse
```

- ρ = 1: 纯MoE模型
- ρ < 1: 减少路由专家，重新分配参数给Engram嵌入槽位

### 4.2 U型缩放定律

**实验设置**：

| 计算预算 | P_tot | P_act | MoE专家数 |
|---------|-------|-------|-----------|
| C = 2×10²⁰ FLOPs | 5.7B | 568M | 106 |
| C = 6×10²⁰ FLOPs | 9.9B | 993M | 99 |

**发现**：验证损失与分配比例ρ呈现**一致U型关系**

```
Figure 3 (left): Validation Loss vs ρ

优化点: ρ ≈ 75%-80%

在10B量级:
- ρ = 100% (纯MoE): Loss = 1.7248
- ρ ≈ 80% (混合): Loss = 1.7109 (Δ = 0.0139)
```

**解释**：

| 区域 | 特征 |
|------|------|
| MoE主导 (ρ → 100%) | 缺乏静态模式专用记忆，被迫通过深度低效重构 |
| Engram主导 (ρ → 0%) | 失去条件计算能力，伤害动态上下文推理任务 |

**关键洞察**：约20%-25%的稀疏参数分配给Engram可提供最佳性能。

### 4.3 无限内存缩放

**设置**：固定MoE主干（P_tot≈3B, P_act=568M），在Engram表上扩展嵌入槽位M

```
M范围: 2.58×10⁵ → 1.0×10⁷ ( ≈13B附加参数)
训练: 100B tokens确保收敛
```

**结果**：验证损失严格遵循幂律（对数空间线性），表明：

```
更大内存 → 持续改进 → 无需额外计算

Engram vs OverEncoding比较:
- OverEncoding简单平均法：从大内存表受益有限
- Engram上下文感知门控：从相同内存预算释放更大缩放潜力
```

## 七、机制分析

### 7.1 Engram是否等效于增加模型深度？

#### 7.1.1 LogitLens分析

**方法**：将每层的隐藏状态通过最终LM head投影，计算与最终输出分布的KL散度

**定义**：衡量中间表示距离"预测就绪"状态的程度

```
D_KL(p_final || p_layer)
```

**结果**：Engram模型的早期层KL散度**系统性地较小**，曲线下降更陡峭

```
Layer-wise KL Divergence:
- MoE: 较高的早期层散度（特征组合完成较慢）
- Engram: 较低的早期层散度（通过显式知识访问加速预测收敛）

→ Engram通过显式访问外部知识减少了计算步骤，
  在网络层次结构中更快达到高置信度、有效预测
```

#### 7.1.2 CKA分析

**CKA定义**：

```
CKA(K, L) = HSIC(K, L) / √(HSIC(K, K) · HSIC(L, L))

其中:
K = XX^T, L = YY^T (线性核的Gram矩阵)
HSIC: Hilbert-Schmidt Independence Criterion
```

**软对齐指数**：

```
a_j = Σ_{i∈ℐ_j} S_{i,j} · i / Σ_{i∈ℐ_j} S_{i,j}

其中:
ℐ_j = argtop_k_i(S_{i,j})  # j层Engram与k个最相似MoE层的集合
```

**结果**：一致性对角线向上偏移

```
Engram Layer 5 ≈ MoE Layer 12
Engram Layer 8 ≈ MoE Layer 15
Engram Layer 12 ≈ MoE Layer 20

→ Engram在更浅的层实现更深层的表示
```

**验证假设**：通过显式查找绕过早期特征组合，Engram功能上等效于增加模型的有效深度。

### 7.2 结构消融实验

#### 7.2.1 层敏感度

**设置**：固定1.6B Engram预算，将单个Engram模块从Layer 1扫描到12

**结果**：

| 插入层 | Val Loss |
|--------|----------|
| Layer 1 | 1.775 |
| **Layer 2** | **1.770** |
| Layer 3 | 1.778 |
| Layer 6 | 1.790 |
| Layer 12 | 1.808 |

**Layer 2最优**的解释：
- 早期插入允许在主干消耗计算深度之前卸载局部模式重构
- 一轮Attention已足够提供足够上下文h_t用于门控
- 仍足够早以取代主干的底层局部聚合

**分层设计**：将1.6B内存分为两个较小模块（Layers 2和6），性能更优（Val Loss = 1.768）

#### 7.2.2 组件消融

| 组件 | Val Loss | 损失 |
|------|----------|-----+|

## 八、相关工作对比

### 8.1 NN-gram建模与嵌入缩放

| 方法 | 核心技术 | 局限 |
|------|---------|------|
| **SuperBPE** | 合并多词表达式为"超级词"token | - |
| **SCONE** | 辅助编码模型的高频模式 | 推理聚焦，需额外训练FLOPs |

1. 作为**一等建模原语**，在严格公平比较协议下验证
2. **算法-系统协同设计**：分层插入使通信-计算重叠
3. 利用**硬件内存层次结构**（Zipf分布）

| 类别 | 代表方法 | 特点 |
|------|---------|------|
| **参数记忆** | PKM, PEER, SelfMem, Memory+, UltraMem | 大规模稀疏键值存储集成到模型层 |
| **非参数记忆** | REALM, RETRO, PlugLM | 外部记忆作为可编辑可扩展的键值存储 |

### 8.4 知识存储机制

**FFN as Key-Value Memory假说**：
- 第1层：模式检测器（"keys"）
- 第2层：将特定信息投影到残差流（"values"）

**知识神经元**：存储特定事实的特定FFN神经元

**因果关系追踪**：将事实回忆的信息流映射到特定FFN层

**模型编辑**：直接更新事实关联无需重训练（ROME, MEMIT）

## 九、技术公式总结

### 9.1 核心公式

**1. 哈希NN-gram检索**

```
z_{t,n,k} = φ_{n,k}(g_{t,n})
e_{t,n,k} = E_{n,k}[z_{t,n,k}]
e_t = ∥_{n=2}^{N} ∥_{k=1}^{K} e_{t,n,k}
```

**2. 上下文感知门控**

```
k_t = W_K · e_t
v_t = W_V · e_t
α_t = σ( (RMSNorm(h_t)ᵀ · RMSNorm(k_t)) / √d )
ṽ_t = α_t · v_t
```

**3. 卷积融合**

```
Y = SiLU( Conv1D( RMSNorm(Ṽ) ) ) + Ṽ
H^(ℓ) ← H^(ℓ) + Y
```

**4. 分支特定门控（M分支）**

```
α_t^(m) = σ( (RMSNorm(h_t^(m))ᵀ · RMSNorm(W_K^(m)·e_t)) / √d )
u_t^(m) = α_t^(m) · (W_V · e_t)
```

**5. 稀疏分配**

```
P_sparse = P_tot - P_act
P_MoE^(sparse) = ρ · P_sparse
P_Engram = (1 - ρ) · P_sparse
```

**6. CKA相似度**

```
K = XX^T, L = YY^T
HSIC(K, L) = trace( K · H · L · H ) / (n-1)²
CKA(K, L) = HSIC(K, L) / √(HSIC(K, K) · HSIC(L, L))
```

## 十、论文贡献与影响

### 10.1 主要贡献

1. **概念**：提出**条件记忆**作为新稀疏维度，与条件计算互补
2. **架构**：现代NN-gram嵌入模块，支持O(1)查找
3. **理论**：**U型缩放定律**——MoE和Engram间的最优分配
4. **系统**：算法-系统协同设计，支持计算-内存解耦
5. **验证**：27B参数模型，在多个领域超越严格iso-参数、iso-FLOPs MoE基准

### 10.2 实验验证

1. **有效深度增加**：早期层Engram实现更深层的MoE层表示
2. **预测加速**：显式知识访问减少预测收敛所需的计算步骤

1. **确定性地址**：支持预取和通信-计算重叠

3. **分层缓存**：利用Zipf分布，热数据HBM，冷数据SSD
4. **低开销**：100B Engram表卸载开销 <3%
