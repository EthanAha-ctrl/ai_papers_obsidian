

# Fisher–Yates Shuffle：从第一性原理出发的深度直觉构建

---

## 一、本质是什么？——一个最本源的问题

想象你有一副扑克牌，52 张，你想把它们**彻底打乱**。什么叫"彻底打乱"？这不是一个模糊的感觉——它的精确定义是：

> **每一个可能的排列（permutation）出现的概率都必须完全相等，即 1/n!。**

这就是 Fisher–Yates shuffle 要解决的核心问题。它是一个**生成均匀随机排列（uniform random permutation）** 的算法。

从第一性原理出发：如果我们有 n 个元素，那么一共有 n! 种排列。如果我们能构造一个过程，使得**每一种排列恰好对应一条唯一的执行路径**，且每条路径的概率相等，那我们就得到了一个无偏的 shuffle。

---

## 二、原始方法：Fisher 和 Yates 的铅笔+纸方案（1938）

 Ronald Fisher 和 Frank Yates 在 1938 年的书中提出了原始方法，那时还没有计算机，他们用的是**铅笔、纸和随机数表**。

### 核心思想（直觉构建）

想象你面前有一列数字 1 到 N，你想要生成一个随机排列。做法极其自然：

1. **写下** 1 到 N 的所有数字（这是"未使用池"）
2. 从剩余的数字中**随机选一个**（范围从 1 到"剩余个数"）
3. 把选中的数字**划掉**，写到结果列表的末尾
4. 重复直到所有数字都被划掉

### 为什么它是无偏的？

让我们用第一性原理来论证：

- 第 1 步：从 n 个中选 1 个，概率各 1/n
- 第 2 步：从剩余 n-1 个中选 1 个，概率各 1/(n-1)
- …
- 最后 1 步：只剩 1 个，概率 1

所以任意一个特定排列被选中的概率 = 1/n × 1/(n-1) × … × 1/1 = **1/n!** ✅

每一条"选择路径"唯一对应一个排列，排列之间没有重叠，概率均等。**这就是无偏性的根源。**

### 问题在哪？

用计算机实现时，"划掉第 k 个未被划掉的数字"这个操作需要**线性扫描**来找到第 k 个未划掉的元素。这导致时间复杂度为 **O(n²)**。对于大规模数据，这是不可接受的。

---

## 三、现代方法：Durstenfeld 的关键洞察（1964）

Richard Durstenfeld 做了一个极其优雅的改进：**不要划掉然后复制到别处——直接 swap！**

### 核心洞察（直觉构建）

想象一个数组，分成两个区域：

```
[  未决定区  |  已决定区  ]
```

- **已决定区**（数组尾部）：这些位置的元素已经被"锁定"，不会再动
- **未决定区**（数组前部）：这些位置的元素还在等待被选择

每次从"未决定区"随机选一个元素，把它和"未决定区"的**最后一个元素**交换，然后把"未决定区"的边界缩小一格——选中的元素就这样自然地进入了"已决定区"。

### 伪代码

```
for i from n-1 down to 1:
    j ← random integer such that 0 ≤ j ≤ i
    exchange a[j] and a[i]
```

### 为什么这等价于原始方法？

关键洞察：**"划掉第 k 个未划掉的元素"和"从 0..i 范围中随机选一个然后 swap 到位置 i"在概率分布上是完全等价的！**

在原始方法中，我们从"剩余元素"中均匀选择一个。在现代方法中，我们从索引 0..i 中均匀选择一个。两者都是"从剩余池中均匀选一个"，只是物理操作不同：

- 原始：划掉 + 复制到别处 → O(n) 因为需要找到"第 k 个未划掉的"
- 现代：swap 到末尾 → O(1) 因为数组支持随机访问

**时间复杂度从 O(n²) 降到 O(n)**，这就是 Durstenfeld 的贡献。

## 四、正向版本：从左到右还是从右到左？

算法可以反向执行：

```python
# 从右到左（经典版）
for i from n-1 down to 1:
    j ← random integer such that 0 ≤ j ≤ i
    swap a[j], a[i]

# 从左到右（正向版）
for i from 0 to n-2:
    j ← random integer such that i ≤ j ≤ n-1
    swap a[i], a[j]
```

直觉上：你可以把"已决定区"放在左边也可以放在右边，效果一样。还可以这样理解——**均匀随机排列的逆也是均匀随机的**，所以方向无关紧要。

## 六、Sattolo's Algorithm：一个危险的"亲戚"

Sattolo 算法只有**一个字符的区别**：

```
Fisher-Yates:  j ← random integer such that 0 ≤ j ≤ i
Sattolo:       j ← random integer such that 0 ≤ j ≤ i-1
```

**j 不能等于 i**——这意味着元素永远不能和自己交换（即不能留在原位）。

### 这个微小变化的巨大后果

Fisher-Yates 产生所有 **n!** 种排列，每种概率 1/n!。

Sattolo 只产生 **(n-1)!** 种排列，每种概率 1/(n-1)!——而且它们全都是**长度为 n 的循环排列（cyclic permutation）**。

**直觉理解：** 如果 j 永远不等于 i，那么每个元素都必须移动到别的位置，没有"不动"的可能。这就强制了整个排列形成一个单一的环。循环排列的总数恰好是 (n-1)!。

### 为什么这很危险？

因为一个 off-by-one 错误（`randint(0, i)` 写成 `randint(0, i-1)`）就会让你的 Fisher-Yates 变成 Sattolo！表面上看起来"也在打乱"，但结果**根本不是均匀分布**——你永远得不到任何元素在原位的排列。

## 七、Naïve Shuffle：一个经典的错误

```python
# 错误示范！
for i in range(n):
    j = randrange(n)  # 每次都从 0..n-1 选！
    swap(a[i], a[j])
```

### 为什么这是错的？

**从第一性原理来算：** 这个算法有 n 轮，每轮从 n 个位置选一个，所以一共有 **nⁿ** 条可能的执行路径。但排列只有 **n!** 种。

关键问题：**nⁿ 不能被 n! 整除**（当 n > 2 时）。

- n = 3 时：3³ = 27 条路径，3! = 6 种排列。27 ÷ 6 = 4.5——不是整数！
- 这意味着**某些排列必然比其他排列多对应一些路径**
- 具体地：[1,2,3]、[3,1,2]、[3,2,1] 各对应 4/27，而 [2,1,3]、[1,3,2]、[2,3,1] 各对应 5/27

**直觉：** 当 n > 2 时，根据 Bertrand 公设，n/2 到 n 之间一定有一个素数 p，p 整除 n! 但不整除 nⁿ，所以 nⁿ/n! 不可能是整数。

## 八、排序法（Sort-based Shuffle）：另一种思路

给每个元素分配一个随机数，然后按随机数排序。

- 优点：容易并行化
- 缺点：
  - 需要 O(n) 额外空间（存随机数）
  - 随机数重复时排序不稳定会导致偏差
  - **用随机比较函数排序是大坑**：因为随机比较函数不满足传递性（transitivity），会导致排序算法崩溃或产生极度不均匀的结果

### 为什么随机比较函数特别糟糕？

以 quicksort 为例：pivot 的最终位置由"比它小"和"比它大"两组的大小决定。如果每次比较都是 50/50 的随机结果，那么 pivot 的位置服从二项分布——中间位置的概率远高于两端。这**严重偏离**了均匀分布。

更深层的原因：**任何有限次数的等概率二值事件（抛硬币）的组合，其概率的分母都是 2 的幂。而 1/n! 的分母通常不是 2 的幂。** 所以纯粹靠抛硬币永远无法得到均匀排列。

## 九、偏差的来源：实现中的陷阱

### 1. 范围错误（Off-by-one）

如前所述，`0 ≤ j ≤ i` 变成 `0 ≤ j < i` 就变成了 Sattolo。

### 2. Modulo Bias

当你用 `random() % range` 来获取随机数时：

- 假设 random() 返回 0..99（100 个值）
- 你要 0..15（16 个值）
- 100 ÷ 16 = 6 余 4
- 所以 0..3 出现 7 次，4..15 出现 6 次
- **0..3 出现的概率比其他高约 17%！**

解决方案：**拒绝采样（rejection sampling）**——如果随机数落在"不完整"的区域（96..99），就丢弃重试。期望重试次数 < 1。

Daniel Lemire 在 2018 年提出了一个几乎不需要 modulo 操作的无偏方法。

### 3. 浮点数偏差

用 `Math.floor(Math.random() * range)` 也有问题——浮点数精度有限，某些区间会有更多的可表示浮点值。

## 十、代码实现

### Python

```python
import random

def shuffle(numbers: list[int]) -> list[int]:
    for i in range(len(numbers) - 1, 0, -1):
        j = random.randint(0, i)       # 关键：0 ≤ j ≤ i，包含 i！
        numbers[i], numbers[j] = numbers[j], numbers[i]
    return numbers
```

### JavaScript

```javascript
function shuffleArray(array) {
    for (let i = array.length - 1; i >= 1; i--) {
        const j = Math.floor(Math.random() * (i + 1));  // 关键：i+1 保证 j 可以等于 i
        [array[i], array[j]] = [array[j], array[i]];
    }
    return array;
}
```

## 十一、并行化

Fisher-Yates 本质上是**顺序的**——每一步依赖于前一步的结果。但研究者已经提出了并行变体：

- **Anderson (1990)**：共享内存的并行版本
- **MERGESHUFFLE (Bacher et al., 2015)**：分块 + Fisher-Yates 各自 shuffle + 随机 merge
- **Shun et al. (2015)**：直接并行化，达到线性工作量（linear work）和 polylog 深度（depth）

## 十二、总结：第一性原理的直觉地图

```
问题：生成均匀随机排列（每种排列概率 = 1/n!）
│
├─ 核心原理：每一步从"剩余池"中均匀选择一个
│   ├─ 路径唯一性：每条选择路径 → 唯一排列
│   └─ 路径等概率：1/n × 1/(n-1) × ... × 1/1 = 1/n!
│
├─ 原始实现（Fisher-Yates 1938）
│   └─ 划掉 + 复制 → O(n²)（找第 k 个未划掉的元素是线性扫描）
│
├─ 现代实现（Durstenfeld 1964）
│   └─ swap 到末尾 → O(n)（数组随机访问是 O(1)）
│
├─ 常见错误
│   ├─ Naïve shuffle：nⁿ ≠ n! 的整除性 → 有偏
│   ├─ Off-by-one → 变成 Sattolo（只产生循环排列）
│   └─ Modulo bias / 浮点偏差
│
└─ 正确实现的关键
    ├─ j 的范围必须包含 i（允许"自己和自己换"= 留在原位）
    └─ 随机数生成必须无偏
```

**一句话总结：** Fisher-Yates shuffle 的精髓就是——**每一步从"还没决定的位置"中均匀随机选一个，然后 swap 到"已决定"的区域。** 这个看似简单的操作，保证了 n! 种排列中的每一种都恰好通过唯一一条等概率路径被产生。所有偏差都来自对这个原则的微小偏离。
