# 探索非抓取式操作如何扩展机器人在多关节故障下的能力

## 一、研究背景与核心问题

### 1.1 问题背景

这篇文章由科罗拉多大学博尔德分校的研究团队完成，发表在arXiv上（2024年10月），核心研究问题非常明确：**当机器人发生锁定多关节故障时，如何保持操作能力**。

**关键定义：**
- **Locked Multi-Joint (LMJ) Failure**：两个或更多关节同时失效，成为不可操作状态
- **Non-Prehensile Manipulation (NPM)**：除抓取外的任何操作方式，包括推、戳、滑、滚等
- **Prehensile Manipulation (PM)**：传统的抓取式操作
- **Ability**：机器人可以调用来操作物体的运动技能
- **Capability**：机器人完成操作任务的程度

### 1.2 LMJ故障带来的挑战

当机器人发生LMJ故障时，会面临以下五种约束：

| 约束类型 | 技术说明 |
|---------|---------|
| **末端执行器耦合** | 末端执行器的位置和姿态与剩余功能关节耦合，失去独立控制能力 |
| **操作空间缩小** | 可操作区域缩小到非故障情况的一个子集 |
| **抓取受限** | 由于任务约束（物体处于不可抓取姿态），抓取可能不可行 |
| **完全丧失末端执行器控制** | 由于不良配置，机器人可能完全失去末端执行器控制能力 |
| **非完整约束** | 末端执行器受到非完整约束约束[8] |

## 三、核心技术细节

### 3.1 Reach-Ability Map 生成算法 (Algorithm 1)

**输入参数说明：**
- **W**: 机器人的完整工作空间
- **X**: 关节约束集合（故障关节的锁定位置）
- **k**: 重新规划尝试次数
- **ε**: 扰动半径

**算法流程解析：**

```python
Algorithm 1: Generate Reach-Ability Map

# 初始化
iter = 0
W_c = ∅  # 故障约束工作空间

# 遍历工作空间每个离散化点
for p ∈ W do
    # 首先尝试 Prehensile Manipulation (PM)
    INTERACTION_TYPE = "PM"
    INTERACTION_AREA = "End Effector"
    success ← INVERSE_KINEMATICS(p, X, INTERACTION_TYPE, INTERACTION_AREA)
    
    # 如果失败，进行扰动重试
    while iter < k do
        if iter > 1 then
            # 在ε-ball内均匀采样扰动
            ρ ← SAMPLE_UNIFORM_EPSILON_BALL(ε)
            p_ρ ← p + ρ
        
        # 超过一半尝试次数后切换到NPM模式
        if iter > k/2 then
            INTERACTION_TYPE, INTERACTION_AREA ← CHANGE_INTERACTION()
        
        success ← INVERSE_KINEMATICS(p_ρ, X, INTERACTION_TYPE, INTERACTION_AREA)
        
        if success then
            W_c ← W_c ∪ p
        iter ← iter + 1

return W_c
```

**关键技术创新：**

1. **分层探索策略**：
   - 首先尝试末端执行器的抓取姿态（PM模式）
   - 约束要求：末端执行器姿态在垂直于工作表面到平行于工作表面之间

2. **自适应模式切换**：
   - 当PM模式失败次数超过k/2时，自动切换到NPM模式
   - NPM模式下：
     * 循环遍历预选的机器人交互点：手、腕、前臂
     * 放松姿态误差约束，仅限制位置误差

3. **扰动重采样**：
   - 在ε-ball内均匀采样扰动点
   - ε设置为离散化单元格内接球的半径

### 3.2 Kinodynamic Edge Bundles 构造 (Algorithm 2)

**数学基础：**

定义以下流形：
- **C**: 所有可能的关节状态流形
- **U**: 有效的控制输入流形
- **W**: 末端执行器工作空间

当LMJ发生时，这些流形退化为：
- **C_c**: 故障约束的配置空间
- **U_c**: 故障约束的控制空间
- **W_c**: 故障约束的工作空间

**维度损失分析：**

对于n个锁定关节，流形C的维度减少n个：

```
dim(C_c) = dim(C) - n
dim(U_c) = dim(U) - n
```

**关键洞察**：由于这些映射是非线性的且依赖于故障配置，约束流形的体积远小于未约束流形。因此，从C_c中均匀随机采样找到有效连接规划的概率接近于零[7]。

**算法详解：**

```python
Algorithm 2: Generate Kinodynamic Manifold

Input: C_c, U_c, W_c, n, Δt
Output: E (Edge Bundle)

E ← ∅

for i = 1 to n do
    e ← ∅  # 初始化edge
    
    # 从故障约束工作空间采样点（仅XYZ）
    ξ_EE ← SAMPLE_POINT(W_c)
    
    # 获取无约束姿态的逆运动学解
    q ← POSITIONAL_INVERSE_KINEMATICS(_EE, C_c)
    
    # 采样随机控制持续时间和控制输入
    t ← SAMPLE_DURATION()
    q̇ ← SAMPLE_CONTROL_INPUT(U_c)
    
    valid ← True
    
    # 动力学仿真
    while TIME_REMAINING(t) do
        # 计算加速度 (使用伪逆)
        q̈ ← J(q)† × (q̇_des - J̇(q)q̇)
        
        # 计算关节力矩 (动力学方程)
        τ ← M(q)q̈ + C(q, q̇)q̇ + g(q) + f(q̇)
        
        # 可操作度计算
        m = det(J(q)J(q)^T)
        
        # 检查约束和碰撞
        if LIMITS_VIOLATED(q, q̈, τ) or IN_COLLISION(q) then
            valid ← False
            break
        
        # 前向积分
        q_{k+1} ← q_k + q̇Δt
        e ← e ∪ {q_k, q̇}
    
    if valid then
        E ← E ∪ e

return E
```

**动力学公式详解：**

1. **加速度计算方程**：
   ```
   q̈ = J†(q) × (ẍ_des - J̇(q)q̇)
   ```
   其中：
   - **q̈**: 关节加速度向量
   - **J†**: Jacobian矩阵的Moore-Penrose伪逆
   - **ẍ_des**: 期望的笛卡尔空间加速度
   - **J̇**: Jacobian矩阵的时间导数
   - **q̇**: 关节速度向量

2. **动力学方程**（拉格朗日形式）：
   ```
   τ = M(q)q̈ + C(q, q̇)q̇ + g(q) + f(q̇)
   ```
   其中：
   - **τ**: 关节力矩向量
   - **M(q)**: 惯性矩阵
   - **C(q, q̇)**: 科氏力和离心力矩阵
   - **g(q)**: 重力向量
   - **f(q̇)**: 摩擦力向量

3. **可操作度**：
   ```
   m = det(J(q)J(q)^T)
   ```
   用于衡量机器人在当前配置下的操作能力，m = 0表示处于奇异配置。

### 3.3 Edge Planner with Sim-in-the-Loop (Algorithm 3)

**三种动作选择机制对比：**

| 机制 | 策略 | 优点 | 缺点 | 适用场景 |
|-----|------|------|------|---------|
| **Random** | 从相交边中均匀随机选择 | 规划时间最短 | 成功率最低 | 快速试探 |
| **Lazy** | 找到第一个使目标朝向移动的动作 | 时间与成功率平衡 | 可能不是最优 | 通用场景 |
| **Greedy** | 评估所有候选动作，选择最优 | 成功率最高 | 规划时间最长 | 高可靠性需求 |

**Greedy算法详细实现：**

```python
Algorithm 3 (Greedy Mode):

Input: E, E (Environment), E (Edges), m="Greedy"
Output: A (Action)

A ← ∅
E_T ← ∅  # 已评分的边
scores ← ∅

# 从相交边中均匀采样子集
E_s ← UNIFORM_SAMPLE_SUBSET(E_I)

# 设置仿真环境
E_sim ← E

for e in SAMPLE_WITHOUT_REPLACEMENT(E_s) do
    # 在仿真中执行边
    SIMULATE_ROBOT_WITH_EDGE(e, E_sim)
    
    # 检查目标是否朝向目标移动
    if TARGET_MOVED_TOWARDS_GOAL(E_sim) then
        # 评分执行质量
        scores ← scores ∪ SCORE_EXECUTION(E_sim)
        E_T ← E_T ∪ e
    
    # 重置仿真环境
    E_sim ← E

# 选择最佳边
A ← SELECT_BEST_EDGE(E_T, scores, m)

return A
```

**评分函数可能包含的指标：**
1. 距离目标的减少量
2. 执行时间
3. 动作可行性（可操作度m）
4. 能量消耗
5. 碰撞风险

## 四、Poking Primitive 动力学分析

### 4.1 Poking 的两个阶段

**Phase 1: Impact（冲击阶段）**

```
F_impact × Δt = m_object × Δv_object
```

其中：
- **F_impact**: 冲击力
- **Δt**: 接触持续时间
- **m_object**: 物体质量
- **Δv_object**: 物体速度变化量

**Phase 2: Free-Sliding（自由滑动阶段）**

使用Coulomb摩擦模型：

```
μ_k × N = f_friction
a_decel = μ_k × g
d_slide = v_initial² / (2 × μ_k × g)
```

其中：
- **μ_k**: 动摩擦系数
- **N**: 法向力（mg cosθ）
- **f_friction**: 摩擦力
- **a_decel**: 减速度
- **g**: 重力加速度
- **d_slide**: 滑动距离

**参考链接：**
- Coulomb摩擦理论: https://www.sciencedirect.com/science/article/pii/S002250962100242
- 非抓取式操作综述: https://ieeexplore.ieee.org/document/8378774

### 4.2 PokeRRT的挑战

文章指出PokeRRT算法[25]面临的主要挑战：

1. **高计算需求**：需要模拟大量动作序列
2. **零测度流形**：在LMJ故障下，有效动作空间是零测度的
3. **均匀随机采样效率低**：在故障约束的配置空间中，均匀采样难以找到可行解

## 六、技术细节深入解析

### 6.1 逆运动学求解策略

**PM模式下的约束：**
```
q_pm ∈ {q | ||EE_pos(q) - target_pos|| < ε_pos ∧
            ||EE_ori(q) - ori_grasp|| < ε_ori ∧
            q ∈ JointLimits ∧
            CollisionFree(q)}
```

其中：
- **EE_pos**: 末端执行器位置
- **EE_ori**: 末端执行器姿态
- **ori_grasp**: 可抓取姿态范围（从垂直到平行于表面）
- **ε_pos**: 位置误差阈值
- **ε_ori**: 姿态误差阈值

**NPM模式下的约束：**
```
q_npm ∈ {q | ||ContactPoint_pos(q) - target_pos|| < ε_pos ∧
             q ∈ JointLimits ∧
             CollisionFree(q)}
```

注意到NPM模式：
1. 姿态约束被移除
2. 交互点可以是：末端执行器、腕部、前臂等全身部位

### 6.2 动力学仿真的挑战

**仿真与现实的偏差来源：**

### 6.3 多查询规划优势

**预计算的优势：**

1. **时间复杂度**：
   - 在线采样规划: O(采样次数 × 动作验证时间)
   - 预计算Edge Bundles: O(预计算时间) + O(查询时间)
   - 对于N个查询，当N > 预计算时间/在线时间时，预计算更高效

2. **概率完备性保证**：
   - Edge Bundles使用Monte Carlo方法
   - 随着采样数量增加，覆盖整个有效动作流形的概率趋向于1[31]

## 七、相关领域知识联想

### 7.1 非完整系统理论

**定义**：系统的配置空间维数大于其控制输入空间的维数

**数学表述**：
```
ẋ = Σ_{i=1}^{m} g_i(x)u_i
```
其中：
- **x ∈ ℝⁿ**: 配置向量（n维）
- **u_i ∈ ℝ**: 控制输入（m < n）
- **g_i(x)**: 向量场

**Chow定理**（可达性条件）：
如果分布D = span{g₁, ..., g_m, [g_i, g_j], ...}满秩（rank = n），则系统为小时间局部可达（STLA）。

其中[g_i, g_j]是李括号：
```
[g_i, g_j] = ∂g_j/∂x · g_i - ∂g_i/∂x · g_j
```

**在本文中的应用**：
LMJ故障后的机器人具有非完整约束，需要利用李括号生成额外的运动方向。

## 八、数学公式深入讲解

### 8.1 Jacobian伪逆计算

**Jacobian矩阵**：
```
J(q) = ∂x/∂q = [∂x/∂q₁, ..., ∂x/∂qₙ] ∈ ℝ^{m×n}
```

对于7自由度机械臂，n = 7，m = 6（位置3 + 姿态3）

**伪逆公式**：
```
J† = J^T(JJ^T)^{-1}
```

**奇异值分解（SVD）计算**：
```
J = UΣV^T
J† = VΣ^+U^T
```

其中：
- **U ∈ ℝ^{m×m}**: 左奇异向量矩阵
- **V ∈ ℝ^{n×n}**: 右奇异向量矩阵
- **Σ ∈ ℝ^{m×n}**: 奇异值对角矩阵
- **Σ^+**: Σ的伪逆（取非零奇异值的倒数）

### 8.2 拉格朗日动力学

**拉格朗日函数**：
```
L(q, q̇) = T(q, q̇) - V(q)
```
其中：
- **T**: 动能 = ½q̇^T M(q) q̇
- **V**: 势能 = V_g(q)（重力势能）

**欧拉-拉格朗日方程**：
```
d/dt(∂L/∂q̇) - ∂L/∂q = τ
```

展开得到：
```
M(q)q̈ + C(q, q̇)q̇ + g(q) = τ
```

**各矩阵元素**：
- **M(q)_{ij} = Σ_k m_k ∂p_k/∂q_i · ∂p_k/∂q_j**: 惯性矩阵
- **C(q, q̇)q̇_i = Σ_j Σ_k Γ_{ijk} q̇_j q̇_k**: 科氏力和离心力
- **g(q)_i = ∂V/∂q_i**: 重力项

### 8.3 Coulomb摩擦模型

**静摩擦**：
```
F_f ≤ μ_s N
```

**动摩擦**：
```
F_f = μ_k N · sign(v)
```

其中：
- **μ_s**: 静摩擦系数（通常 μ_s > μ_k）
- **μ_k**: 动摩擦系数
- **N**: 法向力
- **v**: 相对速度
- **sign()**: 符号函数

**Stribeck曲线**（更精确的模型）：
```
μ(v) = μ_d + (μ_s - μ_d) · exp(-|v/v_s|^δ)
```

## 十六、总结与直觉构建

### 16.1 核心思想总结

这篇文章的核心直觉是：**当机器人的"手"坏了，我们不应该只是修手，而应该学会用"全身"来操作物体**。

类比人类：
- 当手腕受伤无法用手抓东西时，我们会用肘推、用背顶、用脚踢
- 这就是Non-Prehensile Manipulation的本质

### 16.2 关键技术洞察

1. **故障后的空间探索是高效的**：
   - 预计算一次，多次使用
   - 避免在线大量采样失败

2. **Poking是一种强力的操作基元**：
   - 不受机器人臂长限制
   - 可以越过障碍物
   - 适合精确定位

3. **仿真是连接理论和现实的桥梁**：
   - 评估动作质量
   - 平衡探索与利用
   - 降低实际风险

### 16.3 为什么有效？

```
传统方法: 操作能力 = 末端执行器能力 × 关节可用性
         当关节故障 → 能力急剧下降

本文方法: 操作能力 = ∑_{全身部位} (部位能力 × 关节可用性)
         即使末端执行器故障 → 其他部位仍可工作
```

### 16.4 进一步思考方向

**理论方向**：
