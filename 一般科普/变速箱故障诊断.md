### 2.3 轻量级高效通道注意力机制

这是论文的**核心创新点**。

#### 🔹 传统 SE 模块的问题

SE (Squeeze-and-Excitation) 模块通过两个全连接层实现通道注意力：

```
输入 → Global Avg Pool → FC(降维) → ReLU → FC(升维) → Sigmoid → 通道加权
```

**问题**：降维操作会丢失通道间的交互信息，降低模型学习能力。

#### 🔹 ECA 模块的改进

ECA-Net 提出用 **1D 卷积替代全连接层**，实现局部跨通道交互，**不降维**：

```
输入 → Global Avg Pool → 1D Conv(kernel=K) → Sigmoid → 通道加权
```

#### 🔹 LECA 模块的创新

论文在 ECA 基础上进一步改进，设计了 **Lightweight Efficient Channel Attention (LECA)**：

**架构细节（Figure 4）：**

```
输入特征图 f ∈ R^{H×W×C}
         ↓
    ┌────┴────┐
    ↓         ↓
Avg Pool   Max Pool  (多尺度特征提取)
    ↓         ↓
    └────┬────┘
         ↓
     Concat (z₁, z₂)
         ↓
   1D Conv(kernel=K)
         ↓
   Batch Normalization
         ↓
   Hard-Sigmoid 激活
         ↓
     通道权重 w
         ↓
   通道加权输出 X̃c
```

**核心公式：**

1. **多尺度特征聚合：**

$$z_1 = \frac{1}{H \times W} \sum_{i=1}^{h} \sum_{j=1}^{w} f(i,j) \quad \text{(Global Average Pooling)}$$

$$z_2 = \max f(i,j) \quad \text{(Global Max Pooling)}$$

**解释**：
- $z_1$ 捕获**全局信息**（每个通道的整体响应）
- $z_2$ 捕获**局部显著特征**（最强烈的激活点）
- 两者结合 = 更丰富的特征表示

2. **自适应卷积核大小计算：**

$$K = \varphi(C) = \left| \frac{\log_2(C) + b}{\gamma} - \left| \frac{\gamma}{b} \right|_{even} \right|_{odd}$$

**变量解释：**
- $C$：通道数
- $\gamma = 2$，$b = 1$：预设系数
- $|\cdot|_{even}$：取最接近的偶数
- $|\cdot|_{odd}$：取最接近的奇数

**直觉理解**：
- 通道数 $C$ 越大 → $K$ 越大 → 覆盖更多通道的交互
- 减去偶数项是为了进一步**轻量化**（减少计算量）

3. **通道权重生成：**

$$w = \sigma(C1D_k(y))$$

- $C1D_k$：kernel size 为 $K$ 的 1D 卷积
- $\sigma$：**Hard-Sigmoid** 激活函数（比 Sigmoid 计算更快）

$$HardSigmoid(x) = \begin{cases} 0 & x \leq -3 \\ 1 & x \geq 3 \\ \frac{x}{6} + 0.5 & \text{otherwise} \end{cases}$$

4. **特征加权：**

$$\tilde{X}_c = w \otimes f(i,j)$$

- $\otimes$：逐通道相乘
- 输出与输入维度相同

### 2.4 网络架构：LECA-EfficientNetV2

论文以 **EfficientNetV2** 为基础网络，进行以下改进：

#### 🔹 基础模块

| 模块 | 结构 | 特点 |
|------|------|------|
| **MBConv** | 1×1 Conv(升维) → Depthwise Conv → SE → 1×1 Conv(降维) | 深层网络使用，轻量高效 |
| **Fused-MBConv** | 3×3 Conv → SE → 1×1 Conv | 浅层网络使用，训练更快 |

#### 🔹 网络配置

**核心设计策略：**
- **浅层**：使用 Fused-MBConv（标准卷积，速度快）
- **深层**：使用 MBConv + **LECA 模块**（替代 SE）

**完整网络结构（Table 1）：**

| 层 | 操作符 | 输出尺寸 | 通道数 | LECA | 步长 |
|---|--------|---------|-------|------|-----|
| Input | - | 224×224×3 | - | - | - |
| Stage 1 | Conv2d 3×3 | 112×112 | 24 | - | 2 |
| Stage 2 | Fused-MBConv ×2 | 112×112 | 24 | - | 1 |
| Stage 3 | Fused-MBConv ×2 | 56×56 | 48 | - | 2 |
| Stage 4 | Fused-MBConv ×3 | 28×28 | 64 | - | 2 |
| Stage 5 | MBConv ×4 | 14×14 | 128 | ✓ | 2 |
| Stage 6 | Conv2d 1×1 | 14×14 | 1280 | - | 1 |
| Output | Pool + FC | - | 5 | - | - |

**参数量**：1.19M | **FLOPs**：0.98G（非常轻量）

### 2.5 迁移学习策略

论文使用迁移学习解决**小样本**和**跨工况/跨组件**问题。

#### 🔹 迁移学习定义

**源域**：$D_s = \{x_s, P(x_s)\}$，任务 $T_s = \{y_s, f_s(\cdot)\}$

**目标域**：$D_t = \{x_t, Q(x_t)\}$，任务 $T_t = \{y_t, f_t(\cdot)\}$

**目标**：利用 $D_s$ 和 $T_s$ 的知识提升 $f_t(\cdot)$ 的学习效果。

#### 🔹 具体策略

```
1. 源域训练：用充足样本（4000个）训练完整的 LECA-EfficientNetV2

2. 冻结底层：固定前 5 个模块（Conv2d + 4×Fused-MBConv）
   - 底层提取通用特征（边缘、纹理等）

3. 微调高层：用少量目标域样本（250个）微调 MBConv 和全连接层
   - 高层适应特定任务的特征分布

4. 验证测试：用目标域验证集（750个）评估性能
```

**迁移实验设置（Table 5）：**

| 任务 | 组件迁移 | 工况迁移 | 源域 | 目标域 |
|-----|---------|---------|------|-------|
| T1 | Bearing → Bearing | 20Hz-0V → 30Hz-2V | ✓ | 变工况 |
| T2 | Gear → Gear | 20Hz-0V → 30Hz-2V | ✓ | 变工况 |
| T3 | Bearing → Gear | 同工况 | 跨组件 | ✓ |
| T4 | Bearing → Gear | 变工况 | 跨组件 | ✓ |

## 📈 六、Intuition 构建

### 6.1 为什么 LECA 比 SE 更好？

想象你在听一个管弦乐队演奏：

| 机制 | 类比 | 问题/优势 |
|------|------|----------|
| **SE** | 通过"总结报告"了解每个声部的重要性 | 降维 = 信息压缩，可能丢失细节 |
| **ECA** | 每个声部与相邻声部交流，了解局部重要性 | 不降维，但只用单一聚合方式 |
| **LECA** | 同时听取"整体协调"（Avg）和"精彩独奏"（Max），综合判断 | 多尺度 + 轻量 = 最佳平衡 |

### 6.2 为什么迁移学习有效？

```
底层特征（边缘、纹理）→ 通用 → 可迁移
高层特征（故障模式）→ 特定 → 需微调
```

类比：学开汽车 → 学开卡车
- 底层技能（方向盘、油门、刹车概念）可迁移
- 高层技能（换挡方式、车身尺寸感知）需要重新学习
