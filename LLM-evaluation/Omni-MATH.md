# Omni-MATH: 奥林匹克级数学基准测试详解

## 一、论文背景与动机

### 1.1 研究背景

近年来，大语言模型在数学推理方面取得了显著进展。然而，现有的数学基准测试（如GSM8K和MATH）已经逐渐饱和，例如OpenAI o1在MATH数据集上已经达到了94.8%的准确率。这表明这些基准测试已不足以真正挑战当前最先进的模型。

### 1.2 核心问题

现有基准测试存在的主要问题：
- GSM8K和MATH已接近饱和，难以区分模型能力
- 多模态融合等其他能力的耦合干扰了数学推理评估
- 缺乏专注于奥林匹克级纯文本数学推理的综合性基准

### 1.3 研究目标

Omni-MATH旨在创建一个**专用于奥林匹克级数学推理**的通用基准测试，包含4,428道竞赛级问题，涵盖33个子领域和10+个难度级别。

## 四、评估方法详解

### 4.1 三种评估方式

#### 1. GPT-4o评估
- 提供问题、参考答案和模型生成解法
- 让GPT-4o判断解法是否与参考答案一致
- 与人工标注一致性达到**98%**

#### 2. Omni-Judge（开源评估器）
```
训练集：~17,618
验证集：~2,200  
测试集：~2,200
```
- 基于LLaMA系列模型微调
- 需要提取答案、给出判断和详细理由
- 与GPT-4o一致性超过**90%**

#### 3. 规则评估（Omni-MATH-Rule）
- 使用SymPy进行数学表达式匹配
- 采用"last boxed"方法提取答案
- 适用于可自动评估的子集

## 六、技术深度解析

### 6.1 难度一致性公式（Equation 1）

定义趋势强度 $\mathcal{A}$：

$$
\mathcal{A} = \sum_{i=1}^{n-1} 
\begin{cases} 
K \cdot (x_i - x_{i+1}) & \text{if } x_{i+1} > x_i \\
\min\left(\frac{\max(x)}{n}, x_i - x_{i+1}\right) & \text{if } x_{i+1} \leq x_i 
\end{cases}
$$

#### 变量说明：
- $x_i$：第i个难度区间的模型准确率（$0 < x_i < 100$）
- $n$：难度区间数量（这里$n=4$）
- $K$：惩罚系数（这里$K=2$）
- $\max(x)$：所有$x_i$中的最大值
- $\frac{\max(x)}{n}$：单位置的最大影响限制（这里$=25$）

#### 公式理解：

**正向趋势**（$x_{i+1} > x_i$，即准确率随难度增加而上升，不合理）：
- 应用惩罚：$K \cdot (x_i - x_{i+1})$
- 这是"反趋势"情况，使用双倍惩罚

**负向趋势**（$x_{i+1} \leq x_i$，即准确率随难度增加而下降，合理）：
- 奖励：$\min\left(\frac{\max(x)}{n}, x_i - x_{i+1}\right)$
- 限制单位置最大影响为25

**值域**：当$n=4, K=2$时，$\mathcal{A} \in [-200, 75]$

#### 直觉理解：
- $\mathcal{A}$越大，准确率下降趋势越明显（难度分类越合理）
- $\mathcal{A}$越小，准确率随难度增加反而上升（难度分类有问题）

## 七、数据选择实验（Section 4.3）

### 7.1 实验设置

- 基础模型：GPT-4o
- 评估集：从Omni-MATH随机采样500题
- In-context示例：每题4个示例

### 7.2 领域相互影响（Figure 6a）

```
热力图解读（列归一化到对角线）：
- 数论 → 离散数学：正向影响（+效果）
- 离散数学 → 数论：负向影响
- 代数 → 应用数学：干扰（-效果）
- 数论 → 几何：强正向（150%域内准确率）
```

### 7.3 域树距离（Figure 6b）

定义**公共路径长度**（Common Path Length）：
- 域树上从根节点到最近公共祖先的路径长度
- 路径越长，域相似度越高

**发现**：域相似度越高，模型性能一致性越高。

### 7.4 难度影响（Figure 6c）

**结论**：示例难度越高，模型性能越好（T4略有下降）。

## 八、Best-of-N扩展局限性

### 8.1 实验设置

- 方法：Best-of-N
- 奖励模型：Qwen2.5-Math-RM-72B
- 对比：RM@8 vs RM@256

### 8.2 结果分析（Table 8）

| 指标 | Pass@1 | Pass@8 | Pass@16 | Pass@32 |
|------|--------|--------|---------|---------|
| Acc | 40.8% | 57.6% | 63% | 67.2% |
| Correct COT Proportion | 100% | 68.5% | 62.5% | 58.5% |

**问题诊断**：

1. **奖励模型监督能力不足**
   - 无法从众多候选中准确选择正确推理路径
   - 即使有正确解，也可能被错误路径覆盖

2. **策略模型搜索能力有限**
   - 采样32次后仍有33.8%问题无正确解
   - 随着采样增加，干扰项比例增加

### 8.3 推理成本对比

```
OpenAI o1: 最大4096 tokens
RM@256: 远高于4096 tokens

但 o1 仍超越 vanilla 模型 → 推理质量 > 推理数量
```

## 九、关键贡献总结

### 9.1 数据集贡献

1. **规模**：4,428道奥林匹克级问题
2. **覆盖**：33+子领域，10+难度级别
3. **质量**：严格人工标注，交叉验证准确率97.3%
4. **纯文本**：专注数学推理，避免多模态干扰

### 9.2 评估方法贡献

1. **GPT-4o评估**：与人工一致性98%
2. **Omni-Judge**：开源高效评估器，与GPT-4o一致性90%+
3. **规则评估**：提供Omni-MATH-Rule子集

### 9.3 分析洞察贡献

1. **领域特异性**：代数强，离散数学弱
2. **难度一致性**：验证难度分类合理性
3. **Best-of-N局限**：奖励模型需要更强的奥林匹克级监督能力
4. **数据泄露**：对结论影响极小
