# AWQ: Activation-aware Weight Quantization for On-Device LLM Compression and Acceleration

## 一、研究背景与动机

### 1.1 On-device LLM的重要性

文章开篇明确指出了在边缘设备上部署LLMs的紧迫需求：

**关键优势：**
- **延迟消除**：避免数据传输到云服务器造成的延迟
- **离线运行**：支持实时应用，如虚拟助手、聊天机器人、自动驾驶
- **成本降低**：减少云端基础设施的运维和扩展成本
- **隐私保护**：敏感数据保留在本地，降低数据泄露风险

**核心挑战：**
- GPT-3有175B参数，FP16格式下需要350GB存储空间
- 最新的B200 GPU仅192GB显存
- 边缘设备的硬件资源极其有限

### 1.2 现有方法的局限

**量化方法分类：**

| 方法类型 | 代表工作 | 局限性 |
|---------|---------|--------|
| QAT | Bengio等(2013), Gholami等(2021) | 训练成本高，难以扩展到大规模LLM |
| PTQ | Jacob等(2018), GPTQ (Frantar等2022) | GPTQ会过拟合校准集，破坏LLM的泛化能力 |

**GPTQ的具体问题：**
- 在重构过程中对校准集过拟合
- 在分布外领域导致特征失真（Figure 8）
- LLM是通用模型，需要保持跨领域泛化能力

## 二、AWQ方法核心原理

### 2.1 核心洞察：权重不等重要性

文章发现了一个关键观察：**LLM中的权重并不是同等重要的**

**验证实验（Table 1）：**

```
OPT-6.7B在INT3-g128量化下的WikiText PPL性能：

方法                  | 0.1%保留 | 1%保留 | 3%保留
---------------------|---------|--------|-------
RTN（基线）          | -       | 23.54  | 23.54
基于权重幅度的FP16    | 24.23   | 22.37  | 23.41
基于激活幅度的FP16    | 12.87   | 11.58  | 11.36
随机选择FP16          | 24.22   | 22.45  | 23.54
```

**关键发现：**
- 仅保留1%的权重在FP16即可将PPL从43.2降至13.0
- **基于激活分布**选择重要权重最有效
- 基于权重幅度选择几乎无效（与随机选择相当）
- 激活幅度较大的输入特征更重要，保持对应权重FP16可保留这些特征

### 2.2 数学推导：缩放保护显著权重

#### 2.2.1 量化函数定义

基本量化公式：
```
Q(w) = Δ · Round(w/Δ)
Δ = max(|w|) / (2^N - 1)
```
其中N是量化位数，Δ是量化缩放因子。

#### 2.2.2 缩放变换误差分析

考虑权重元素w，乘以s>1后，同时对输入x进行逆缩放：

变换后的计算：
```
Q(w·s)(x/s) = Δ'·Round(ws/Δ')·x·(1/s)
```

**三个关键假设：**
1. **舍入误差不变性**：RoundErr(·) ~ 0.25（均匀分布在[0,0.5]）
2. **缩放因子稳定性**：Δ' ≈ Δ（单个元素缩放通常不改变组内最大值）
3. **FP16无量化误差**：Δ和x以FP16表示，无量化误差

**误差公式：**

原始量化误差：
```
Err(Q(w)x) = Δ · RoundErr(w/Δ) · x
```

缩放后量化误差：
```
Err(Q(w·s)(x/s)) = Δ' · RoundErr(ws/Δ') · x · (1/s)
```

**相对误差比：**
```
New Error / Original Error = (Δ'/Δ) · (1/s)
```

**关键结论：**
当Δ' ≈ Δ且s > 1时，相对误差更小！

#### 2.2.3 验证实验（Table 2）

```
OPT-6.7B量化统计（1%显著通道缩放）：

s值    | Δ'≠Δ比例 | 平均Δ'/Δ | 平均(Δ'/Δ)·(1/s) | Wiki-2 PPL
-------|---------|----------|-------------------|------------
s=1    | 0%      | 1.000    | 1.000             | 23.54
s=1.25 | 2.8%    | 1.005    | 0.804             | 12.87
s=1.5  | 4.4%    | 1.013    | 0.676             | 12.48
s=2    | 8.2%    | 1.038    | 0.519             | 11.92
s=4    | 21.2%   | 1.213    | 0.303             | 12.36
```

**观察：**
- 最佳PPL在s=2时达到
- s增大会降低显著通道的误差
- 但过大会增加非显著通道的误差（Δ'增大放大非显著通道误差）
- 需要在显著和非显著通道间权衡

### 2.3 最优缩放因子搜索

#### 2.3.1 优化目标

最小化量化后输出差异：
```
s* = argmin_s L(s)
L(s) = ||Q(W·diag(s))(diag(s)^-1·X) - WX||
```

其中：
- Q：量化函数（如INT3/INT4，group size 128）
- W：原始FP16权重
- X：校准集输入特征
- s：per-(input) channel缩放因子

#### 2.3.2 搜索空间设计

由于量化函数不可微，不能直接使用反向传播。通过分析影响缩放因子选择的因素，定义搜索空间：

```
s = s_X^α
α* = argmin_α L(s_X^α)
```

**核心思想：**
- s_X：每通道激活平均幅度（activation-aware原理）
- α：单个超参数，平衡显著和非显著通道保护
- α ∈ [0,1]网格搜索（0=不缩放，1=最激进缩放）

#### 2.3.3 完整AWQ算法流程

```
Algorithm AWQ:
Input: 模型W，校准集Xcal
Output: 量化模型Q(W_quant)

1. For each layer:
   a. 使用校准集收集激活统计
   b. 计算每通道激活幅度 s_X
   c. 网格搜索α∈[0,1]优化L(s_X^α)
   d. 应用缩放：W_scaled = W · diag(s_X^α*)
   e. 应用权重裁剪减少MSE
   f. 量化：W_quant = Q(W_scaled)
   
2. 保存量化模型和缩放因子
```

**优势总结：**
1. ✅ 无需反向传播或重构
2. ✅ 最小校准集依赖（仅测量平均激活幅度）
3. ✅ 防止过拟合，保持跨领域泛化
4. ✅ 硬件友好的单精度格式

## 三、TinyChat系统实现

### 3.1 性能瓶颈分析（Figure 3）

#### 3.1.1 Context vs Generation延迟

**Llama-2-7B在RTX 4090上的实测：**
- Context阶段（200 tokens）：10ms
- Generation阶段（20 tokens）：310ms
- **Generation阶段比Context阶段慢30倍！**

#### 3.1.2 Generation阶段是Memory-bound

**Roofline分析：**
- RTX 4090：峰值165 TFLOPS，内存带宽1TB/s
- 临界算术强度：165 FLOPs/Byte
- FP16 Generation阶段算术强度≈1 FLOPs/Byte
- **结论：远低于临界值，受内存带宽限制**

#### 3.1.3 Weight Access占主导

**内存访问分解（Figure 3c）：**
- Weight access >>> Activation access
- 4-bit量化：内存访问↓4倍
- 算术强度提升至约4 FLOPs/Byte
- 理论峰值性能提升至4 TFLOPS

### 3.2 TinyChat核心优化技术

#### 3.2.1 On-the-fly权重解量化

**问题：** 硬件不支持INT4×FP16乘法

**解决方案：**
```
传统方法：
权重(INT4) → DRAM → 解量化 → FP16 → DRAM → 计算

TinyChat优化：
权重(INT4) → 融合Kernel → 解量化+计算
```
- 避免中间DRAM写入
- 同时支持MM（矩阵-矩阵）和MV（矩阵-向量）乘积

#### 3.2.2 SIMD感知权重打包（Figure 4）

**ARM NEON 128位SIMD示例：**

传统打包（32个4-bit权重）：
```
[w0, w1, w2, ..., w31]
→ 需要96次标量指令解包（3次/权重）
```

TinyChat打包：
```
[w0, w16, w1, w17, ..., w15, w31]
→ 仅需3次SIMD指令解包全部32个权重
```

**通用公式：**
- 对于2^n位SIMD寄存器
- 相邻权重索引偏移 = 1/8 × 2^n
- 每个寄存器容纳 1/8 × 2^n 个8位整数

**GPU打包策略：**
```
每8个权重打包为：w{0,2,4,6,1,3,5,7}
```
性能提升：1.2×

#### 3.2.3 Kernel融合

**Layer Normalization融合：**
```
原始：
multiply → divide → sqrt → multiply
（4个kernel调用）

TinyChat：
单kernel融合全部操作
```

**Attention层融合：**
- QKV投影融合到单kernel
- 在线位置编码计算
- KV cache预分配和更新

**性能影响：**
- 单个FP16 kernel耗时：~0.01ms（4090）
- 相当于GPU kernel launch开销
- 减少kernel调用次数直接提速

## 五、方法对比与架构解析

### 5.1 AWQ vs GPTQ核心差异

```
对比维度：

方面           | GPTQ                          | AWQ
--------------|-------------------------------|--------------------
核心思想       | 二阶信息误差补偿               | 激活感知权重缩放
依赖优化       | 重构（Hessian逆矩阵）          | 网格搜索单参数α
校准集需求     | 大（需要准确估计Hessian）      | 小（仅测激活幅度）
过拟合风险     | 高（重构可能扭曲特征）         | 低（仅用幅度统计）
跨领域泛化     | 差（Figure 8证据）             | 好（Figure 8证据）
硬件友好性     | 需要reorder技巧               | 直接可部署
计算复杂度     | 高（需要矩阵求逆）             | 低（简单统计）
实现难度       | 复杂（需要Hessian处理）        | 简单（统计+搜索）
```

### 5.2 AWQ完整架构流程

```
┌─────────────────────────────────────────────────────┐
│                    AWQ量化流程                        │
├─────────────────────────────────────────────────────┤
│                                                     │
│  原始LLM模型 (FP16)                                  │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Step 1: 激活统计收集                        │   │
│  │ - 使用小校准集（如16序列）                  │   │
│  │ - 收集每层每通道激活幅度                    │   │
│  │ - 得到激活缩放向量 s_X                     │   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Step 2: 网格搜索最优α                       │   │
│  │ - 定义搜索空间 s = s_X^α                    │   │
│  │ - α ∈ [0,1]，步长1/20                       │   │
│  │ - 优化目标：min ||Q(W·diag(s))(X·diag(1/s)) - WX│   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Step 3: 应用缩放                             │   │
│  │ - W_scaled = W · diag(s_X^α*)               │   │
│  │ - 应用权重裁剪减少MSE                        │   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Step 4: 量化                                 │   │
│  │ - W_quant = Q_group(W_scaled, bits, group) │   │
│  │ - 支持INT3/INT4，group size 128            │   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Step 5: 保存模型和元数据                    │   │
│  │ - 量化权重 (INT3/INT4)                      │   │
│  │ - 每通道缩放因子 s_X^α*                     │   │
│  │ - 每组量化缩放因子 Δ                        │   │
│  └─────────────────────────────────────────────┘   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────┐
│                  TinyChat推理引擎                      │
├─────────────────────────────────────────────────────┤
│                                                     │
│  用户输入 → Tokenization                             │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Context阶段（并行处理）                      │   │
│  │ - Attention (QKV融合, KV cache)             │   │
│  │ - LayerNorm (融合kernel)                    │   │
│  │ - FFN (量化Linear层)                        │   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ Generation阶段（自回归）                     │   │
│  │ ├─ Attention (仅新token)                     │   │
│  │ ├─ LayerNorm (融合)                          │   │
│  │ └─ FFN (量化Linear层) ← 主要优化目标         │   │
│  │    • SIMD-aware权重打包                      │   │
│  │    • On-the-fly解量化                        │   │
│  │    • Kernel融合                               │   │
│  └─────────────────────────────────────────────┘   │
│       ↓                                             │
│  ┌─────────────────────────────────────────────┐   │
│  │ 量化Linear层详细流程：                       │   │
│  │                                             │   │
│  │  输入x (FP16)                                │   │
│  │      ↓                                       │   │
│  │  逆缩放: x' = x / s_channel                  │   │
│  │      ↓                                       │   │
│  │  ┌───────────────────────────────────────┐  │   │
│  │  │ TinyChat Quantized GEMM Kernel:      │  │   │
│  │  │                                     │  │   │
│  │  │  1. 从SRAM读取打包权重:             │  │   │
│  │  │     [w0,w16,w1,w17,...,w15,w31]    │  │   │
│  │  │                                     │  │   │
│  │  │  2. SIMD解权重:                    │  │   │
│  │  │     shift → AND → scale            │  │   │

│  │  │  4. 应用组量化缩放Δ                  │  │   │
│  │  │     y = y * Δ                        │  │   │
│  │  │                                     │  │   │
│  │  │  5. 写出结果到寄存器                │  │   │
│  │  └───────────────────────────────────────┘  │   │
│  │      ↓                                       │   │
│  │  输出y (FP16)                                 │   │

│                                                     │
└─────────────────────────────────────────────────────┘
```

## 八、核心技术公式汇总

### 8.1 核心量化公式

```
1. 基本量化：
   Q(w) = Δ · Round(w/Δ)
   Δ = max(|w|) / (2^N - 1)

2. 缩放变换：
   Q(w·s)(x/s) = Δ'·Round(ws/Δ')·x·(1/s)

3. 误差公式：
   Err(Q(w)x) = Δ · RoundErr(w/Δ) · x
   Err(Q(w·s)(x/s)) = Δ' · RoundErr(ws/Δ') · x · (1/s)
   
4. 相对误差比：
   Ratio = (Δ'/Δ) · (1/s) ≈ 1/s (当Δ'≈Δ)

5. 优化目标：
   s* = argmin_s ||Q(W·diag(s))(X·diag(1/s)) - WX||
   
6. 搜索空间：
   s = s_X^α, α* = argmin_α L(s_X^α), α∈[0,1]
   
7. 算术强度：
   AI = FLOPs / Memory_Access
   
8. 理论峰值提升：
   Speedup_theoretical = (Bit_FP16 / Bit_quantized) = 4
```

### 8.2 性能关键公式

```
1. 内存带宽限制：
   Compute_bound when AI > Compute_peak / Bandwidth
   Memory_bound when AI < Compute_peak / Bandwidth
   
2. RTX 4090：
   Compute_peak = 165 TFLOPS
   Bandwidth = 1 TB/s
   Critical_AI = 165 FLOPs/Byte
   
3. FP16 LLM Generation：
   AI ≈ 1 FLOPs/Byte (Memory_bound!)
   
4. W4A16后：
   AI ≈ 4 FLOPs/Byte
   Peak ≈ 4 TFLOPS
   Speedup ≈ 4×
```
