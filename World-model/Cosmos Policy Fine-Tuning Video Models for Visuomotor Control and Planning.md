# Cosmos Policy: Fine-Tuning Video Models for Visuomotor Control and Planning

## 论文核心思想

Cosmos Policy 提出了一个**创新性想法**：将大型预训练视频生成模型（如 NVIDIA Cosmos-Predict2-2B）通过**单个阶段**的 post-training 转换为有效的机器人策略，而**不需要任何架构修改**。关键洞察是：视频模型已经学会了时空因果性、隐式物理和运动模式，这些先验知识对机器人控制极其有价值。

## 技术架构详解

### 1. 基础视频模型：Cosmos-Predict2-2B

基于 **Wan2.1 spatiotemporal VAE tokenizer** 和 **EDM denoising score matching** 训练的潜在视频扩散模型。

**核心训练公式**：
```
L(D_θ,σ) = E_{x0,c,n}[||D_θ(x0 + n; σ, c) - x0||²₂]
```

**变量解释**：
- `x0`：干净的 VAE 编码图像序列（潜在空间表示）
- `c`：文本描述，通过 T5-XXL 编码器得到
- `n ~ N(0, σ²I)`：高斯噪声，用于破坏 `x0`
- `σ`：噪声水平
- `D_θ`：Diffusion Transformer 去噪网络
- `θ`：网络参数

**噪声水平采样**：
原始视频模型使用对数正态分布：
```
ln(σ) ~ N(P_mean, P_std²)
```
其中 `P_mean = 1.39`, `P_std = 1.2`。

**Cosmos Policy 的改进**：使用**混合对数正态-均匀分布**：
- 70% 概率从原始对数正态分布采样
- 30% 概率从均匀分布 U[1.0, 85.0] 采样

这样在高噪声水平（扩散过程的开始阶段）赋予更多权重，提高动作预测精度。

**VAE 压缩细节**：
```
输入：(1+T) × H × W × 3
输出：(1+T') × H' × W' × 16
```
其中 `T' = T/4`，`H' = H/8`，`W' = W/8`。第一帧不进行时间压缩，用于条件化。

---

### 2. 问题形式化：MDP 和模仿学习

机器人操作任务建模为有限时域马尔可夫决策过程：
```
MDP = ⟨S, A, T, R, H⟩
```

**变量含义**：
- `S`：状态集合
- `A`：动作集合
- `T: S × A → Π(S)`：状态转移函数
- `R: S × A → ℝ`：奖励函数
- `H ∈ ℕ`：时间步长
- `t ∈ {1, 2, ..., H}`：离散时间索引

**奖励设计**：
```
R(s_t, a_t) = 0                    for t < H
R(s_H, a_H) ∈ [0, 1]               稀疏终端奖励
```

**策略学习**：
训练策略 `π: S → Π(A)` 通过模仿学习在专家演示数据上。

**动作块预测**：
预测 `K` 个时间步的动作序列以改善平滑度：
```
a = [a_1, a_2, ..., a_K]
```

---

### 3. 世界模型和价值函数

**世界模型**：
```
Ť: S × A → Π(S)
```
学习预测给定当前状态和动作的未来状态，近似真实环境动力学。

**价值函数**：
对于策略 `π` 在状态 `s` 的价值：
```
V^π(s) = E_{τ~π}[ Σ_{k=t}^H γ^{k-t} R(s_k, a_k) | s_t = s ]
```

在稀疏奖励设置下简化为：
```
V^π(s) = E_{τ~π}[ γ^{H-t} R(s_H, a_H) | s_t = s ]
```

**变量解释**：
- `γ`：折扣因子，将终端奖励反向传播
- `τ`：轨迹 `(s_1, a_1, s_2, a_2, ..., s_H, a_H)`
- `γ^{H-t} R(s_H, a_H)`：蒙特卡洛回报

---

### 4. 核心创新：Latent Frame Injection（潜在帧注入）

这是 Cosmos Policy 的**关键技术创新**：通过将新的模态（机器人状态、动作块、状态值）作为新的潜在帧直接注入到视频模型的潜在扩散序列中，而无需修改架构。

#### 潜在序列结构示例（双静态相机 + 手腕相机）

对于一个具有两个静态第三人称相机和一个手腕相机的机器人平台，潜在序列包含 **11 个潜在帧**：

| 索引 | 内容 | 类型 |
|------|------|------|
| 1 | 空白占位符 | 必要的 VAE 压缩要求 |
| 2 | 机器人本体感觉 s | 新模态 |
| 3 | 手腕相机图像 | 多视角 |
| 4 | 第三人称相机1 | 主相机 |
| 5 | 第三人称相机2 | 多视角 |
| 6 | 动作块 a | 新模态 |
| 7 | 未来机器人本体感觉 s' | 未来状态 |
| 8 | 未来手腕相机图像 | 未来状态 |
| 9 | 未来第三人称相机1 | 未来状态 |
| 10 | 未来第三人称相机2 | 未来状态 |
| 11 | 未来状态值 V(s') | 新模态 |

#### 序列语义表示：
```
(s, a, s', V(s'))
```
这允许从左到右的自回归解码。

#### 新模态编码过程

对于动作块（形状：`K × d_act`）：
1. **归一化**：每个动作维度归一化到 `[-1, +1]`
2. **展平**：展平为向量 `(K × d_act)`
3. **重复复制**：重复 `(H' × W' × C') / (K × d_act)` 次
4. **重塑**：重塑为 `H' × W' × C'` 体积
5. **覆盖**：覆盖对应的潜在帧

对于值函数（标量）：
简单地将标量归一化值重复填充整个潜在帧。

#### 推理时的提取

提取动作块预测：
```
a_pred = mean(action_latent_volume) / ((H' × W' × C') / (K × d_act))
```
然后反归一化到原始尺度。

提取值预测：
```
V_pred = mean(value_latent_volume)
```

### 5. 联合训练方案：策略 + 世界模型 + 价值函数

#### 批次分配策略

每个训练批次的样本按以下比例分配：

| 组成部分 | 比例 | 数据来源 | 训练目标 |
|----------|------|----------|----------|
| 策略 | 50% | 演示数据集 | `p(a, s', V(s') | s)` |
| 世界模型 | 25% | Rollout 数据集 | `p(s', V(s') | s, a)` |
| 价值函数 | 25% | Rollout 数据集 | `p(V(s') | s, a, s')` |

#### 条件化方案

通过**条件掩码**决定潜在扩散序列的哪些部分作为条件、哪些作为目标生成：

- **策略训练**：`s`（条件）→ `a, s', V(s')`（目标）
- **世界模型训练**：`s, a`（条件）→ `s', V(s')`（目标）
- **价值函数训练**：`s, a, s'`（条件）→ `V(s')`（目标）

#### 辅助学习目标

**策略辅助损失**：
不仅学习 `p(a | s)`，还联合学习 `p(a, s', V(s') | s)`。

**世界模型辅助损失**：
不仅学习 `p(s' | s, a)`，还联合学习 `p(s', V(s') | s, a)`。

这些辅助监督提供额外的梯度信号，提高性能。

---

### 6. 解码策略：并行 vs 自回归

#### 并行解码
- **优势**：速度快
- **场景**：直接策略评估（不需要规划）
- **输出**：同时生成动作、未来状态和值

#### 自回归解码
- **优势**：更高质量的预测
- **场景**：需要规划（需要精确的未来状态和值预测）
- **输出**：先预测动作 `a`，再预测未来状态 `s'`，最后预测值 `V(s')`

---

### 7. 从 Rollout 经验学习

#### 数据收集
在多样化初始条件下部署 Cosmos Policy，记录：
- 轨迹
- 结果（成功/失败或分数）

#### Fine-tuning 策略
在汇聚的 rollout 数据集上 fine-tuning 基础 checkpoint，加权分配：
- 90%：世界模型（45%）和价值函数（45%）
- 10%：策略

#### 双部署策略
1. **策略模型**：原始 Cosmos Policy checkpoint（用于动作生成）
2. **规划模型**：fine-tuned checkpoint（用于世界模型和价值函数预测）

这种确保 refined 世界模型和价值函数在策略收集的 on-policy 数据上训练。

---

### 8. 模型基规划

#### Best-of-N Sampling 算法

```
1. 从策略模型采样 N 个动作提案：{a_i} (i=1..N)
2. 对于每个 a_i：
   a. 查询世界模型 3 次得到 {s'_i,1, s'_i,2, s'_i,3}
   b. 对于每个 s'_i,j，查询价值函数 5 次得到 {V_i,j,1, ..., V_i,j,5}
3. 对于每个 a_i：
   a. 有 15 个值预测：{V_i,1,1, ..., V_i,3,5}
   b. 使用"多数均值"聚合：
      i. 确定多数预测成功或失败（阈值判断）
      ii. 在多数组内求平均值
4. 执行具有最高聚合值的动作
```

#### 多数均值聚合

当值预测呈现双峰分布或高方差时，比朴素平均更鲁棒。

```
V_aggregated = mean(V_i,k,l | majority(V_i,k,l) = majority_label)
```

#### 硬件加速
- 使用 **N 个并行 GPU** 进行最佳-N 采样
- 执行**整个动作块**（仅部分）以避免进一步增加计算成本

## 实验评估

### 1. 基准任务

#### LIBERO 模拟基准
- 单臂 Franka Emika Panda 机器人
- 四个任务套件：LIBERO-Spatial、LIBERO-Object、LIBERO-Goal、LIBERO-Long
- 每个任务套件 10 个任务 × 50 个演示 = 500 个训练演示

#### RoboCasa 模拟基准
- 24 个静态厨房操作任务
- 单臂 Franka Emika Panda 机器人
- 每个任务 50 个人类遥操作演示
- 5 个评估场景（每种任务 50 次试验 × 5 个场景 = 250 次试验/任务）
- **3 个随机种子，总共 3600 次试验**
- 两个场景包含训练中**未见过的风格**

#### 真实世界 ALOHA 机器人任务
- 双臂 ViperX 300 S 机器人
- 控制频率：25 Hz（从 50 Hz 降低）
- 输入：
  - 机器人本体感觉状态（14 个关节角度）

  - 任务描述
- 输出：50 时间步动作块（2 秒）

**四个任务**：
1. "put X on plate"（80 个演示）：根据语言指令将物体放在盘子上

## 局限性与未来方向

### 1. 推理速度
- 模型基规划的高延迟（~5秒/动作块）限制了在动态任务中的应用
- 未来工作需要加速搜索过程

### 2. 数据效率
- 有效规划需要大量rollout数据以获得演示分布外的准确预测
- 从更少的rollout中学习可提高可访问性

### 3. 搜索深度
- 当前使用单层best-of-N搜索
- 扩展世界模型预测时域和深度规划可带来更有效的搜索

## 直觉建立

### 为什么视频模型适用于机器人控制？

**时空先验**：视频模型从数百万视频中学习了：
- 时空因果性（时间上的因果关系）
- 隐式物理（物理规律）
- 运动模式（运动模式）

这与静态图像-文本模型仅学习语义概念不同。机器人控制本质上是**时序预测问题**，而不是静态理解问题。

---

### 为什么动作可以建模为潜在帧？

**分布建模能力**：视频模型擅长建模复杂、高维、多模态分布。动作本质上是：
- 高维（多个关节/坐标）
- 多模态（多种方式完成任务）
- 时序连贯（平滑的轨迹）

扩散过程可以自然地捕捉这些特性。L1回归（如某些VLA）难以建模多模态动作分布。

---

### 为什么联合学习策略、世界模型、价值函数？

**梯度共享**：
- 策略从世界模型学习状态过渡
- 世界模型从价值函数学习什么重要
- 价值函数从策略学习什么目标可实现

这种交叉监督提供了丰富的训练信号，使模型学习到更好的表示。

---

### 为什么未来状态预测对策略重要？

**隐式规划**：预测未来状态迫使策略考虑：
- "如果我做这个动作，会发生什么？"
- "这会导致成功状态吗？"

这种"想象"能力使策略在执行前进行隐式规划，减少错误动作。

---

### 为什么模型基规划优于模型自由？

**数据效率**：
- 在有限的rollout数据下，学习准确Q函数困难
- Q函数直接从 `(s,a)` 预测值，输入维度更高，容易过拟合
- V函数从 `s'` 预测值，利用学习到的环境动力学

**泛化**：
- 世界模型捕获环境动力学，可泛化到新的状态-动作对
- Q函数需要显式覆盖状态-动作空间

## 数学公式详细解析

### 1. EDM Denoising Score Matching

对于扩散模型，条件生成目标是最小化噪声预测误差：
```
L(D_θ,σ) = E_{x0,c,n}[||D_θ(x0 + n; σ, c) - x0||²₂]
```

**直觉解释**：
- 给定噪声数据 `x_σ = x0 + n`（其中 `n ~ N(0, σ²I)`）
- 网络 `D_θ` 学习预测原始干净的 `x0`
- 通过不同噪声水平 `σ` 进行训练，模型学习逐步去噪

**自适应层归一化（AdaLN）**：
```
h = LayerNorm(x)
h = h * (1 + scale(σ)) + shift(σ)
```
其中 `scale(σ)` 和 `shift(σ)` 是基于噪声水平的可学习函数。

---

### 2. 值函数的 Monte Carlo 估计

在稀疏奖励设置下，值函数估计为：
```
V(s) = γ^{H-t} · R(s_H, a_H)
```

**变量含义**：
- `t`：当前时间步
- `H`：最终时间步
- `γ`：折扣因子（通常接近1，如0.99）
- `R(s_H, a_H)`：终端奖励（成功=1，失败=0）

**举例**：
- 如果 `H=100`，`t=50`，`γ=0.99`，`R=1`：
  ```
  V(s_50) = 0.99^{50} · 1 ≈ 0.605
  ```
- 如果 `t=90`：
  ```
  V(s_90) = 0.99^{10} · 1 ≈ 0.904
  ```
  
越接近终端，价值越高（折扣衰减越少）。

---

### 3. Batch 损失计算

对于策略训练（50%批次）：
```
L_policy = ||a_pred - a_gt||₁ + λ_s'||s'_pred - s'_gt||₁ + λ_V||V_pred - V_gt||₁
```

对于世界模型训练（25%批次）：
```
L_world = ||s'_pred - s'_gt||₁ + λ_V||V_pred - V_gt||₁
```

对于价值函数训练（25%批次）：
```
L_value = ||V_pred - V_gt||₁
```

其中 `||·||₁` 是L1损失，`λ_s'` 和 `λ_V` 是损失权重。

```
def evaluate_direct_policy(s):
    # 1. Tokenize observations
    latent_frame_1 = vae.encode(wrist_image)
    latent_frame_2 = vae.encode(camera1_image)
    latent_frame_3 = vae.encode(camera2_image)

    latent_sequence = [placeholder, latent_frame_4, 
                       latent_frame_1, latent_frame_2, 
                       latent_frame_3, noise, noise, noise]

    # 3. Denoise (parallel)
    for step in range(num_denoising_steps):
        sigma = get_noise_level(step, num_denoising_steps)
        denoised = diffusion_model(
            corrupted=latent_sequence, 
            sigma=sigma, 
            conditioning=[placeholder, latent_frame_4, 
                         latent_frame_1, latent_frame_2, 
                         latent_frame_3]
        )
        latent_sequence = denoised

    # 4. Extract action
    action_latent = latent_sequence[5]  # Action chunk
    action = extract_action(action_latent)
    
    return action
```

```
# 模型基规划（Best-of-N）
def evaluate_with_planning(s, N=8):
    best_action = None
    best_value = -inf

    # Sample N action proposals
    for i in range(N):
        a_i = sample_action_from_policy(s)

    # Evaluate each action
    for a_i in actions:
        value_predictions = []
        for world_ensemble in range(3):
            s_prime = predict_future_state(s, a_i)
            for value_ensemble in range(5):
                V = predict_value(s_prime)
                value_predictions.append(V)

        # Majority mean aggregation
        predictions = np.array(value_predictions)
        majority_success = predictions > threshold
        if majority_success.sum() > len(predictions)/2:
            aggregated_value = predictions[majority_success].mean()
        else:
            aggregated_value = predictions[~majority_success].mean()

        if aggregated_value > best_value:
            best_value = aggregated_value
            best_action = a_i

    return best_action
```

## 计算复杂度分析

### 训练复杂度

对于C个通道的输入视频或图像序列，假设时间步数为 `T+1`，空间维度为 `H' × W'`：

- VAE 编码：`O(T × H × W × 3)`（原始图像）
- 潜在尺寸：`(T/4 + 1) × (H/8) × (W/8) × 16`
- Diffusion Transformer 计算复杂度：`O((T/4 + 1) × (H/8) × (W/8) × 16 × d_model²)`（粗略估计）

对于 Cosmos-Predict2-2B（20亿参数）：
- 每步梯度更新内存需求：~40GB（混合精度）
- 64 × H100 训练：需要分布式数据并行

---

### 推理复杂度

直接策略评估（K个去噪步骤）：
- 时间：`O(K × T × H × W × d_model²)`（粗略估计）
- 实测：0.61秒（5步）/ 0.95秒（10步）单H100

模型基规划（N个候选，M世界模型查询，L价值查询）：
```
Time = N × K_action × T_action + N × M × K_world × T_world + N × M × L × K_value × T_value
```

对于 N=8, M=3, L=5：
- 实测：~4.9秒（8个并行H100）

## 总结

Cosmos Policy 通过创新性的**Latent Frame Injection**技术，成功地将预训练视频模型适应为机器人策略，无需架构修改。其核心优势在于：

1. **单阶段训练**：简化复杂度
2. **统一架构**：策略 + 世界模型 + 价值函数在一个模型中
3. **强先验**：利用视频模型的时空物理知识
4. **数据高效**：在RoboCasa中仅用50个演示达到SOTA
5. **多模态处理**：扩散过程自然处理多模态动作分布
6. **规划能力**：可扩展到模型基规划

在 LIBERO（98.5%）和 RoboCasa（67.1%）上建立了新的SOTA，并在真实世界 ALOHA 机器人任务中取得93.6%平均得分。
