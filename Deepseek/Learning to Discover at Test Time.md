## **一、核心思想与动机**

### **1.1 问题背景**

传统测试时扩展（test-time scaling）方法如**AlphaEvolve**使用**冻结的LLM**进行搜索：
- LLM生成多个候选解
- 使用手工构造的和领域特定的启发式方法存储过去的尝试
- 用这些尝试生成新的prompts

**关键限制：** LLM本身不会改善，就像一个永远无法内化新想法的学生。
### **1.2 核心洞察**

论文提出在测试时对LLM进行**强化学习（RL）**，使LLM能够针对特定的测试问题继续训练。这种学习有两种特殊的性质：
1. **目标是产生一个最佳解**，而不是在多个问题上平均表现良好
2. **目标是解决这个具体问题**，而不是泛化到其他问题

---

## **二、形式化定义（§2）**

### **2.1 发现问题作为MDP**

每个科学问题被形式化为一个**马尔可夫决策过程（MDP）**：

```
⟨S, A, T, R, d⟩
```

| 符号 | 含义 | 示例 |
|------|------|------|
| **d** | 问题描述 | 优化任务的自然语言描述 |
| **s** | 状态 | 候选解（如kernel实现、数学构造） |
| **a** | 动作 | thinking tokens + Python/Triton代码 |
| **T(s,a) → s'** | 转移函数 | 解析代码得到新的候选解 |
| **R(s) ∈ ℝ** | 奖励函数 | 连续奖励（如运行时间的倒数） |

### **2.2 发现的定义**

**定义（Discovery）：** 一个发现事件是指找到一个状态s，使得
```
R(s) > R(s_sota)
```
其中：
- `s_sota` = 当前已知最好解
- `R(s_sota)` = 当前已知最好奖励

## **三、核心方法：TTT-Discover（§3）**

### **3.1 Algorithm 1 伪代码**

```
Algorithm 1: Test-Time Training to Discover (TTT-Discover)

Input: 问题描述d，初始策略π_θ0
Output: 发现的最优解 s_i*

1: R, T = get_env(d)                    // 获取环境的奖励和转移函数
2: ℋ_0 = {(<empty>, R(<empty>), {})}   // 初始化缓冲区
3: 
4: for i = 0, 1, ..., N-1 do
5:     s_i, c_i ∼ reuse(ℋ_i)           // 从缓冲区采样初始状态和上下文
6:     a_i ∼ π_θi(·|d, s_i, c_i)       // 从策略采样动作
7:     s'_i = T(a_i)                    // 转移到下一状态
8:     r_i = R(s'_i)                    // 评估奖励
9:     ℋ_{i+1} = ℋ_i ∪ {(s_i, a_i, s'_i, r_i)}  // 添加到缓冲区
10:    θ_{i+1} = train(θ_i, (s_i, a_i, r_i))    // 更新模型权重
11: end for
12: return s_i*, where i* = arg max r_i
```

---

### **3.2 关键组件1：熵目标（Entropic Objective）**

#### **3.2.1 数学公式**

**标准RL目标：** 优化期望奖励
```
J(θ) = E_{a∼π_θ}[R(s,a)]
```

**熵目标（§3.2）：** 最大化期望的指数加权奖励的对数
```
J_β(θ) = E_{s∼reuse(ℋ)}[log E_{a∼π_θ(·|s)}[e^{β(s)·R(s,a)}]]
```

#### **3.2.2 梯度公式**

目标函数的梯度为：
```
∇_θ J_β(θ) = E_{s∼reuse(ℋ), a∼π_θ(·|s)}[w_β(s)(a) · ∇_θ log π_θ(a|s)]
```

其中**重要性权重**为：
```
w_β(s)(a) = e^{β(s)·R(s,a)} / E_{π_θ(·|s)}[e^{β(s)·R(s,a)}]
```

#### **3.2.3 变量解释**

| 变量 | 含义 | 作用 |
|------|------|------|
| **θ** | 模型权重 | 需要优化的参数 |
| **π_θ** | 策略函数 | 给定状态生成动作的概率分布 |
| **β(s)** | 温度参数（每状态自适应） | 控制探索-利用权衡 |
| **R(s,a)** | 奖励 | 从状态s执行动作a后的奖励 |
| **w_β(s)(a)** | 重要性权重 | 对高奖励动作赋予更高权重 |

#### **3.2.4 自适应β(s)（附录A.1）**

**问题：** 固定β值导致：
- 早期训练：β过大导致不稳定
- 后期训练：β过小导致优势消失

**解决方案：** 通过KL约束自适应设置β(s)
```
KL(q_β(s)(·|s) || π_θ(·|s)) = γ
```

其中：
```
q_β(τ|s) = π_θ(τ|s) · exp(β·r(τ;s)) / E_{π_θ}[exp(β·r(τ;s))]
```

**实用细节：**
- `γ = ln(2)` （固定超参数）
- 通过二分搜索求解β(s)
- 使用留一（LOO）归一化避免数值不稳定

### **3.3 关键组件2：PUCT状态重用（§3.2）**

#### **3.3.1 PUCT评分公式**

```
score(s) = Q(s) + c · scale · P(s) · √{1+T} / (1+n(s))
```

#### **3.3.2 变量解释**

| 变量 | 含义 | 作用 |
|------|------|------|
| **Q(s)** | 从状态s出发的最大可达奖励 | 利用项 |
| **P(s)** | 状态s的先验概率（基于reward排序） | 探索优先级 |
| **n(s)** | 状态s及其后代被扩展的次数 | 探索惩罚 |
| **T** | 总扩展次数 | 全局探索进度 |
| **c** | 探索系数（c=1.0） | 控制-利用平衡 |
| **scale** | 奖励范围（R_max - R_min） | 归一化 |

#### **3.3.3 Q(s)的特殊设计**

与传统AlphaZero使用**均值**不同，TTT-Discover使用**最大值**：
```
Q(s) = { m(s)      if n(s) > 0
       { R(s)      if n(s) = 0
```

其中 `m(s)` 是状态s的所有子状态中的最大奖励。

**直观理解：** 我们关心从某个状态出发能否达到最佳结果，而不是平均结果。

#### **3.3.4 先验P(s)的设计**

```
P(s) = (|ℋ_t| - rank(s)) / Σ_{s'∈ℋ_t}(|ℋ_t| - rank(s'))
```

其中 `rank(s)` 是状态按reward降序排列的排名（rank 0是最好状态）。

**直观理解：** 高reward状态更可能产生高reward子状态。

## **九、技术细节深度解析**

### **9.1 熵目标的理论直觉**

#### **9.1.1 从期望奖励到熵目标**

标准RL：
```
J(θ) = Σ_a π_θ(a) · R(a)
```

熵目标：
```
J_β(θ) = log Σ_a π_θ(a) · e^{β·R(a)}
```

**关键洞察：** 当β → ∞时，
```
e^{β·R(a)} / Σ_{a'} e^{β·R(a')} → 1 (对于最优a)
                           → 0 (对于其他a)
```

因此，熵目标趋向于**最大化最优动作的概率**。

#### **9.1.2 与Risk-Sensitive RL的联系**

熵目标是**CRRA风险敏感目标**的一种形式：
```
max E[e^{β·R}]
```

在RL中称为**指数效用**或**风险敏感RL**。

### **9.2 PUCT与AlphaZero PUCT的对比**

| 特征 | AlphaZero PUCT | TTT-Discover PUCT |
|------|----------------|-------------------|
| Q(s,a) | 均值 | 最大值 |
| P(s,a) | 学习的动作先验 | 基于reward排序 |
| 计数传播 | 仅边 | 所有祖先 |
| 探索惩罚 | Virtual Loss | 谱系阻断 |

**最大vs均值的哲学差异：**
- AlphaZero：目标是找到平均胜率最高的策略
- TTT-Discover：目标是找到一条通往最高奖励的路径

### **9.3 自适应β(s)的数学推导**

#### **9.3.1 倾斜分布（Tilted Distribution）**

定义：
```
q_β(τ|s) = π_θ(τ|s) · e^{β·r(τ;s)} / Z_β(s)
```

其中 `Z_β(s) = Σ_{τ} π_θ(τ|s) · e^{β·r(τ;s)}` 是配分函数。

#### **9.3.2 KL约束求解**

目标：
```
min_{β≥0} | KL(q_β(·|s) || π_θ(·|s)) - γ |
```

在离散样本上：
```
KL(q_β || u) = Σ_n q_β(n) · log(N · q_β(n))
```

其中 q_β(n) = e^{β·r_n} / Σ_m e^{β·r_m}

**数值稳定性技巧：**
使用 `r' = r - r_max` 避免大数：
```
q_β(n) = e^{β·(r_n - r_max)} / Σ_m e^{β·(r_m - r_max)}
```

| Prompt工程 | **权重更新** |

### **11.3 熵目标的工作原理**

想象一个简化场景：

假设从状态s出发有3个可选动作：
```
a₁: R = 10（最佳）
a₂: R = 8
a₃: R = 1
```

**标准RL梯度：**
```
优势A(a₁) ≈ 4.7（相对于均值6.3）
优势A(a₂) ≈ 1.7
优势A(a₃) ≈ -5.3
```

**熵目标（β=1）：**
```
w_β(a₁) ≈ 0.55
w_β(a₂) ≈ 0.25
w_β(a₃) ≈ 0.02

优势A(a₁) ≈ -0.45（相对于均值为1）
优势A(a₂) ≈ -0.75
优势A(a₃) ≈ -0.98
```

**关键区别：** 熵目标的**相对差异**更大（-0.45 vs -0.98 vs 4.7 vs -5.3），使得梯度对最优动作更敏感。

## **十二、关键公式总结**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              核心公式库                                         │
└─────────────────────────────────────────────────────────────────────────────┘

1. 熵目标：
   J_β(θ) = E_s[log E_a[exp(β(s)·R(s,a))]]

2. 梯度：
   ∇_θ J_β(θ) = E_{s,a}[w_β(s)(a) · ∇_θ log π_θ(a|s)]

3. 重要性权重：
   w_β(s)(a) = exp(β(s)·R(s,a)) / E_{π_θ}[exp(β(s)·R(s,a))]

4. 带KL惩罚的优势：
   A(a;s) = w_β(s)(a) - 1 - λ·log(π_θ(a|s)/π_θ₀(a|s))

5. PUCT评分：
   score(s) = Q(s) + c·scale·P(s)·√{1+T}/(1+n(s))

6. 自适应β（KL约束）：
   KL(q_β(s)(·|s) || π_θ(·|s)) = γ

7. 倾斜分布：
   q_β(τ|s) = π_θ(τ|s)·exp(β·r(τ;s)) / E_{π_θ}[exp(β·r(τ;s))]
```
