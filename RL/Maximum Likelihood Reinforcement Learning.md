# Maximum Likelihood Reinforcement Learning 详细技术讲解

## 1. 核心问题动机

**论文提出的关键洞察**：在采样-based设置（如导航、代码生成、数学问题求解）中，RL实际上只是对最大似然目标的一阶近似。

### 为什么RL不优化正确的目标？

考虑一个输入x，模型参数θ，令：
- **pθ(x)**：模型对输入x产生正确输出的概率
- **y*(x)**：输入x的正确答案
- **I{·}**：指示函数

论文对比了两个梯度的表达式：

**RL梯度**：
```
∇θJRL = E[∇θpθ(x)]
```

**ML梯度**：
```
∇θJML = E[∇θlog pθ(x)] = E[∇θpθ(x) / pθ(x)]
```

**关键区别**：ML目标引入了**逆概率重加权** (1/pθ(x))，这意味着对于困难、低成功率的输入，ML会给予更大的梯度权重，而RL则不会。

## 2. 理论框架：Maclaurin展开

### 2.1 Pass Rate定义

```
p_passθ(x) := pθ(y*(x)|x) = E_{y∼pθ(·|x)}[I{y = y*(x)}]
```

**Pass@k**：k次独立采样中至少一次正确的概率：
```
pass@k(x) := P(∃i∈[k] s.t. yi = y*(x))
```

**Fail@k**：所有k次采样都失败的概率：
```
fail@k(x) := 1 - pass@k(x)
```

### 2.2 ML目标的Maclaurin展开

**核心公式（公式4）**：
```
JML(x) = log p = -Σ_{k=1}^{∞} (1-p)^k / k = -Σ_{k=1}^{∞} fail@k(x) / k
```

这是log(1-(1-p))的Maclaurin展开，其中p = p_passθ(x)。

**梯度形式（公式5）**：
```
∇θJML(x) = Σ_{k=1}^{∞} (1/k) · ∇θpass@k(x)
```

**重要洞察**：
- ML优化一个pass@k梯度的**无限调和级数**
- 高阶项编码了稀有成功事件
- 当p很小时，这些高阶项至关重要

### 2.3 MaxRL目标函数

**截断Maclaurin展开（公式6）**：
```
J^{(T)}_MaxRL(x) := -Σ_{k=1}^{T} (1-p)^k / k
```

**截断梯度（公式7）**：
```
∇θJ^{(T)}_MaxRL(x) = Σ_{k=1}^{T} (1/k) · ∇θpass@k(x)
```

**关键性质**：
- T=1：恢复标准RL
- T→∞：恢复最大似然
- 中间T值：插值于RL和ML之间

**截断水平T的控制作用**：
```
MaxRL提供了一个通过增加采样计算来更高精度近似ML目标的原则性框架
```

## 3. 核心理论结果

### 3.1 定理1：ML梯度的条件形式

**陈述**：ML梯度可以表示为成功条件分布下的期望：
```
∇θJML(x) = E[∇θlog mθ(z|x) | f(z) = y*(x)]
```

**证明思路**（附录B）：
1. 从REINFORCE身份开始：∇θp_passθ(x) = E[I{f(z)=y*(x)} ∇θlog mθ(z|x)]
2. 应用链式法则：∇θlog p_passθ(x) = ∇θp_passθ(x) / p_passθ(x)
3. 利用条件期望定义：E[X|A] = E[X·I{A}] / P(A)
4. 识别A = {z: f(z) = y*(x)}，P(A) = p_passθ(x)

**直观解释**：ML梯度是**仅从成功轨迹**中平均得到的梯度。

### 3.2 定理2：估计器-目标等价性

**陈述**：估计器ĝ_N(x)是MaxRL梯度的无偏估计，T=N：
```
E[ĝ_N(x)] = ∇θJ^{(N)}_MaxRL(x)
```

**估计器定义（公式9）**：
```
ĝ_N(x) = { (1/K) Σ_{i=1}^{N} r_i S_i,  K ≥ 1
          0,                        K = 0 }
```

其中：
- r_i = I{f(z_i) = y*(x)}：奖励（成功=1，失败=0）
- S_i = ∇θlog mθ(z_i|x)：score函数
- K = Σ_{i=1}^{N} r_i：成功样本数量

**证明思路**（附录B）：
1. 当K≥1时，成功样本是成功条件分布的i.i.d.样本
2. 由定理1：E[ĝ_N(x)|K≥1] = ∇θlog p_passθ(x)
3. 当K=0时，ĝ_N(x) = 0
4. 利用全期望公式：
   ```
   E[ĝ_N(x)] = ∇θlog p_passθ(x) · P(K≥1)
             = ∇θlog p_passθ(x) · pass@N(x)
   ```
5. 利用pass@k(x) = 1 - (1-p)^k展开得到截断MaxRL梯度

**重要性质**：
```
增加采样次数N会得到更精确的ML梯度近似
```

### 3.3 与REINFORCE的对比

**Table 1对比**：

| 方法 | 估计器 | 无偏估计目标 |
|------|--------|-------------|
| REINFORCE | (1/N) Σ_{i=1}^{N} r_i S_i | ∇θpass@1(x) |
| MaxRL | (1/K) Σ_{i=1}^{N} r_i S_i | Σ_{k=1}^{N} (1/k)∇θpass@k(x) |

**关键差异**：
- REINFORCE除以总样本数N
- MaxRL除以成功样本数K

**增加采样的效果**：
- REINFORCE：仅降低固定目标的方差
- MaxRL：提高对ML的近似阶数

## 4. 方差降减：控制变量

### 4.1 无偏控制变量

定义无条件平均score作为控制变量：
```
V_N := (1/N) Σ_{i=1}^{N} ∇θlog mθ(z_i|x)
```

满足E[V_N] = 0。

### 4.2 方差降减估计器（公式10）

```
ê_N(x) = (1/K) Σ_{i=1}^{N} r_i S_i - (1/N) Σ_{i=1}^{N} S_i
       = Σ_{i=1}^{N} (r_i/K - 1/N) S_i
```

当K=0时，第一项为零。

## 5. 统一的权重函数视角

### 5.1 通用形式

所有目标的梯度可以写成：
```
∇θJ = E_{x∼ρ}[w(pθ(x)) ∇θpθ(x)]
```

其中pθ(x) = p_passθ(x)，w(p)是仅依赖于pass率的标量权重。

### 5.2 各方法的权重函数

**Table 2总结**：

| 方法 | w(p) |
|------|------|
| RL | 1 |
| GRPO | 1/√(p(1-p)) |
| MaxRL(T) | [1-(1-p)^T]/p |
| ML | 1/p |

**Figure 1可视化**：展示了不同方法的权重函数随pass率变化的曲线。

**关键洞察**：
- 权重函数w(p)决定学习信号如何分配给不同难度的输入
- **目标间的区别在于它们对困难、低pass率输入的强调程度**
- 当T增加时，MaxRL在低pass率下接近ML权重

### 5.3 GRPO的重新解释

虽然GRPO启发式地使用经验标准差的Z归一化，这种归一化产生了与REINFORCE**根本不同**的群体级目标。

**GRPO权重函数**：当p很小时，w(p) ≈ 1/√p，位于经典RL和ML之间。

**GRPO的异常行为**：
- 对于足够大的pass率，GRPO权重函数反转
- 当p→1时，权重增加
- 不像基于似然的目标

这意味着GRPO在存在非常容易的输入时，会给予这些输入更大权重，与其他方法形成对比。

**设置**：ImageNet上的ResNet-50分类

**Figure 2结果**：
- **REINFORCE**：即使使用非常高的采样预算，也未能实现有意义的改进
- **精确ML训练**：产生稳定的增益
- **MaxRL**：在相同样本和稀疏成功轨迹下，通过似然启发的重加权更有效地利用有限的学习信号
- 随着计算增加（更高的rollout数），MaxRL持续改进并紧密跟踪精确ML

**Takeaway 1**：
```
当存在直接最大似然优化时，MaxRL随采样计算增加而收敛到它
```

## 7. 算法实现

### Algorithm 1：MaxRL的On-Policy实现

```python
# Require: 输入批次B, rollout数量N, 潜在策略mθ(·|·)
for each input x ∈ B do:
    # 采样N个潜在轨迹
    Sample z1, ..., zN ∼ mθ(·|x)
    
    for j = 1 to N do:
        # 计算奖励
        rj ← I{f(zj) = y*(x)}
        # 计算score函数
        Sj ← ∇θlog mθ(zj|x)
    
    # 计算基线（每任务平均奖励）
    br(x) ← (1/N) Σ_{j=1}^{N} rj
    
    # 计算梯度（MaxRL修改的关键行）
    bg(x) ← { (1/N) · (1/br(x)) Σ_{j=1}^{N} (rj - br(x)) · Sj,  br(x) > 0
               0,                                        otherwise }

# 批次平均梯度
bg ← (1/|B|) Σ_{x∈B} bg(x)
return bg
```

**关键修改**：
- 优势被**每任务平均奖励**归一化
- 不像RLOO那样未归一化
- 不像GRPO那样被奖励标准差归一化

## 8. 相关工作

### 8.1 RL训练导致分布锐化

最近的发现：
- 某些推理技能（如反思）已经存在于预训练模型中
- 好的推理行为对RLVR成功至关重要
- **RLVR通过减少pass@k降低了模型多样性**

**本文的发现**：
- 这归因于RL目标本身
- 优化期望奖励倾向于**边缘化来自困难提示的学习信号**
- 这导致分布锐化

### 8.2 学习解决困难问题

由于RLVR的覆盖率收缩，大量注意力被集中在缓解pass@k崩溃的新RL算法上：
- 训练期间直接优化pass@k
- 在RL中采用探索奖励

**本文的贡献**：
- Pass@k优化目标是我们目标的特例
- 它优化pass@k目标的无限调和级数

### 8.3 紧密相关的工作

**Xiong et al. (2025b)**：
- 也考虑RL中pass率的非线性函数
- 关注自适应rollout预算分配和采样策略
- 将非线性权重选择视为算法设计空间的一部分

**Davis and Recht (2025)**：
- 提供理论论证，表征特定二进制奖励RL算法在渐近极限下接近的群体级目标
- 表明某些程序诱导pass率的log式加权

**本文的互补问题**：
- 有限采样如何定义显式的群体级目标
- 在每次有限rollout计数建立精确的估计器-目标等价性

## 9. 总结和未来方向

### 9.1 核心贡献

1. **形式化**：将基于正确性的强化学习框架化为潜在生成的最大似然问题
2. **理论推导**：通过Maclaurin展开引入计算索引的目标族
3. **简单估计器**：导出一个简单的on-policy估计器，其预期梯度精确匹配计算索引的似然目标近似
4. **实证验证**：在所有测试的模型和任务上，MaxRL Pareto主导现有方法

### 9.2 关键洞察

```
一些归因于基础模型强化学习的限制源于目标选择，而非优化或采样
```

### 9.3 当前限制

- 假设二元奖励设置
- 不直接扩展到连续或任意值奖励
- 可考虑优化ML之外的目标

### 9.4 未来方向

- 将MaxRL推广到连续奖励
- 多轮强化学习
- PPO风格的off-policy设置

## 直觉建立

这篇论文的核心直觉可以总结为：

1. **RL不是最优的**：在采样-based设置中，RL只优化ML目标的一阶近似
2. **计算→精度**：通过增加采样计算，我们可以逐步逼近ML目标
3. **重加权关键**：ML对困难样本给予更大权重（1/p重加权），这导致更好的扩展性
4. **简单但强大**：MaxRL的实现只需要对标准RL算法做一行修改（归一化方式）
5. **实际收益**：在完美验证器下，可达到高达20×的测试时间效率增益

这个框架为我们理解为什么RL训练可能导致分布锐化（pass@k退化）提供了理论基础，并提供了一个原则性的解决方案。
