# 《Correlated Proxies: A New Definition and Improved Mitigation for Reward Hacking》深度解析

## 一、论文背景与核心问题

这篇文章来自 **UC Berkeley**，发表于 **arXiv**，主要解决了 **Reinforcement Learning** 领域的一个核心问题：**Reward Hacking**（奖励黑客）。

### 1.1 问题的直觉理解

在真实世界中，我们很难精确定义复杂的目标。因此，我们经常使用 **proxy reward function**（代理奖励函数）来近似真实的奖励。然而，**optimizing** 这些代理奖励函数常常会导致 **reward hacking**：优化后的奖励函数不再是一个好的代理，最终的策略在未指定的真实奖励上表现很差。

### 1.2 具体案例

论文给出了多个现实世界的例子：
- **Traffic control**：使用平均车速作为代理奖励来优化交通流量
- **Pandemic mitigation**：使用感染率作为代理奖励来优化疫情控制
- **Blood glucose regulation**：使用胰岛素成本作为代理奖励来优化血糖控制
- **RLHF**：使用学习的奖励模型来优化 LLM 的输出

## 二、核心概念定义

### 2.1 基础符号定义

考虑无限时间尺度的 **Markov Decision Process (MDP)**：

- **State space**: 𝒮
- **Action space**: 𝒜
- **Initial state distribution**: μ₀(s)
- **Transition probability**: p(sₜ₊₁ | sₜ, aₜ)
- **Discount factor**: γ ∈ [0,1)
- **Policy**: π(a|s) 映射状态到动作分布

**Return（回报）**的定义：
```
J(π,R) = (1-γ) E_π[Σᵗ₌₀^∞ γᵗ R(sₜ,aₜ)]
```
其中 **Eₚₗ** 表示在策略 **π** 下运行的分布的期望。

**State-Action Occupancy Measure（状态-动作占据度量）**：
```
μ_π(s,a) = (1-γ) E_π[Σᵗ₌₀^∞ γᵗ 𝟙{sₜ = s ∧ aₜ = a}]
```
这个度量表示在状态 **s** 下采取动作 **a** 的期望折扣访问次数。

### 2.2 Correlated Proxy Reward 的定义

**Definition 4.1 (Correlated proxy reward)**：

一个 **r-correlated** 代理奖励 **R̃** 相对于参考策略 **π_ref** 定义为：在从参考策略采样的状态-动作对上，代理奖励与真实奖励的相关性为 **r > 0**：

```
E_{μ_{π_ref}}[(R̃(s,a) - J(π_ref,R̃))/σ_{R̃} · (R(s,a) - J(π_ref,R))/σ_R] = r
```

其中：
- **σ_R̃²** = E_{μ_{π_ref}}[(R̃(s,a) - J(π_ref,R̃))²]
- **σ_R²** = E_{μ_{π_ref}}[(R(s,a) - J(π_ref,R))²]

**变量解释**：
- **R(s,a)**：真实奖励函数
- **R̃(s,a)**：代理奖励函数
- **J(π,R)**：策略 **π** 在奖励函数 **R** 下的回报
- **μ_{π_ref}**：参考策略的占据度量
- **σ_R**, **σ_R̃**：在参考策略下的标准差
- **r**：相关性系数，范围在 [-1, 1]

### 2.3 Reward Hacking 的正式定义

**Definition 4.2 (Reward hacking)**：

假设 **R̃** 是相对于 **π_ref** 的 **r-correlated** 代理奖励。当优化 **R̃** 得到的策略 **π** 的真实奖励低于参考策略的真实奖励时，即发生 **reward hacking**：

```
J(π,R) < J(π_ref,R)
```

如果优化 **R̃** 的最优策略表现出 **reward hacking**，那么我们说 **R̃** 是一个 **hackable correlated proxy**：

```
J(π,R) < J(π_ref,R) for some π ∈ argmax_π J(π,R̃)
```

## 三、理论框架与核心定理

### 3.1 Theorem 5.1：正则化防止 Reward Hacking

这是论文的核心理论结果：

**定理陈述**：

假设 **R̃** 是真实奖励 **R** 的 **r-correlated-proxy**，令 **σ_R̃** 和 **σ_R** 定义如前。对于任何满足 **μ_π ≪ μ_{π_ref}** 的策略 **π**（即 **μ_{π_ref}(s,a) = 0 ⇒ μ_π(s,a) = 0**），有：

```
[J(π,R) - J(π_ref,R)]/σ_R ≥ (1/r)[(J(π,R̃) - J(π_ref,R̃))/σ_{R̃} - √{(1-r²) χ²(μ_π ‖ μ_{π_ref})}]
```

其中 **χ²(μ_π ‖ μ_{π_ref})** 是 **χ²** 散度：

```
χ²(μ_π ‖ μ_{π_ref}) = E_{μ_π}[μ_π(s,a)/μ_{π_ref}(s,a) - 1]
```

**公式深度解析**：

这个不等式给出了策略 **π** 相对于参考策略 **π_ref** 在真实奖励上的改进下界。右边由两项组成：
1. 第一项衡量策略 **π** 在代理奖励上的归一化改进
2. 第二项惩罚策略 **π** 的占据度量与参考策略的偏离

**直觉理解**：
- 如果只优化第一项（代理奖励），通常会导致 **reward hacking**
- 通过同时优化高代理奖励和保持与参考策略的接近，策略 **π** 可以获得高真实奖励

### 3.2 正则化目标函数

通过缩放并移除常数项，我们得到以下正则化策略优化目标：

```
maximize J(π,R̃) - λ√{χ²(μ_π ‖ μ_{π_ref})}
where λ = σ_{R̃}√{1-r²}
```

**关键洞察**：
- 正则化强度随着相关性 **r** 的减弱而增加
- 相关性越高，正则化强度 √{1-r²} 越低

## 四、为什么 Occupancy Measure 正则化优于 Action Distribution 正则化

### 4.1 当前 RLHF 的方法

当前的 **RLHF** 在策略的动作分布之间应用 **KL divergence** 惩罚：

```
maximize J(π,R̃) - λ(1-γ)E_π[Σᵗ₌₀^∞ γᵗ D_KL(π(·|sₜ) ‖ μ_{π_ref}(·|sₜ))]
```

这是对策略 **π** 和 **π_ref** 的动作分布的期望 **KL divergence** 的惩罚。

### 4.2 AD vs OM 的关键差异

**Action Distribution (AD)** 正则化与 **Occupancy Measure (OM)** 正则化的关键区别：

#### 4.2.1 在 RLHF 中的等价性

在 **RLHF** 中，由于 **γ = 0**（本质上是 **contextual bandit** 问题），**OM** 和 **AD** 散度是等价的。

**Lemma A.6**：

对于任何 **f-divergence**，在 **contextual bandit** 中：

```
D_f(μ_π ‖ μ_π′) = E_{s∼μ₀(·)}[D_f(π(·|s) ‖ π′(·|s))]
```

#### 4.2.2 在长期环境中的差异

然而，在其他情况下，**AD** 正则化可能不足以防止 **reward hacking** 行为。这是因为：

1. **累积效应**：在长时间尺度的环境中，单个状态下动作分布的微小变化可能导致到达不可期望状态的概率显著增加
2. **状态可达性**：**AD** 正则化只关注当前状态的动作分布，不考虑这些动作导致的未来状态分布

**Theorem A.5** 显示：
- 几乎任何形式的 **AD** 正则化都无法保证在真实奖励上相对于参考策略的改进
- 存在反例，其中策略增加正则化目标但降低真实奖励

### 4.3 实际案例分析：Glucose Monitoring Environment

**Figure 3** 的例子说明了这一点：
- 策略 **π** 在 **AD** 上与 **π_ref** 接近：只注射稍少的胰岛素
- 然而，**π** 对代理奖励的优化导致其占据度量显著不同，典型的血糖水平远超健康范围
- 因此，正则化 **AD** 不足以防止 **reward hacking**
- **OM** 正则化通过直接防止血糖水平的分布与参考策略过于不同来避免这个问题

## 五、为什么 χ² Divergence 优于 KL Divergence

### 5.1 散度的比较

#### 5.1.1 KL Divergence vs χ² Divergence

两种散度都可以写成占据度量对数比的期望：

```
D_KL(μ_π ‖ μ_{π_ref}) = E_{μ_π}[d(s,a) + e^{-d(s,a)}]
χ²(μ_π ‖ μ_{π_ref}) = E_{μ_π}[e^{d(s,a)} + e^{-d(s,a)}]
where d(s,a) = log(μ_π(s,a)/μ_{π_ref}(s,a))
```

**变量解释**：
- **d(s,a)**：对数比，表示优化策略在状态-动作对 **(s,a)** 上的相对可能性
- 当 **d(s,a)** 增加时，优化策略访问不太可能在参考策略下访问的状态-动作对

#### 5.1.2 惩罚函数的形状

**KL divergence**：
- 对 **d(s,a)** 线性惩罚
- 惩罚增长较慢

**χ² divergence**：
- 对 **d(s,a)** 指数惩罚
- 惩罚增长更快

**Figure 4** 展示了 **χ²** divergence 在实践中更好地防止了 **RLHF** 中的 **reward hacking**。

### 5.2 理论分析

**为什么 χ² 更有效**：

1. **强惩罚**：**χ²** divergence 更强烈地惩罚分布外的状态-动作对
2. **即使系数低也有效**：由于指数增长，即使使用低系数，**χ²** 也能提供强正则化
3. **鲁棒性**：在 **Figure 5** 中，**χ²** 和 **OM** 正则化在更广的系数范围内表现良好

## 六、ORPO 算法实现

### 6.1 算法概述

**Occupancy-Regularized Policy Optimization (ORPO)** 使用判别器网络来近似 **OM** 散度：

**Algorithm 1: ORPO**

```
1: for iteration i = 1,...,I do
2:   Collect n trajectories D_π from π_θ
3:   Collect n trajectories D_{π_ref} from π_ref
4:   Optimize ϕ via SGD to minimize:
5:     E_{D_π}[log(1+e^{-d̂_ϕ(s,a)})] + E_{D_{π_ref}}[log(1+e^{d̂_ϕ(s,a)})]
6:   Calculate χ̂² = E_{D_π}[e^{d̂_ϕ(s,a)} - 1]
7:   Transform D_π to D_π′ by replacing rewards with:
8:     R′(s,a) = R̃(s,a) - λ/√{χ̂²} (e^{d̂_ϕ(s,a)} - 1)
9:   Optimize θ via SGD to minimize PPO loss L_PPO(D_π′)
10: end for
```

**变量解释**：
- **π_θ**：参数化的策略
- **d̂_ϕ**：判别器网络，近似对数比
- **D_π**, **D_{π_ref}**：从策略和参考策略收集的轨迹
- **R′**：调整后的奖励函数
- **χ̂²**：估计的 **χ²** 散度

### 6.2 判别器训练的理论基础

**目标函数**：
```
ϕ = argmin_ϕ E_{μ_π}[log(1+e^{-d̂_ϕ(s,a)})] + E_{μ_{π_ref}}[log(1+e^{d̂_ϕ(s,a)})]
```

**最优解**：
```
d̂_ϕ(s,a) = log(μ_π(s,a)/μ_{π_ref}(s,a))
```

**χ² 散度的估计**：
```
χ²(μ_π ‖ μ_{π_ref}) = E_{μ_π}[μ_π(s,a)/μ_{π_ref}(s,a) - 1]
                     = E_{μ_π}[e^{d̂_ϕ(s,a)} - 1] ≕ χ̂²
```

### 6.3 调整后的奖励

最终使用的调整奖励：
```
R′(s,a) = R̃(s,a) - λ/√{χ̂²} e^{d̂_ϕ(s,a)}
```

**KL 散度的类似实现**：
```
R′(s,a) = R̃(s,a) - λ d̂_ϕ(s,a)
```

### 6.4 判别器近似的准确性

**Figure 6** 显示：
- 判别器输出很好地近似了占据度量的对数比
- 这使得对 **OM** 散度的准确近似成为可能

## 八、理论补充与深度分析

### 8.1 下界的优化性

**Lemma A.2**：

在 **Theorem 5.1** 的相同条件下，下界 **L(π)** 满足：

```
L(π) ≤ [(1-√{1-r²})/r] √{χ²(μ_π ‖ μ_{π_ref})}
```

这表明：
1. 下界最多是占据度量散度的 **(1-√{1-r²})/r** 倍
2. 这个因子随着 **r** 增加而增加，当 **r → 0** 时渐近于 **r/2**

### 8.2 学习奖励函数的 Correlation

**Lemma A.8**：

令 **R** 是真实奖励函数，**R̃** 是学习到的奖励函数。假设：

```
E_{μ_{π_ref}}[(R(s,a) - R̃(s,a))²] ≤ ε σ_R²
```

那么学习到的奖励函数是 **r-correlated** 代理，其中 **r ≥ 1 - ε**。

**重要含义**：
- 学习到的奖励函数通常具有低均方误差
- 这表明它们是 **correlated proxies**
- 通过在从参考策略采样的训练数据集上进行最小二乘回归可以实现这一点

### 8.3 状态仅占据度量

对于只依赖于状态的环境（即 **R(s,a) = R(s)**），**Theorem 5.1** 也适用于状态仅占据度量：

```
μ_π(s) = (1-γ) E_π[Σᵗ₌₀^∞ γᵗ 𝟙{sₜ = s}]
```

## 九、与其他工作的比较

### 9.1 Reward Hacking 的其他定义

1. **Skalse et al. (2022)**：将 **reward hacking** 定义为代理奖励增加伴随着真实奖励下降
   - 缺点：不满足第一个期望要求（代理是合理的优化目标）
   - 缺点：任何真实奖励下降都被认为是 **reward hacking**，过于严格

2. **Kwa et al. (2024)**：将 **"catastrophic Goodhart"** 定义为优化某个在某些基本策略下具有低预期误差的代理，导致"任意高的[代理]奖励，尽管没有获得比基本策略更多的[真实奖励]"
   - 相似但不捕获真实奖励在 **reward hacking** 发生时的下降
   - 在实践中通常不可能获得任意高的代理奖励

3. **Fluri et al. (2024)**：将 **"error-regret mismatch"** 定义为代理奖励在训练分布上相对于真实奖励具有低误差，但优化代理产生次优策略
   - 相似但让 **reward hacking** 的阈值模糊

### 9.2 其他正则化方法

**当前 RLHF 的实践**：
- 使用 **KL divergence** 在动作分布之间进行惩罚
- 在 **RLHF** 中有效，因为本质上是 **contextual bandit** 问题（**γ = 0**）
- 缺乏理论保证，不清楚在其他环境中是否有效

**替代方法**：
- **Quantilizers** (Taylor, 2016)
- **"Mild" optimization** (Taylor et al., 2020)
- **Impact regularization** (Turner et al., 2020)
- **Constrained RL** (Dalal et al., 2018; Chow et al., 2019)

## 十、局限性与未来工作

### 10.1 理论限制

**Theorem 5.1** 不保证真实奖励能够被增加，但至少允许我们可证明地避免 **reward hacking**。

**Theorem A.3** 显示：
- 存在 **MDP**，其中策略在真实和代理奖励上都改进，但下界不能为正
- 这表明很难指定下界可以为正的一般条件
- 因此，我们依赖实验证据来证明下界通常是可优化的

### 10.2 实践限制

1. **计算复杂性**：**OM** 正则化比 **AD** 正则化更难实现
2. **判别器近似**：需要使用判别器网络来近似 **OM** 散度
3. **超参数调整**：需要选择适当的正则化系数

### 10.3 未来方向

1. **多轮对话**：当 **LLM** 代理在多轮对话或工具使用上进行优化时，**OM** 和 **AD** 正则化将会有所不同
2. **更复杂的参考策略**：探索如何学习更好的参考策略
3. **自适应正则化**：根据代理与真实奖励的相关性动态调整正则化强度

## 十一、结论与影响

### 11.1 主要贡献总结

1. **新定义**：提出了基于代理奖励和未知真实奖励之间相关性的 **reward hacking** 新定义，该相关性在优化代理时会破坏
2. **理论保证**：表明优化相关代理奖励配合 **χ² OM** 正则化可以可证明地改进未知真实奖励
3. **实际实现**：展示了如何实际实现 **χ² OM** 正则化，并证明它优于当前通过 **AD** 正则化防止 **reward hacking** 的标准

### 11.2 对 RLHF 的启示

**当前问题**：
- 目前 **RLHF** 使用启发式的 **KL** 惩罚
- 在当前的 **contextual bandit** 公式中，**OM** 和 **AD** 正则化是等价的

**未来影响**：
- 当 **LLM** 代理在多轮对话或工具使用上进行优化时，它们将会有所不同
- 提供了一条继续确保日益强大的 **AI** 系统安全的原则性路径
- 建议用更原则性的正则化形式替换启发式的 **KL** 惩罚

### 11.3 更广泛的影响

这项工作对以下领域有重要意义：
1. **Safe AI**：提供了一种防止 AI 系统利用奖励函数漏洞的方法
2. **Reinforcement Learning**：为奖励黑客问题提供了理论框架和实际解决方案
3. **Value Alignment**：为对齐复杂目标提供了新的工具
